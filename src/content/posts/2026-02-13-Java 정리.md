# Java 정리 (2026-02-13)

간단한 Java 개요와 빠르게 시작할 수 있는 프로젝트 예시를 정리합니다.

## 1. Java 개요
- 플랫폼: JVM(Java Virtual Machine) 위에서 동작
- 주요 특징: 객체지향, 강한 타입, 풍부한 표준 라이브러리
- 빌드 도구: `Maven`, `Gradle`
- 패키지 및 모듈: `package` 키워드로 네임스페이스 관리

## 2. 개발 환경
- JDK 설치(일반적으로 OpenJDK 17+ 권장)
- IDE: IntelliJ IDEA, VS Code (+Extension)
- 빌드/실행 예시:

Maven으로 컴파일/실행:

```bash
mvn compile
mvn exec:java -Dexec.mainClass="com.example.Main"
```

Gradle으로 빌드/실행 (Gradle Wrapper 사용):

```bash
./gradlew build
./gradlew run
```

## 3. 핵심 문법 요약
- 진입점: `public static void main(String[] args)`
- 클래스/필드/메서드, 접근 제어자(`public`, `private`, `protected`)
- 기본 자료형: `int`, `long`, `double`, `boolean`, `char`
- 컬렉션: `List`, `Set`, `Map` (컬렉션 프레임워크)
- 예외 처리: `try` / `catch` / `finally` / `throws`
- 람다 & 스트림 (Java 8+)

## 4. 간단한 Maven 콘솔 프로젝트 예시

프로젝트 구조:

```
simple-maven-app/
├─ pom.xml
└─ src/main/java/com/example/Main.java
```

`pom.xml` (필수 최소 설정):

```xml
<project xmlns="http://maven.apache.org/POM/4.0.0" ...>
  <modelVersion>4.0.0</modelVersion>
  <groupId>com.example</groupId>
  <artifactId>simple-maven-app</artifactId>
  <version>1.0.0</version>
  <properties>
    <maven.compiler.source>17</maven.compiler.source>
    <maven.compiler.target>17</maven.compiler.target>
  </properties>
</project>
```

`Main.java` 예시 — 간단한 ToDo 콘솔 앱 스켈레톤:

```java
package com.example;

import java.util.ArrayList;
import java.util.List;
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        List<String> todos = new ArrayList<>();
        System.out.println("간단한 ToDo 앱 (종료: exit)");
        while (true) {
            System.out.print("> ");
            String line = sc.nextLine().trim();
            if (line.equalsIgnoreCase("exit")) break;
            if (line.startsWith("add ")) {
                todos.add(line.substring(4));
                System.out.println("추가됨");
            } else if (line.equals("list")) {
                for (int i = 0; i < todos.size(); i++) {
                    System.out.printf("%d. %s\n", i + 1, todos.get(i));
                }
            } else if (line.startsWith("remove ")) {
                try { int idx = Integer.parseInt(line.split(" ")[1]) - 1; todos.remove(idx); System.out.println("삭제됨"); }
                catch (Exception e) { System.out.println("잘못된 인덱스"); }
            } else {
                System.out.println("명령: add <text>, list, remove <index>, exit");
            }
        }
        sc.close();
    }
}
```

## 5. 간단한 Spring Boot 예시 (REST API)

Spring Initializr 또는 Gradle/Maven으로 `spring-boot-starter-web` 추가 후, 아래와 같은 컨트롤러 작성으로 빠르게 API를 만들 수 있습니다.

`build.gradle` 또는 `pom.xml`에 `spring-boot-starter-web` 의존성 추가.

`HelloController.java`:

```java
package com.example;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {
    @GetMapping("/hello")
    public String hello() {
        return "Hello, Spring Boot!";
    }
}
```

애플리케이션 실행 후 `http://localhost:8080/hello` 호출하면 `Hello, Spring Boot!` 응답을 받습니다.

## 7. 초보자용: JDK 설치 및 환경 설정 (Windows / macOS / Linux)

Java를 처음 설치하는 분들을 위해 단계별로 정리합니다.

1) JDK 설치
- 권장: OpenJDK 17 또는 21
- Windows: AdoptOpenJDK / Temurin 인스톨러(.msi) 사용 권장
- macOS/Linux: `sdkman`(권장) 또는 패키지 매니저(예: `brew install openjdk@17`) 사용

2) 환경 변수 설정 (`JAVA_HOME` + `PATH`)
- Windows (GUI):
    - `제어판` → `시스템` → `고급 시스템 설정` → `환경 변수`
    - 사용자 변수에 `JAVA_HOME`을 JDK 설치 경로로 추가(e.g. C:\Program Files\Java\jdk-17)
    - `Path`에 `%JAVA_HOME%\bin` 추가
    - PowerShell로 설정하려면:

```powershell
[Environment]::SetEnvironmentVariable("JAVA_HOME", "C:\Program Files\Java\jdk-17", "User")
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";$env:JAVA_HOME\\bin", "User")
```

- macOS / Linux (bash/zsh):

```bash
export JAVA_HOME="/Library/Java/JavaVirtualMachines/jdk-17.jdk/Contents/Home" # macOS 예시
export PATH="$JAVA_HOME/bin:$PATH"
```

3) 설치 확인

```bash
java -version
javac -version
```

4) Maven / Gradle 설치
- Maven: 배포판(zip) 압축을 풀고 `M2_HOME`을 설정하거나 `.../apache-maven/bin`을 `Path`에 추가합니다. Windows에서는 Chocolatey로 설치(`choco install maven`)하면 편리합니다.

    Windows PowerShell로 Maven 설치 후 확인 예시:

    ```powershell
    choco install maven -y
    mvn -v
    ```

- Gradle: 권장 방식은 프로젝트별 `Gradle Wrapper`(gradlew) 사용 — 로컬에 Gradle을 전역 설치할 필요가 없습니다. 전역 설치가 필요하면 공식 배포판을 설치하고 `GRADLE_HOME`을 설정하거나 Chocolatey(`choco install gradle`) 사용.

    전역 설치 확인:

    ```powershell
    gradle -v
    ```

    Gradle Wrapper 사용(프로젝트에서):

    ```powershell
    gradlew.bat build   # Windows에서 실행
    ```

1) VS Code / IDE 설정
- 권장 IDE: IntelliJ IDEA (Java 개발에 최적)
- VS Code: `Extension Pack for Java`, `Language Support for Java(TM) by Red Hat`, `Debugger for Java`, `Maven for Java` 등을 설치

1) Windows에서 흔한 문제와 해결
- 여러 JDK가 설치되어 있을 때 경로 충돌 발생 가능: `java -version`과 `where java`로 확인
- 환경 변수 변경 후에는 새 터미널을 열어야 반영됩니다

1) Mac/Linux 대안: SDKMAN

```bash
curl -s "https://get.sdkman.io" | bash
source "$HOME/.sdkman/bin/sdkman-init.sh"
sdk install java 17.0.8-tem
```

8) 빠른 체크리스트
- `java -version`, `javac -version`, `mvn -v` 또는 `./gradlew -v` 실행
- IDE에서 JDK가 올바르게 선택되었는지 확인

필요하면 운영체제별 스크린샷, PowerShell 명령 예시, 또는 `pom.xml`/`build.gradle`과 함께 실제 프로젝트 초기화까지 진행해 드리겠습니다.

## 8. 문법 상세 및 예제

### 변수와 타입
- Java는 정적 타입 언어입니다. 명시적 타입 선언 또는 `var`(지역 변수 타입 추론)를 사용합니다.

```java
String name = "Geom";
int age = 32;
var list = new ArrayList<String>();
```

### 제어문
- `if`, `for`, `while`, `switch` 등은 JS와 문법이 유사하지만 `switch`에서 `break`를 빠뜨리면 fall-through가 발생합니다.

```java
for (int i = 0; i < 5; i++) {
    System.out.println(i);
}
```

### 메서드와 오버로딩
- Java는 같은 이름의 메서드를 매개변수 타입/개수로 구분하는 오버로딩을 지원합니다.

```java
public int add(int a, int b) { return a + b; }
public double add(double a, double b) { return a + b; }
```

### 클래스/생성자/접근 제어

```java
public class User {
    private String id;
    public User(String id) { this.id = id; }
    public String getId() { return id; }
}
```

### 인터페이스와 구현

```java
public interface Greeter { String greet(String who); }
public class SimpleGreeter implements Greeter {
    public String greet(String who) { return "Hello " + who; }
}
```

### 제네릭

```java
Map<String, List<Integer>> map = new HashMap<>();
```

### 예외 처리

```java
try {
    risky();
} catch (IOException e) {
    e.printStackTrace();
} finally {
    // 정리
}
```

### 스트림 API

```java
List<String> filtered = users.stream()
    .map(User::getId)
    .filter(id -> id.length() > 3)
    .toList();
```

### 동시성 간단 예


## 8. Java vs JavaScript — 주요 차이 정리

- 타입 및 컴파일
    - Java: 컴파일 타임에 타입 검증. 런타임 이전에 많은 오류를 잡을 수 있음.
    - JS: 런타임에 타입이 결정. TypeScript를 사용하면 JS에 정적 타입을 추가할 수 있음.

- 함수와 클래스
    - JS는 함수가 1급 객체(함수를 변수에 할당, 인자로 전달 가능). Java는 메서드가 클래스에 속하지만 람다로 함수형 스타일을 지원.

- 비동기 모델
    - JS: 이벤트 루프 + 논블로킹 I/O(브라우저/Node). `Promise`/`async-await` 기본.
    - Java: 스레드 기반(블로킹/논블로킹 API 공존). `CompletableFuture`로 비동기 흐름을 작성.

- 널 처리
    - Java: `null` 존재. `Optional<T>`로 null-safe 코드 권장.
    - JS: `null`과 `undefined` 존재, 느슨한 비교(`==`)는 주의.

- 표준 라이브러리 범위
    - Java: 광범위한 표준 API(파일, 네트워크, 동시성, 컬렉션 등).
    - JS: 실행 환경(브라우저/Node)에 따라 제공되는 API가 다름.

### 예시 비교: 클래스

Java:

```java
public class Counter {
    private int c = 0;
    public synchronized void inc() { c++; }
    public int get() { return c; }
}
```

JavaScript (Node / modern JS):

```javascript
class Counter {
    constructor(){ this.c = 0 }
    inc(){ this.c++ }
    get(){ return this.c }
}
```

### 비동기 비교: Promise vs CompletableFuture

JavaScript:

```javascript
async function fetchData(){
    const res = await fetch(url);
    return res.json();
}
```

Java:

```java
CompletableFuture.supplyAsync(() -> httpClient.send(...))
        .thenApply(resp -> parse(resp));
```

## 9. 간단한 Spring Boot CRUD 예제 (조금 더 자세히)

작은 ToDo REST 서비스 예시 파일 구조와 핵심 코드 스니펫:

`Todo.java` (엔티티/모델):

```java
package com.example.todo.model;
public class Todo {
    private Long id;
    private String title;
    private boolean done;
    // 생성자, 게터/세터
}
```

`TodoService.java` (간단한 인메모리 서비스):

```java
@Service
public class TodoService {
    private final Map<Long, Todo> store = new ConcurrentHashMap<>();
    private final AtomicLong seq = new AtomicLong(1);

    public List<Todo> list() { return new ArrayList<>(store.values()); }
    public Todo create(Todo t) { long id = seq.getAndIncrement(); t.setId(id); store.put(id, t); return t; }
}
```

`TodoController.java` (요청 처리):

```java
@RestController
@RequestMapping("/api/todos")
public class TodoController {
    private final TodoService svc;
    public TodoController(TodoService svc){ this.svc = svc; }

    @GetMapping
    public List<Todo> all(){ return svc.list(); }

    @PostMapping
    public Todo create(@RequestBody Todo t){ return svc.create(t); }
}
```

`pom.xml`에 `spring-boot-starter-web`과 `spring-boot-starter-test`를 추가하면 기본 실행과 테스트가 가능합니다.
