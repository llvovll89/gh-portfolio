# Docker & Docker Compose 정리

**Docker**는 애플리케이션을 컨테이너(Container)라는 격리된 환경에서 실행할 수 있게 해주는 플랫폼입니다.
"내 컴퓨터에서는 됐는데요?" 문제를 해결하고, 개발/테스트/운영 환경을 일치시킬 수 있습니다.

## 1. Docker란?

### 1.1 가상머신 vs 컨테이너

| 구분 | 가상머신(VM) | 컨테이너(Container) |
|------|------------|-------------------|
| **격리 단위** | OS 전체 | 프로세스 |
| **크기** | 수 GB | 수 MB ~ 수백 MB |
| **시작 시간** | 수 분 | 수 초 이내 |
| **리소스** | 많이 사용 | 호스트 OS 공유 |
| **이식성** | 낮음 | 높음 |

컨테이너는 호스트 OS의 커널을 공유하면서 프로세스만 격리하므로 훨씬 가볍고 빠릅니다.

### 1.2 Docker의 장점

- **환경 일관성**: 개발/스테이징/운영 환경이 동일
- **빠른 배포**: 이미지만 있으면 어디서든 동일하게 실행
- **격리**: 서비스 간 의존성 충돌 없음
- **스케일링**: 컨테이너 수를 늘려 손쉬운 수평 확장

## 2. 핵심 개념

### 2.1 이미지 (Image)

컨테이너 실행에 필요한 **읽기 전용 템플릿**. 레이어(Layer) 구조로 쌓입니다.

```
Ubuntu 레이어
  + Node.js 레이어
    + 앱 코드 레이어
      = 최종 이미지
```

### 2.2 컨테이너 (Container)

이미지를 실행한 **인스턴스**. 같은 이미지로 여러 컨테이너를 만들 수 있습니다.

```
이미지 (붕어빵 틀) → 컨테이너 (붕어빵)
```

### 2.3 레지스트리 (Registry)

이미지를 저장하고 배포하는 저장소. 대표적으로 **Docker Hub**, AWS ECR, GitHub Container Registry가 있습니다.

### 2.4 Dockerfile

이미지를 만드는 **설계도(빌드 스크립트)**.

## 3. 설치 및 기본 명령어

### 3.1 설치

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) 설치 (Windows/macOS)
- Linux: `sudo apt install docker.io`

```bash
# 설치 확인
docker --version
docker compose version
```

### 3.2 자주 쓰는 명령어 요약

```bash
# 이미지
docker pull nginx            # 이미지 다운로드
docker images                # 이미지 목록
docker rmi nginx             # 이미지 삭제

# 컨테이너
docker run nginx             # 컨테이너 실행
docker ps                    # 실행 중인 컨테이너
docker ps -a                 # 모든 컨테이너
docker stop <id>             # 컨테이너 중지
docker rm <id>               # 컨테이너 삭제

# 로그 & 접속
docker logs <id>             # 로그 확인
docker exec -it <id> bash    # 컨테이너 접속
```

## 4. 이미지(Image) 다루기

### 4.1 이미지 검색 및 다운로드

docker search node           # Docker Hub 검색

```

### 4.2 이미지 태그

```
이미지명:태그
node:20          # Node.js 20 LTS

```bash

docker build -t my-app:1.0 -f Dockerfile.prod .  # 파일 지정
```

## 5. 컨테이너(Container) 다루기

### 5.1 docker run 주요 옵션

```bash
docker run \
  -d \                        # 백그라운드 실행 (detached)
  -p 3000:3000 \              # 포트 매핑 (호스트:컨테이너)
  -v $(pwd):/app \            # 볼륨 마운트
  -e NODE_ENV=production \    # 환경변수
  --name my-container \       # 컨테이너 이름
  --rm \                      # 종료 시 자동 삭제
  my-app:1.0
```

### 5.2 컨테이너 상태 관리

```bash
docker start my-container    # 중지된 컨테이너 시작
docker stop my-container     # 정상 종료 (SIGTERM)
docker kill my-container     # 강제 종료 (SIGKILL)
docker restart my-container  # 재시작
docker pause my-container    # 일시 중지
docker unpause my-container  # 재개
```

### 5.3 컨테이너 내부 접속

```bash
docker exec -it my-container bash   # bash 쉘 접속
docker exec -it my-container sh     # sh 쉘 (alpine)
docker exec my-container ls /app    # 단일 명령 실행
```

## 6. Dockerfile 작성법

### 6.1 기본 구조

```dockerfile
# 베이스 이미지
FROM node:20-alpine

# 작업 디렉토리 설정
WORKDIR /app

# 의존성 파일 먼저 복사 (캐시 활용)
COPY package*.json ./

# 의존성 설치
RUN npm ci --only=production

# 소스 코드 복사
COPY . .

# 빌드
RUN npm run build

# 포트 노출 (문서용)
EXPOSE 3000

# 실행 명령
CMD ["node", "dist/index.js"]
```

### 6.2 주요 명령어

| 명령어 | 설명 |
|--------|------|
| `FROM` | 베이스 이미지 지정 |
| `WORKDIR` | 작업 디렉토리 설정 |
| `COPY` | 파일 복사 (호스트 → 이미지) |
| `ADD` | COPY + URL/압축 해제 지원 |
| `RUN` | 빌드 시 명령 실행 (레이어 생성) |
| `CMD` | 컨테이너 시작 시 기본 명령 |
| `ENTRYPOINT` | 컨테이너 시작 시 고정 명령 |
| `ENV` | 환경변수 설정 |
| `ARG` | 빌드 시 변수 |
| `EXPOSE` | 포트 문서화 |

### 6.3 멀티 스테이지 빌드 (Multi-stage Build)

빌드 환경과 실행 환경을 분리하여 최종 이미지 크기를 줄입니다.

```dockerfile
# --- 빌드 스테이지 ---
FROM node:20-alpine AS builder

WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

# --- 실행 스테이지 ---
FROM node:20-alpine AS runner

WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules

EXPOSE 3000
CMD ["node", "dist/index.js"]
```

> 빌드 도구(TypeScript 컴파일러 등)가 최종 이미지에 포함되지 않아 이미지가 훨씬 가벼워집니다.

### 6.4 .dockerignore

`.gitignore`처럼 이미지 빌드 시 제외할 파일을 지정합니다.

```
node_modules
.git
.env
dist
*.log
```

## 7. 볼륨(Volume) & 네트워크

### 7.1 볼륨 (Volume)

컨테이너가 삭제되어도 데이터를 **영속**시킵니다.

```bash
# 볼륨 생성
docker volume create my-data

# 볼륨 마운트
docker run -v my-data:/var/lib/postgresql/data postgres

# 바인드 마운트 (호스트 디렉토리 직접 연결)
docker run -v $(pwd)/data:/app/data my-app
```

| 종류 | 명령어 | 용도 |
|------|--------|------|
| **Named Volume** | `-v my-data:/path` | DB 데이터 영속화 |
| **Bind Mount** | `-v /host/path:/path` | 개발 시 코드 실시간 반영 |
| **tmpfs** | `--tmpfs /path` | 임시 데이터 (메모리) |

### 7.2 네트워크 (Network)

```bash
# 네트워크 생성
docker network create my-network

# 네트워크에 컨테이너 연결
docker run --network my-network --name api my-app
docker run --network my-network --name db postgres

# 같은 네트워크의 컨테이너는 이름으로 통신 가능
# api 컨테이너에서 → postgres://db:5432 접속 가능
```

## 8. Docker Compose

여러 컨테이너를 **하나의 설정 파일**로 정의하고 함께 실행합니다.

### 8.1 docker-compose.yml 기본 구조

```yaml
version: "3.9"

services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=development
    volumes:
      - .:/app
      - /app/node_modules
    depends_on:
      - db

  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: password
      POSTGRES_DB: mydb
    volumes:
      - postgres-data:/var/lib/postgresql/data

volumes:
  postgres-data:
```

### 8.2 주요 명령어

```bash
docker compose up           # 서비스 시작 (포그라운드)
docker compose up -d        # 백그라운드 실행
docker compose down         # 서비스 중지 및 삭제
docker compose down -v      # 볼륨까지 삭제

docker compose logs -f      # 로그 스트리밍
docker compose ps           # 서비스 상태
docker compose exec app sh  # 컨테이너 접속
docker compose restart app  # 특정 서비스 재시작

docker compose build        # 이미지 빌드
docker compose pull         # 이미지 업데이트
```

### 8.3 환경별 Compose 파일 분리

```bash
# 기본 설정
docker-compose.yml

# 개발 환경 오버라이드
docker-compose.override.yml  # 자동으로 병합됨

# 운영 환경
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d
```

## 9. 실전 예제 - React + Node + PostgreSQL

### 9.1 프로젝트 구조

```
my-project/
├── frontend/
│   ├── Dockerfile
│   └── ...
├── backend/
│   ├── Dockerfile
│   └── ...
└── docker-compose.yml
```

### 9.2 backend/Dockerfile

```dockerfile
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:20-alpine AS runner
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
EXPOSE 4000
CMD ["node", "dist/index.js"]
```

### 9.3 frontend/Dockerfile

```dockerfile
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM nginx:alpine AS runner
COPY --from=builder /app/dist /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### 9.4 docker-compose.yml

```yaml
version: "3.9"

services:
  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend

  backend:
    build: ./backend
    ports:
      - "4000:4000"
    environment:
      DATABASE_URL: postgresql://user:password@db:5432/mydb
      JWT_SECRET: ${JWT_SECRET}
    depends_on:
      db:
        condition: service_healthy

  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: password
      POSTGRES_DB: mydb
    volumes:
      - postgres-data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U user -d mydb"]
      interval: 10s
      timeout: 5s
      retries: 5

volumes:
  postgres-data:
```

## 10. Docker Hub & 이미지 배포

```bash
# 로그인
docker login

# 이미지 태그 (사용자명/이미지명:태그)
docker tag my-app:1.0 username/my-app:1.0

# 푸시
docker push username/my-app:1.0

# 풀
docker pull username/my-app:1.0
```

### GitHub Actions를 활용한 자동 배포

```yaml
# .github/workflows/docker.yml
name: Build and Push Docker Image

on:
  push:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Login to Docker Hub
        uses: docker/login-action@v3
        with:
          username: ${{ secrets.DOCKERHUB_USERNAME }}
          password: ${{ secrets.DOCKERHUB_TOKEN }}

      - name: Build and push
        uses: docker/build-push-action@v5
        with:
          push: true
          tags: username/my-app:latest
```

## 11. Best Practices

### 11.1 이미지 최적화

```dockerfile
# 경량 베이스 이미지 사용
FROM node:20-alpine  # ubuntu 대신 alpine

# RUN 명령어 체이닝 (레이어 최소화)
RUN apt-get update && \
    apt-get install -y curl && \
    rm -rf /var/lib/apt/lists/*

# 의존성을 먼저 복사 (캐시 활용)
COPY package*.json ./
RUN npm ci
COPY . .  # 코드 변경 시 이 레이어부터만 재빌드

# non-root 사용자 실행 (보안)
RUN addgroup -S appgroup && adduser -S appuser -G appgroup
USER appuser
```

### 11.2 환경변수 관리

```bash
# .env 파일 사용 (절대 커밋 금지!)
# .env
DB_PASSWORD=secret
JWT_SECRET=mysecret

# docker-compose.yml에서 참조
env_file:
  - .env
```

### 11.3 헬스체크 설정

```dockerfile
HEALTHCHECK --interval=30s --timeout=10s --retries=3 \
  CMD curl -f http://localhost:3000/health || exit 1
```

### 11.4 주요 Best Practices 요약

| 항목 | 권장 |
|------|------|
| 베이스 이미지 | `alpine` 계열 사용으로 경량화 |
| 태그 | `latest` 대신 명시적 버전 태그 사용 |
| 레이어 | 자주 변하는 파일은 나중에 COPY |
| 보안 | non-root 사용자로 실행 |
| 시크릿 | `.env` + `.dockerignore`로 관리, 절대 이미지에 포함 X |
| 멀티 스테이지 | 빌드/실행 환경 분리로 이미지 최소화 |
| 헬스체크 | `HEALTHCHECK` 또는 Compose `healthcheck` 설정 |
