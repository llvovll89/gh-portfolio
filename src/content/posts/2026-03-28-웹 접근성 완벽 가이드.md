웹 접근성(Accessibility, a11y)은 장애를 가진 사용자를 포함한 **모든 사람**이 웹을 동등하게 이용할 수 있도록 만드는 것입니다. 전 세계 인구의 약 15%가 어떤 형태로든 장애를 가지고 있으며, 접근성은 법적 의무이자 더 나은 사용자 경험으로 이어집니다. 접근성이 좋은 사이트는 SEO 점수도 높아집니다.

> 목표: WCAG 2.1 기준을 이해하고, 시맨틱 HTML과 ARIA를 올바르게 사용해 모든 사용자가 사용할 수 있는 컴포넌트를 구현할 수 있다.

## 1. WCAG 4대 원칙 (POUR)

WCAG(Web Content Accessibility Guidelines)는 접근성 표준입니다. 현재 기준은 WCAG 2.1이며, 4가지 핵심 원칙(POUR)으로 구성됩니다.

| 원칙 | 의미 | 예시 |
|------|------|------|
| **P**erceivable (인식 가능) | 모든 콘텐츠를 감지할 수 있어야 함 | 이미지에 대체 텍스트, 자막 제공 |
| **O**perable (운용 가능) | 모든 기능을 사용할 수 있어야 함 | 키보드만으로 모든 기능 사용 가능 |
| **U**nderstandable (이해 가능) | 콘텐츠와 UI가 이해 가능해야 함 | 명확한 레이블, 오류 메시지 |
| **R**obust (견고성) | 다양한 보조 기술과 호환되어야 함 | 유효한 HTML, ARIA 올바른 사용 |

### 적합성 수준

- **Level A**: 최소 요구사항 (반드시 충족)
- **Level AA**: 권장 표준 (대부분의 법규 기준)
- **Level AAA**: 최고 수준 (일부 콘텐츠에만 적용 권장)

## 2. 시맨틱 HTML — 접근성의 기반

접근성의 가장 강력한 도구는 **올바른 HTML 요소 선택**입니다. 시맨틱 요소는 브라우저와 보조 기술에 역할 정보를 자동으로 전달합니다.

### 나쁜 예 vs 좋은 예

```html
<!-- Bad — div로 모든 것을 구성 -->
<div class="header">
  <div class="nav">
    <div class="nav-item" onclick="navigate()">홈</div>
    <div class="nav-item" onclick="navigate()">블로그</div>
  </div>
</div>
<div class="main">
  <div class="article">
    <div class="title">제목</div>
    <div class="content">본문...</div>
  </div>
</div>

<!-- Good — 시맨틱 HTML -->
<header>
  <nav aria-label="주 내비게이션">
    <ul>
      <li><a href="/">홈</a></li>
      <li><a href="/blog">블로그</a></li>
    </ul>
  </nav>
</header>
<main>
  <article>
    <h1>제목</h1>
    <p>본문...</p>
  </article>
</main>
```

### 랜드마크 요소 활용

| 요소 | 역할(role) | 용도 |
|------|-----------|------|
| `<header>` | banner | 페이지/섹션 헤더 |
| `<nav>` | navigation | 내비게이션 |
| `<main>` | main | 주요 콘텐츠 (페이지당 1개) |
| `<aside>` | complementary | 보조 콘텐츠 |
| `<footer>` | contentinfo | 페이지/섹션 푸터 |
| `<section>` | region | 의미 있는 섹션 (aria-label 권장) |
| `<article>` | article | 독립적인 콘텐츠 단위 |

### 제목(Heading) 계층 구조

```html
<!-- Bad — 헤딩 계층 건너뜀 -->
<h1>페이지 제목</h1>
<h3>섹션 제목</h3> <!-- h2 없이 h3 사용 -->

<!-- Good — 순서대로 -->
<h1>페이지 제목</h1>
  <h2>섹션 제목</h2>
    <h3>소제목</h3>
  <h2>다른 섹션</h2>
```

> 헤딩은 시각적 크기가 아닌 **문서 구조**를 위해 사용하세요. 크기는 CSS로 조정하세요.

## 3. ARIA — 시맨틱이 부족할 때

ARIA(Accessible Rich Internet Applications)는 시맨틱 HTML로 표현하기 어려운 복잡한 UI 패턴에 접근성 정보를 추가합니다.

### ARIA의 황금률

> **ARIA를 사용하지 않는 것이 사용하는 것보다 낫다.** 시맨틱 HTML로 해결할 수 있다면 ARIA는 필요 없습니다.

### 핵심 ARIA 속성

```html
<!-- aria-label: 보이지 않는 레이블 -->
<button aria-label="검색창 닫기">✕</button>

<!-- aria-labelledby: 다른 요소를 레이블로 참조 -->
<section aria-labelledby="section-title">
  <h2 id="section-title">최근 포스트</h2>
</section>

<!-- aria-describedby: 추가 설명 참조 -->
<input
  type="password"
  aria-describedby="pw-hint"
/>
<p id="pw-hint">8자 이상, 특수문자 포함</p>

<!-- aria-hidden: 보조 기술에서 숨기기 -->
<span aria-hidden="true">🎉</span>

<!-- aria-expanded: 펼침/접힘 상태 -->
<button aria-expanded="false" aria-controls="menu">메뉴</button>
<ul id="menu" hidden>...</ul>

<!-- aria-current: 현재 위치 표시 -->
<nav>
  <a href="/" aria-current="page">홈</a>
  <a href="/blog">블로그</a>
</nav>

<!-- aria-live: 동적 콘텐츠 알림 -->
<div aria-live="polite" aria-atomic="true">
  장바구니에 상품이 추가되었습니다.
</div>
```

### ARIA 역할(role) 예시

```html
<!-- 탭 인터페이스 -->
<div role="tablist" aria-label="설정 탭">
  <button role="tab" aria-selected="true" aria-controls="panel-1" id="tab-1">
    일반
  </button>
  <button role="tab" aria-selected="false" aria-controls="panel-2" id="tab-2">
    보안
  </button>
</div>
<div role="tabpanel" id="panel-1" aria-labelledby="tab-1">
  일반 설정 내용
</div>
<div role="tabpanel" id="panel-2" aria-labelledby="tab-2" hidden>
  보안 설정 내용
</div>
```

## 4. 키보드 내비게이션

마우스를 사용할 수 없는 사용자는 키보드로만 웹을 탐색합니다.

### 핵심 키보드 단축키

| 키 | 동작 |
|----|------|
| `Tab` | 다음 포커스 가능한 요소로 이동 |
| `Shift + Tab` | 이전 요소로 이동 |
| `Enter` / `Space` | 버튼/링크 활성화 |
| `Arrow keys` | 라디오 버튼, 탭, 슬라이더 내 이동 |
| `Escape` | 모달/메뉴 닫기 |
| `Home` / `End` | 목록의 처음/끝으로 이동 |

### 포커스 관리

```css
/* 포커스 표시를 절대 숨기지 마세요 */
/* Bad */
*:focus { outline: none; }

/* Good — 커스텀 포커스 스타일 */
*:focus-visible {
  outline: 2px solid #3b82f6;
  outline-offset: 2px;
  border-radius: 4px;
}
```

### tabindex 올바른 사용

```html
<!-- tabindex="0": 자연스러운 탭 순서에 포함 -->
<div tabindex="0" role="button" onclick="doSomething()">
  커스텀 버튼
</div>

<!-- tabindex="-1": 프로그래매틱 포커스만 가능 (탭 순서에서 제외) -->
<div tabindex="-1" id="modal-content">
  모달 내용
</div>

<!-- tabindex 양수 값은 사용 금지 — 탭 순서를 예측 불가능하게 만듦 -->
<!-- Bad -->
<button tabindex="3">버튼 3</button>
<button tabindex="1">버튼 1</button>
```

### 포커스 트랩 (모달)

```tsx
function Modal({ isOpen, onClose, children }) {
  const modalRef = useRef<HTMLDivElement>(null);

  useEffect(() => {
    if (!isOpen) return;

    // 모달 열릴 때 첫 번째 포커스 가능한 요소로 이동
    const focusable = modalRef.current?.querySelectorAll(
      'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
    );
    (focusable?.[0] as HTMLElement)?.focus();

    // 포커스 트랩
    function handleKeyDown(e: KeyboardEvent) {
      if (e.key === 'Escape') onClose();
      if (e.key !== 'Tab') return;

      const first = focusable?.[0] as HTMLElement;
      const last = focusable?.[focusable.length - 1] as HTMLElement;

      if (e.shiftKey && document.activeElement === first) {
        e.preventDefault();
        last?.focus();
      } else if (!e.shiftKey && document.activeElement === last) {
        e.preventDefault();
        first?.focus();
      }
    }

    document.addEventListener('keydown', handleKeyDown);
    return () => document.removeEventListener('keydown', handleKeyDown);
  }, [isOpen, onClose]);

  if (!isOpen) return null;

  return (
    <div
      role="dialog"
      aria-modal="true"
      aria-labelledby="modal-title"
      ref={modalRef}
    >
      <h2 id="modal-title">모달 제목</h2>
      {children}
      <button onClick={onClose}>닫기</button>
    </div>
  );
}
```

## 5. 색상 대비와 시각적 접근성

### WCAG 색상 대비 기준

| 콘텐츠 유형 | Level AA | Level AAA |
|------------|----------|-----------|
| 일반 텍스트 | 4.5:1 | 7:1 |
| 큰 텍스트 (18pt 이상 또는 14pt Bold) | 3:1 | 4.5:1 |
| UI 컴포넌트, 그래픽 | 3:1 | — |

```css
/* Bad — 연한 회색 텍스트 (대비 낮음) */
.text { color: #999; background: #fff; } /* 2.85:1 — 실패 */

/* Good — 충분한 대비 */
.text { color: #595959; background: #fff; } /* 7:1 — AAA */
.text-secondary { color: #767676; background: #fff; } /* 4.54:1 — AA */
```

### 색상에만 의존하지 않기

```html
<!-- Bad — 색상만으로 상태 표시 -->
<span style="color: red">오류 메시지</span>
<span style="color: green">성공 메시지</span>

<!-- Good — 아이콘 + 텍스트 조합 -->
<span role="alert">
  <svg aria-hidden="true"><!-- 오류 아이콘 --></svg>
  <strong>오류:</strong> 이메일 형식이 올바르지 않습니다.
</span>

<span>
  <svg aria-hidden="true"><!-- 체크 아이콘 --></svg>
  <strong>성공:</strong> 저장되었습니다.
</span>
```

### 움직임과 애니메이션 (prefers-reduced-motion)

```css
/* 기본 애니메이션 */
.spinner {
  animation: spin 1s linear infinite;
}

/* 움직임 감소 설정 사용자 존중 */
@media (prefers-reduced-motion: reduce) {
  .spinner {
    animation: none;
  }

  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

### 이미지 대체 텍스트 가이드

```html

```

### 동영상 접근성

```html
<video controls>
  <source src="video.mp4" type="video/mp4" />

  <!-- 자막 (청각 장애 사용자) -->
  <track kind="subtitles" src="subtitles-ko.vtt" srclang="ko" label="한국어" default />

  <!-- 음성 해설 (시각 장애 사용자) -->

</video>
```

## 7. 폼 접근성

### 레이블 연결

```html
<!-- Good — label의 for 속성으로 연결 -->
<label for="email">이메일 *</label>
<input type="email" id="email" required aria-describedby="email-error" />
<span id="email-error" role="alert" aria-live="polite"></span>

<!-- Good — label로 감싸기 -->
<label>
  비밀번호
  <input type="password" />
</label>

<!-- Bad — placeholder만 사용 (레이블 없음) -->
<input type="email" placeholder="이메일 입력" />
```

### 오류 메시지 접근성

```tsx
function EmailInput() {
  const [value, setValue] = useState('');
  const [error, setError] = useState('');

  function validate(val: string) {
    if (!val) return '이메일을 입력해주세요.';
    if (!/^[^@]+@[^@]+\.[^@]+$/.test(val)) return '올바른 이메일 형식이 아닙니다.';
    return '';
  }

  return (
    <div>
      <label htmlFor="email">
        이메일
        <span aria-hidden="true"> *</span>
        <span className="sr-only"> (필수)</span>
      </label>
      <input
        id="email"
        type="email"
        value={value}
        onChange={e => setValue(e.target.value)}
        onBlur={e => setError(validate(e.target.value))}
        aria-required="true"
        aria-invalid={error ? 'true' : 'false'}
        aria-describedby={error ? 'email-error' : undefined}
      />
      {error && (
        <span id="email-error" role="alert" style={{ color: '#dc2626' }}>
          <span aria-hidden="true">⚠ </span>
          {error}
        </span>
      )}
    </div>
  );
}
```

### 스크린리더 전용 텍스트

## 8. 동적 콘텐츠와 라이브 리전

스크린리더는 기본적으로 DOM 변경을 감지하지 못합니다. `aria-live`로 알림을 보냅니다.

```html
<!-- polite: 현재 읽기가 끝난 후 알림 -->
<div aria-live="polite" aria-atomic="true" class="sr-only">
  <!-- JS로 메시지 삽입 -->
</div>

<!-- assertive: 즉시 알림 (중요한 오류에만 사용) -->
<div aria-live="assertive" role="alert">
  <!-- 오류 메시지 -->
</div>
```

```tsx
function Toast({ message }: { message: string }) {
  return (
    // role="status"는 aria-live="polite"와 동일
    <div role="status" aria-live="polite" aria-atomic="true">
      {message}
    </div>
  );
}

// 스피너 — 로딩 상태 알림
function LoadingSpinner({ isLoading }: { isLoading: boolean }) {
  return (
    <div aria-live="polite" aria-busy={isLoading}>
      {isLoading && (
        <>
          <div className="spinner" aria-hidden="true" />
          <span className="sr-only">로딩 중...</span>
        </>
      )}
    </div>
  );
}
```

### jest-axe로 자동화 테스트

## 10. React에서 접근성 구현 패턴

### Headless UI 라이브러리 활용

접근성이 내장된 headless 컴포넌트 라이브러리를 사용하면 ARIA 처리를 직접 하지 않아도 됩니다.

```bash
npm install @radix-ui/react-dialog @radix-ui/react-tabs
```

```tsx
import * as Dialog from '@radix-ui/react-dialog';
import * as Tabs from '@radix-ui/react-tabs';

// 접근성이 내장된 다이얼로그
function AccessibleModal() {
  return (
    <Dialog.Root>
      <Dialog.Trigger asChild>
        <button>모달 열기</button>
      </Dialog.Trigger>
      <Dialog.Portal>
        <Dialog.Overlay className="fixed inset-0 bg-black/50" />
        <Dialog.Content className="fixed top-1/2 left-1/2 -translate-x-1/2 -translate-y-1/2 bg-white p-6 rounded-xl">
          <Dialog.Title>모달 제목</Dialog.Title>
          <Dialog.Description>모달 설명</Dialog.Description>
          {/* 포커스 트랩, Escape 닫기, aria 모두 자동 처리 */}
          <Dialog.Close asChild>
            <button aria-label="닫기">✕</button>
          </Dialog.Close>
        </Dialog.Content>
      </Dialog.Portal>
    </Dialog.Root>
  );
}
```

### 페이지 이동 시 포커스 관리 (SPA)

```tsx
// Next.js App Router — 페이지 이동 후 포커스를 main으로 이동
'use client';
import { useEffect, useRef } from 'react';
import { usePathname } from 'next/navigation';

export function SkipLink() {
  const pathname = usePathname();
  const mainRef = useRef<HTMLElement>(null);

  useEffect(() => {
    // 라우트 변경 시 main 콘텐츠로 포커스
    mainRef.current?.focus();
  }, [pathname]);

  return (
    <>
      {/* 스킵 내비게이션 링크 */}
      <a
        href="#main"
        className="sr-only focus:not-sr-only focus:fixed focus:top-4 focus:left-4 focus:z-50 focus:px-4 focus:py-2 focus:bg-blue-600 focus:text-white focus:rounded"
      >
        본문으로 바로가기
      </a>
      <main id="main" tabIndex={-1} ref={mainRef}>
        {/* 페이지 콘텐츠 */}
      </main>
    </>
  );
}
```

접근성은 특정 사용자를 위한 선택이 아닌 **모든 사용자를 위한 기본값**입니다. 시맨틱 HTML을 올바르게 사용하고, 키보드 내비게이션을 지원하고, 색상 대비를 확인하는 것만으로도 접근성의 80%를 해결할 수 있습니다. `jest-axe`를 CI에 통합해 접근성 회귀를 방지하고, 정기적으로 스크린리더로 직접 테스트하는 습관을 들이세요.
