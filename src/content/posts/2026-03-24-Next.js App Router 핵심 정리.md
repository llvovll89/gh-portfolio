Next.js 13부터 도입된 **App Router**는 기존 Pages Router와 완전히 다른 패러다임을 제시합니다. React Server Component(RSC)를 기반으로 서버와 클라이언트의 경계를 명확하게 나누고, 더 나은 성능과 DX(Developer Experience)를 제공합니다.

> **목표:** App Router의 핵심 개념을 이해하고, 실전 프로젝트에서 Server Component, Server Action, 레이아웃 시스템을 자유롭게 활용할 수 있다.

## 1. App Router vs Pages Router

| 항목 | Pages Router | App Router |
|------|-------------|------------|
| 폴더 위치 | `pages/` | `app/` |
| 기본 렌더링 | CSR / SSR | **Server Component** |
| 데이터 페칭 | `getServerSideProps`, `getStaticProps` | `async/await` in 컴포넌트 |
| 레이아웃 | `_app.tsx` 단일 | 중첩 `layout.tsx` |
| 라우팅 | 파일 기반 | 폴더 기반 |
| React 버전 | 18 | 18+ (RSC 필요) |

App Router는 **기본이 서버**입니다. `'use client'` 없이 작성한 모든 컴포넌트는 서버에서 실행되며, 번들 사이즈에 포함되지 않습니다.

## 2. 폴더 구조와 파일 컨벤션

```
app/
├── layout.tsx          # 루트 레이아웃 (필수)
├── page.tsx            # / 라우트
├── loading.tsx         # 로딩 UI (자동 Suspense)
├── error.tsx           # 에러 바운더리
├── not-found.tsx       # 404 페이지
├── globals.css
├── blog/
│   ├── layout.tsx      # /blog 전용 레이아웃
│   ├── page.tsx        # /blog
│   └── [slug]/
│       ├── page.tsx    # /blog/:slug
│       └── loading.tsx
└── api/
    └── hello/
        └── route.ts    # API 라우트 핸들러
```

### 특수 파일 요약

| 파일 | 역할 |
|------|------|
| `page.tsx` | 라우트의 실제 UI |
| `layout.tsx` | 공유 레이아웃 (리렌더 없음) |
| `loading.tsx` | Suspense 폴백 UI |
| `error.tsx` | 에러 바운더리 (Client Component) |
| `not-found.tsx` | 404 처리 |
| `route.ts` | API 엔드포인트 (GET, POST 등) |
| `template.tsx` | 레이아웃과 달리 매 이동마다 새 인스턴스 |

## 3. Server Component vs Client Component

### Server Component (기본값)

서버에서만 실행됩니다. DB 쿼리, 파일 시스템 접근, API 키 사용이 안전합니다.

```tsx
// app/blog/page.tsx — Server Component (별도 선언 불필요)
import { db } from '@/lib/db'

export default async function BlogPage() {
  const posts = await db.post.findMany() // 서버에서 직접 DB 접근

  return (
    <ul>
      {posts.map(post => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}
```

**Server Component에서 불가능한 것:**
- `useState`, `useEffect` 등 React 훅
- 브라우저 API (`window`, `document`)
- 이벤트 핸들러 (`onClick`, `onChange`)

### Client Component

```tsx
'use client' // 파일 최상단에 선언

import { useState } from 'react'

export default function Counter() {
  const [count, setCount] = useState(0)

  return (
    <button onClick={() => setCount(c => c + 1)}>
      클릭 수: {count}
    </button>
  )
}
```

### 컴포넌트 합성 패턴

Server Component가 Client Component를 **children으로 감쌀 수 있습니다:**

```tsx
// app/page.tsx (Server)
import ClientSidebar from './ClientSidebar' // 'use client'

export default async function Page() {
  const data = await fetchData()

  return (
    <div>
      <ClientSidebar>
        <ServerContent data={data} /> {/* Server → Client children 전달 가능 */}
      </ClientSidebar>
    </div>
  )
}
```

> **핵심 규칙:** Client Component 안에서 Server Component를 **import**하면 안 됩니다. children으로 넘기는 건 OK.

## 4. 레이아웃과 중첩 라우팅

```tsx
// app/layout.tsx — 루트 레이아웃
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="ko">
      <body>
        <Header />
        <main>{children}</main>
        <Footer />
      </body>
    </html>
  )
}
```

```tsx
// app/dashboard/layout.tsx — 중첩 레이아웃
export default function DashboardLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <div className="flex">
      <Sidebar />
      <section className="flex-1">{children}</section>
    </div>
  )
}
```

중첩 레이아웃은 **라우트 이동 시 리렌더되지 않습니다.** 상태를 유지하면서 필요한 부분만 교체합니다.

### 병렬 라우트 (Parallel Routes)

`@folder` 컨벤션으로 같은 레이아웃 안에 여러 페이지를 동시에 렌더링합니다:

```
app/
├── layout.tsx
├── @modal/
│   └── login/
│       └── page.tsx   # /login 접근 시 모달로 표시
└── page.tsx
```

```tsx
// app/layout.tsx
export default function Layout({
  children,
  modal,
}: {
  children: React.ReactNode
  modal: React.ReactNode
}) {
  return (
    <>
      {children}
      {modal} {/* 오버레이 모달 */}
    </>
  )
}
```

## 5. 데이터 페칭 패턴

App Router에서는 `async/await`를 컴포넌트 안에서 직접 씁니다.

### 기본 데이터 페칭

```tsx
async function getPost(slug: string) {
  const res = await fetch(`https://api.example.com/posts/${slug}`, {
    next: { revalidate: 3600 }, // ISR: 1시간마다 재검증
  })
  if (!res.ok) throw new Error('Failed to fetch')
  return res.json()
}

export default async function PostPage({
  params,
}: {
  params: { slug: string }
}) {
  const post = await getPost(params.slug)
  return <article>{post.content}</article>
}
```

### 캐싱 옵션

```tsx
// 정적 (빌드 시 한 번만)
fetch(url, { cache: 'force-cache' })

// 항상 최신 (SSR)
fetch(url, { cache: 'no-store' })

// ISR (N초마다 재검증)
fetch(url, { next: { revalidate: 60 } })

// 태그 기반 재검증
fetch(url, { next: { tags: ['posts'] } })
```

### 병렬 데이터 페칭

```tsx
export default async function Page() {
  // 순차 ❌ — 느림
  const user = await getUser()
  const posts = await getPosts()

  // 병렬 ✅ — 빠름
  const [user, posts] = await Promise.all([getUser(), getPosts()])

  return <Dashboard user={user} posts={posts} />
}
```

## 6. Server Action

폼 제출이나 데이터 변경을 **서버 함수**로 처리합니다. API 라우트 없이 서버 로직을 바로 호출할 수 있습니다.

```tsx
// app/actions.ts
'use server'

import { revalidatePath } from 'next/cache'
import { db } from '@/lib/db'

export async function createPost(formData: FormData) {
  const title = formData.get('title') as string
  const content = formData.get('content') as string

  await db.post.create({ data: { title, content } })
  revalidatePath('/blog') // 해당 경로 캐시 무효화
}
```

```tsx
// app/blog/new/page.tsx
import { createPost } from '@/app/actions'

export default function NewPostPage() {
  return (
    <form action={createPost}>
      <input name="title" placeholder="제목" />
      <textarea name="content" placeholder="내용" />
      <button type="submit">작성</button>
    </form>
  )
}
```

### useFormState / useFormStatus

```tsx
'use client'

import { useFormStatus } from 'react-dom'
import { createPost } from '@/app/actions'

function SubmitButton() {
  const { pending } = useFormStatus()
  return (
    <button type="submit" disabled={pending}>
      {pending ? '저장 중...' : '작성'}
    </button>
  )
}
```

## 7. 로딩 UI와 Suspense

`loading.tsx` 파일을 만들면 라우트 전환 시 자동으로 Suspense 폴백으로 사용됩니다.

```tsx
// app/blog/loading.tsx
export default function Loading() {
  return (
    <div className="animate-pulse">
      <div className="h-8 bg-gray-200 rounded mb-4" />
      <div className="h-4 bg-gray-200 rounded w-3/4" />
    </div>
  )
}
```

세밀한 제어가 필요할 땐 직접 `Suspense`를 씁니다:

```tsx
import { Suspense } from 'react'
import PostList from './PostList'
import CommentList from './CommentList'

export default function Page() {
  return (
    <div>
      <Suspense fallback={<PostSkeleton />}>
        <PostList /> {/* 독립적으로 스트리밍 */}
      </Suspense>
      <Suspense fallback={<CommentSkeleton />}>
        <CommentList /> {/* 독립적으로 스트리밍 */}
      </Suspense>
    </div>
  )
}
```

## 8. 에러 핸들링

```tsx
// app/blog/error.tsx
'use client' // 에러 바운더리는 반드시 Client Component

export default function Error({
  error,
  reset,
}: {
  error: Error & { digest?: string }
  reset: () => void
}) {
  return (
    <div>
      <h2>문제가 발생했습니다.</h2>
      <p>{error.message}</p>
      <button onClick={reset}>다시 시도</button>
    </div>
  )
}
```

`notFound()` 함수로 404 처리도 가능합니다:

```tsx
import { notFound } from 'next/navigation'

export default async function PostPage({ params }: { params: { id: string } }) {
  const post = await getPost(params.id)

  if (!post) notFound() // not-found.tsx 렌더링

  return <article>{post.content}</article>
}
```

## 9. 메타데이터 API

SEO를 위한 메타데이터를 정적 또는 동적으로 설정합니다.

```tsx
// 정적 메타데이터
export const metadata = {
  title: '블로그',
  description: '개발 블로그입니다.',
  openGraph: {
    title: '블로그',
    description: '개발 블로그입니다.',
    images: ['/og-image.png'],
  },
}
```

```tsx
// 동적 메타데이터
export async function generateMetadata({
  params,
}: {
  params: { slug: string }
}) {
  const post = await getPost(params.slug)

  return {
    title: post.title,
    description: post.excerpt,
    openGraph: {
      title: post.title,
      images: [post.thumbnail],
    },
  }
}
```

## 10. Pages Router → App Router 마이그레이션

### 단계별 전략

점진적 마이그레이션이 가능합니다. `pages/`와 `app/`을 **동시에** 운영할 수 있습니다.

```
1단계: app/ 폴더 생성, 루트 layout.tsx만 추가
2단계: 새로운 라우트는 app/에 추가
3단계: 기존 라우트를 하나씩 app/으로 이동
4단계: _app.tsx, _document.tsx 로직을 layout.tsx로 통합
5단계: pages/ 완전 제거
```

### 주요 변환 대응표

| Pages Router | App Router |
|-------------|------------|
| `getServerSideProps` | 컴포넌트에서 `async/await` |
| `getStaticProps` | `fetch` with `force-cache` |
| `getStaticPaths` | `generateStaticParams` |
| `useRouter().push()` | `import { useRouter } from 'next/navigation'` |
| `next/head` | `export const metadata` |
| `_app.tsx` | `app/layout.tsx` |

### 주의사항

- `next/router` → `next/navigation` (import 경로 변경)
- `useRouter`의 `query` → `useSearchParams`, `useParams`로 분리
- Client Component에서 `headers()`, `cookies()` 사용 불가 (Server Component 전용)

## 정리

| 개념 | 핵심 포인트 |
|------|------------|
| Server Component | 기본값, 서버에서만 실행, DB/API 직접 접근 |
| Client Component | `'use client'` 선언, 상호작용/훅 사용 가능 |
| 중첩 레이아웃 | 경로별 독립 레이아웃, 리렌더 없음 |
| 데이터 페칭 | `async/await` + `fetch` 캐싱 옵션 |
| Server Action | `'use server'` 선언, API 없이 서버 함수 호출 |
| Suspense | `loading.tsx` 또는 직접 `<Suspense>` 사용 |

App Router는 초기 학습 곡선이 있지만, 서버와 클라이언트를 명확하게 분리함으로써 **더 빠른 성능**과 **더 안전한 데이터 처리**를 가능하게 합니다. 새 프로젝트라면 App Router를 기본으로 시작하는 것을 추천합니다.
