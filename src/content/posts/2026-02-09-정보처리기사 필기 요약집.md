# 1과목: 소프트웨어 설계

## 1-1. 요구사항 확인

### 소프트웨어 생명주기 모델

| 모델 | 특징 |
|------|------|
| **폭포수(Waterfall)** | 순차적 진행, 각 단계 완료 후 다음 단계, 가장 오래된 모델 |
| **프로토타이핑** | 시제품(프로토타입)을 만들어 요구사항 확인 |
| **나선형(Spiral)** | 계획→위험분석→개발→평가 반복, **위험 관리** 중심 |
| **애자일(Agile)** | 반복·점진적 개발, 고객 소통 중시, 변화에 유연 |

### 애자일 방법론

- **스크럼(Scrum)**: 스프린트(2~4주) 단위 반복, 제품 백로그 → 스프린트 백로그, 데일리 스크럼(15분 회의)
  - 역할: **제품 책임자(PO)**, **스크럼 마스터(SM)**, **개발팀**
- **XP(eXtreme Programming)**: 짧은 릴리스, 고객 상주, **페어 프로그래밍**, TDD, 리팩토링, CI
  - 5가지 가치: **의사소통, 단순성, 피드백, 용기, 존중**
- **칸반(Kanban)**: 시각적 작업 흐름 관리, WIP(Work In Progress) 제한
- **린(Lean)**: 낭비 제거, 가치 흐름 최적화

### 요구사항 개발 프로세스

1. **도출(Elicitation)**: 인터뷰, 설문, 브레인스토밍, 워크숍, 프로토타이핑
2. **분석(Analysis)**: 요구사항 분류·우선순위, 모델링(DFD, UML)
3. **명세(Specification)**: 기능/비기능 요구사항 문서화 (정형/비정형 명세)
4. **확인(Validation)**: 검토, 인스펙션, 워크스루, 프로토타이핑

### 요구사항 분류

- **기능 요구사항**: 시스템이 수행해야 할 기능 (입력·처리·출력)
- **비기능 요구사항**: 성능, 보안, 품질, 제약사항 등

## 1-3. 애플리케이션 설계

### UML(Unified Modeling Language)

**구조 다이어그램 (정적)**

| 다이어그램 | 설명 |
|-----------|------|
| **클래스** | 클래스 간 관계 표현 (속성, 메서드) |
| **객체** | 인스턴스 간 관계 |
| **컴포넌트** | 물리적 컴포넌트 관계 |
| **배치(Deployment)** | HW/SW 배치 구조 |
| **패키지** | 패키지 간 의존 관계 |
| **복합 구조** | 클래스 내부 구조 |

**행위 다이어그램 (동적)**

| 다이어그램 | 설명 |
|-----------|------|
| **유스케이스** | 액터-시스템 상호작용, 기능 모델링 |
| **시퀀스** | 객체 간 메시지 흐름 (시간 순서) |
| **커뮤니케이션** | 객체 간 메시지 + 관계 |
| **상태** | 객체의 상태 변화 |
| **활동** | 워크플로우(흐름도) |
| **타이밍** | 시간 흐름에 따른 상태 변화 |

### UML 관계

- **연관(Association)**: 클래스 간 관계 (실선)
- **의존(Dependency)**: 일시적 사용 관계 (점선 화살표)
- **일반화(Generalization)**: 상속 (삼각형 화살표)
- **실체화(Realization)**: 인터페이스 구현 (점선 삼각형)
- **집합(Aggregation)**: 전체-부분 (빈 마름모) — 독립적 존재 가능
- **합성(Composition)**: 전체-부분 (채운 마름모) — 생명주기 동일

### 디자인 패턴

**생성 패턴**

| 패턴 | 설명 |
|------|------|
| **싱글톤(Singleton)** | 인스턴스 1개만 생성 보장 |
| **팩토리 메서드(Factory Method)** | 객체 생성을 서브클래스에 위임 |
| **추상 팩토리(Abstract Factory)** | 관련 객체 그룹 생성 |
| **빌더(Builder)** | 복잡한 객체를 단계별 생성 |
| **프로토타입(Prototype)** | 기존 객체를 복제해서 생성 |

**구조 패턴**

| 패턴 | 설명 |
|------|------|
| **어댑터(Adapter)** | 호환되지 않는 인터페이스 연결 |
| **브리지(Bridge)** | 추상층과 구현층 분리 |
| **데코레이터(Decorator)** | 객체에 기능 동적 추가 |
| **퍼사드(Facade)** | 복잡한 서브시스템에 단순한 인터페이스 제공 |
| **프록시(Proxy)** | 대리 객체를 통한 접근 제어 |
| **컴포지트(Composite)** | 트리 구조로 전체-부분 표현 |
| **플라이웨이트(Flyweight)** | 공유로 메모리 절약 |

**행위 패턴**

| 패턴 | 설명 |
|------|------|
| **옵서버(Observer)** | 상태 변화 시 관련 객체들에게 통지 |
| **전략(Strategy)** | 알고리즘을 캡슐화해 교체 가능 |
| **템플릿 메서드(Template Method)** | 알고리즘 뼈대 정의, 세부 단계 서브클래스 위임 |
| **커맨드(Command)** | 요청을 객체로 캡슐화 |
| **상태(State)** | 상태에 따라 행위 변경 |
| **중재자(Mediator)** | 객체 간 상호작용을 캡슐화 |
| **반복자(Iterator)** | 컬렉션 요소 순차 접근 |

# 2과목: 소프트웨어 개발

## 2-1. 데이터 입·출력 구현

### 자료구조

**선형 구조**

| 구조 | 특징 |
|------|------|
| **배열(Array)** | 인덱스 접근, 고정 크기 |
| **연결 리스트(Linked List)** | 노드+포인터, 동적 크기 |
| **스택(Stack)** | LIFO(Last In First Out) |
| **큐(Queue)** | FIFO(First In First Out) |
| **데크(Deque)** | 양쪽에서 삽입·삭제 가능 |

**비선형 구조**

| 구조 | 특징 |
|------|------|
| **트리(Tree)** | 계층 구조, 루트-노드-리프 |
| **그래프(Graph)** | 정점(Vertex)+간선(Edge) |

### 트리 순회

- **전위(Preorder)**: 루트 → 왼쪽 → 오른쪽
- **중위(Inorder)**: 왼쪽 → 루트 → 오른쪽
- **후위(Postorder)**: 왼쪽 → 오른쪽 → 루트

### 정렬 알고리즘

| 알고리즘 | 평균 시간복잡도 | 특징 |
|----------|:-------------:|------|
| **버블 정렬** | O(n²) | 인접 요소 비교·교환 |
| **선택 정렬** | O(n²) | 최솟값 찾아 교환 |
| **삽입 정렬** | O(n²) | 적절한 위치에 삽입 |
| **셸 정렬** | O(n^1.5) | 간격(gap) 기반 삽입 정렬 개선 |
| **퀵 정렬** | O(n log n) | 피벗 기준 분할, 최악 O(n²) |
| **병합 정렬** | O(n log n) | 분할 후 병합, 안정 정렬 |
| **힙 정렬** | O(n log n) | 힙 자료구조 이용 |
| **기수 정렬** | O(d·n) | 자릿수별 정렬, 비교하지 않음 |

### 검색 알고리즘

- **순차 검색**: O(n), 정렬 불필요
- **이진 검색**: O(log n), **정렬 필수**
- **해싱**: O(1) 평균, 해시 함수로 직접 접근

## 2-4. 애플리케이션 테스트 관리

### 테스트 레벨

1. **단위 테스트(Unit)**: 모듈/컴포넌트 단위
2. **통합 테스트(Integration)**: 모듈 간 인터페이스
3. **시스템 테스트(System)**: 전체 시스템 기능·비기능
4. **인수 테스트(Acceptance)**: 사용자 관점 확인
   - 알파 테스트: 개발자 환경에서 사용자 테스트
   - 베타 테스트: 실제 환경에서 사용자 테스트

### 테스트 기법

**화이트박스 테스트** (구조 기반)

- 문장 커버리지 / 분기(결정) 커버리지 / 조건 커버리지 / 조건-결정 커버리지 / 변경 조건-결정 커버리지 / 다중 조건 커버리지
- 기본 경로 테스트, 루프 테스트

**블랙박스 테스트** (명세 기반)

| 기법 | 설명 |
|------|------|
| **동치 분할** | 유효/무효 값 영역으로 나눠 대표값 테스트 |
| **경계값 분석** | 경계 부근 값으로 테스트 |
| **상태 전이** | 상태 변화 기반 테스트 |
| **결정 테이블** | 조건 조합표 기반 테스트 |
| **원인-결과 그래프** | 입력 조건과 결과 간 관계 |
| **오류 추정** | 경험 기반 오류 예측 |

### 통합 테스트 전략

- **상향식(Bottom-Up)**: 하위 모듈부터 (**드라이버** 사용)
- **하향식(Top-Down)**: 상위 모듈부터 (**스텁** 사용)
- **빅뱅**: 모든 모듈 동시에 통합
- **샌드위치**: 상향식 + 하향식 결합

### 테스트 용어

- **검증(Verification)**: "제품을 올바르게 만들고 있는가?" (개발 과정)
- **확인(Validation)**: "올바른 제품을 만들었는가?" (결과물)

# 3과목: 데이터베이스 구축

## 3-1. 논리 데이터베이스 설계

### 데이터 모델링 절차

**개념적 → 논리적 → 물리적 설계**

### 관계 데이터 모델

**키(Key) 종류**

| 키 | 설명 |
|----|------|
| **슈퍼키** | 유일성을 만족하는 속성의 집합 |
| **후보키** | 유일성 + 최소성 |
| **기본키(PK)** | 후보키 중 대표로 선택된 키 (NOT NULL, 유일) |
| **대체키** | 후보키 중 기본키가 아닌 키 |
| **외래키(FK)** | 다른 릴레이션의 기본키를 참조 |

### 관계 대수 연산자

**일반 집합 연산자**

| 연산 | 기호 | 설명 |
|------|:----:|------|
| 합집합 | ∪ | 두 릴레이션의 합 |
| 교집합 | ∩ | 두 릴레이션의 공통 |
| 차집합 | - | R에 있고 S에 없는 것 |
| 카티션 곱 | × | 모든 조합 |

**순수 관계 연산자**

| 연산 | 기호 | 설명 |
|------|:----:|------|
| 셀렉트(Select) | σ | 행(튜플) 선택 (조건) |
| 프로젝트(Project) | π | 열(속성) 선택 |
| 조인(Join) | ⋈ | 공통 속성으로 결합 |
| 디비전(Division) | ÷ | S의 모든 값을 포함하는 R의 튜플 |

### 정규화(Normalization)

| 정규형 | 조건 |
|--------|------|
| **1NF** | 모든 속성이 **원자값(Atomic)** |
| **2NF** | 1NF + **부분 함수 종속 제거** |
| **3NF** | 2NF + **이행 함수 종속 제거** |
| **BCNF** | 모든 결정자가 **후보키** |
| **4NF** | BCNF + **다치 종속 제거** |
| **5NF** | 4NF + **조인 종속 제거** |

> 암기: **도부이결다조** (도메인→부분함수종속→이행함수종속→결정자→다치종속→조인종속)

### 이상 현상(Anomaly)

- **삽입 이상**: 불필요한 데이터가 함께 삽입
- **삭제 이상**: 유용한 데이터가 함께 삭제
- **갱신 이상**: 일부만 수정되어 불일치 발생

### 반정규화(Denormalization)

- 성능 향상을 위해 의도적으로 정규화 위반
- 테이블 병합, 테이블 분할(수평/수직), 중복 속성 추가

## 3-2. 물리 데이터베이스 설계

### 인덱스(Index)

- **B-Tree**: 균형 트리, 범위 검색에 유리
- **B+-Tree**: 리프 노드에만 데이터, 순차 탐색에 유리
- **해시 인덱스**: 등가 검색에 유리, 범위 검색 불리
- **비트맵 인덱스**: 카디널리티가 낮은 컬럼에 유리

### 클러스터

- **클러스터드 인덱스**: 데이터 물리적 정렬 = 인덱스 순서 (테이블당 1개)
- **넌클러스터드 인덱스**: 인덱스와 데이터 별도 저장

### 뷰(View)

- 가상 테이블, 논리적 독립성, 보안 용이
- CREATE VIEW / DROP VIEW (ALTER 불가, 재생성)

### 파티셔닝

- **범위(Range)**: 값의 범위로 분할
- **해시(Hash)**: 해시 값으로 분할
- **리스트(List)**: 특정 값 목록으로 분할
- **라운드로빈**: 순차적 분배


## 3-4. 트랜잭션 관리

### 트랜잭션 특성 (ACID)

| 특성 | 설명 |
|------|------|
| **원자성(Atomicity)** | 전부 실행 or 전부 취소 |
| **일관성(Consistency)** | 실행 전후 데이터 일관성 유지 |
| **격리성(Isolation)** | 동시 트랜잭션 간 간섭 없음 |
| **지속성(Durability)** | 완료된 결과는 영구 반영 |

### 병행 제어(Concurrency Control)

**문제점**

- **갱신 분실(Lost Update)**: 같은 데이터 동시 수정 시 하나가 사라짐
- **비완료 의존(Dirty Read)**: 커밋되지 않은 데이터 읽기
- **모순(Inconsistency)**: 일부만 갱신된 결과 읽기
- **연쇄 복귀(Cascading Rollback)**: 한 트랜잭션 취소가 다른 것에 영향

**기법**

- **로킹(Locking)**: 공유 락(S) / 배타 락(X), 2단계 로킹(2PL)
- **타임스탬프**: 트랜잭션 시작 시간 기준 순서 결정
- **낙관적 검증**: 실행 후 검증
- **MVCC**: 다중 버전 동시성 제어

### 회복(Recovery) 기법

- **로그 기반**: 지연(Deferred) / 즉시(Immediate) 갱신
- **체크포인트**: 일정 시점의 상태 저장
- **그림자 페이징**: 복사본으로 복구

# 4과목: 프로그래밍 언어 활용

## 4-1. 서버 프로그램 구현

### 개발 프레임워크 특성

- **모듈화(Modularity)**: 기능별 독립 모듈
- **재사용성(Reusability)**: 컴포넌트·라이브러리 재사용
- **확장성(Extensibility)**: 기능 추가·변경 용이
- **제어의 역흥(IoC)**: 프레임워크가 흐름 제어

### 모듈 결합도 (낮을수록 좋음)

> **데스공외제내** (데이터→스탬프→제어→외부→공통→내용)

| 결합도 | 설명 |
|--------|------|
| **데이터 결합도** | 파라미터로 데이터만 전달 (최선) |
| **스탬프 결합도** | 자료구조(레코드) 전달 |
| **제어 결합도** | 제어 요소(플래그) 전달 |
| **외부 결합도** | 외부에서 선언한 변수 공유 |
| **공통 결합도** | 전역 변수 공유 |
| **내용 결합도** | 직접 내부 접근 (최악) |

### 모듈 응집도 (높을수록 좋음)

> **우논시절통순기** (우연→논리→시간→절차→통신→순차→기능)

| 응집도 | 설명 |
|--------|------|
| **기능적 응집도** | 단일 기능 수행 (최선) |
| **순차적 응집도** | 한 활동의 출력이 다음의 입력 |
| **통신적 응집도** | 같은 데이터를 사용하는 활동 |
| **절차적 응집도** | 순서대로 수행해야 하는 활동 |
| **시간적 응집도** | 같은 시점에 실행하는 활동 |
| **논리적 응집도** | 유사한 성격의 처리를 모아 놓음 |
| **우연적 응집도** | 관련 없는 기능이 한 모듈에 (최악) |

## 4-2. 프로그래밍 언어 활용

### C언어 핵심

```c
// 포인터

struct Student {

    int age;
};
```

**주요 출력 함수/서식**

- `%d`: 정수, `%f`: 실수, `%c`: 문자, `%s`: 문자열
- `%o`: 8진수, `%x`: 16진수

### Java 핵심

```java
// 클래스와 상속
class Animal {

class Dog extends Animal {

    void sound() { System.out.println("멍멍"); }

// 추상 클래스
abstract class Shape {

}

// 인터페이스

}
```

- **오버로딩(Overloading)**: 같은 이름, 다른 매개변수 (컴파일 시)
- **오버라이딩(Overriding)**: 상위 메서드 재정의 (실행 시, 다형성)
- **접근 제어자**: `public` > `protected` > `default` > `private`

### Python 핵심

```python
# 리스트 컴프리헨션

```

7. 논리 `&&` → `||`
8. 삼항 `?:`
9. 대입 `=`, `+=` 등

## 4-3. 운영체제 기초

### 프로세스 스케줄링

**선점형(Preemptive)**

| 알고리즘 | 설명 |
|----------|------|
| **RR(Round Robin)** | 시간 할당량(Time Quantum)만큼 실행 후 교체 |
| **SRT(Shortest Remaining Time)** | 남은 시간이 짧은 프로세스 우선 |
| **MLQ(Multi-Level Queue)** | 여러 큐에 우선순위별 분류 |
| **MLFQ(Multi-Level Feedback Queue)** | MLQ + 큐 간 이동 가능 |

**비선점형(Non-Preemptive)**

| 알고리즘 | 설명 |
|----------|------|
| **FCFS(First Come First Served)** | 먼저 온 순서대로 |
| **SJF(Shortest Job First)** | 실행 시간 짧은 것 우선 |
| **HRN(Highest Response Ratio Next)** | (대기시간+서비스시간)/서비스시간 높은 것 우선 |
| **우선순위(Priority)** | 우선순위 기반 |

### 교착상태(Deadlock)

**발생 조건 (4가지 모두 충족 시)**

1. **상호 배제(Mutual Exclusion)**: 자원 독점
2. **점유와 대기(Hold & Wait)**: 점유 상태에서 추가 자원 대기
3. **비선점(Non-Preemption)**: 강제 회수 불가
4. **환형 대기(Circular Wait)**: 순환 형태 대기

**해결 방법**

- **예방(Prevention)**: 4가지 조건 중 하나 제거
- **회피(Avoidance)**: **은행원 알고리즘** (안전 상태 유지)
- **탐지(Detection)**: 교착 상태 발생 후 탐지
- **복구(Recovery)**: 프로세스 종료 또는 자원 선점

### 메모리 관리

**페이징(Paging)**

- 물리 메모리를 고정 크기(프레임)로 분할
- 논리 메모리를 같은 크기의 페이지로 분할
- **내부 단편화** 발생 가능

**세그먼테이션(Segmentation)**

- 가변 크기의 세그먼트(논리적 단위)로 분할
- **외부 단편화** 발생 가능

**페이지 교체 알고리즘**

- **FIFO**: 먼저 들어온 페이지 교체
- **LRU(Least Recently Used)**: 가장 오래 사용하지 않은 페이지
- **LFU(Least Frequently Used)**: 사용 빈도 가장 낮은 페이지
- **OPT(Optimal)**: 가장 오래 사용되지 않을 페이지 (이론적 최적)

## 4-4. 네트워크 기초

### OSI 7계층

| 계층 | 이름 | 프로토콜/장비 | 단위 |
|:----:|------|-------------|------|
| 7 | **응용(Application)** | HTTP, FTP, SMTP, DNS, SNMP | 데이터 |
| 6 | **표현(Presentation)** | JPEG, MPEG, SSL/TLS | 데이터 |

| 4 | **전송(Transport)** | TCP, UDP | 세그먼트 |
| 3 | **네트워크(Network)** | IP, ICMP, ARP, RARP, OSPF | 패킷 |
| 2 | **데이터링크(Data Link)** | Ethernet, PPP, HDLC | 프레임 |
| 1 | **물리(Physical)** | RS-232, 허브, 리피터 | 비트 |

> 암기: **아파서티나다피** (응용→표현→세션→전송→네트워크→데이터링크→물리)

### TCP vs UDP

| 구분 | TCP | UDP |
|------|-----|-----|
| 연결 | 연결 지향 (3-Way Handshake) | 비연결 |
| 신뢰성 | 높음 (재전송, 순서 보장) | 낮음 |
| 속도 | 느림 | 빠름 |
| 용도 | 웹, 이메일, 파일 전송 | 스트리밍, DNS, VoIP |

### IP 주소 체계

**IPv4 클래스**

| 클래스 | 범위 | 서브넷 마스크 |
|--------|------|-------------|
| A | 0~127 | 255.0.0.0 |
| B | 128~191 | 255.255.0.0 |
| C | 192~223 | 255.255.255.0 |

**IPv6**

- 128비트, 16비트씩 8그룹 (콜론 구분)
- IPv4 → IPv6 전환: **듀얼 스택, 터널링, 주소 변환(NAT-PT)**

### 라우팅 프로토콜

- **RIP**: 거리 벡터, 홉 수(최대 15), 소규모 네트워크
- **OSPF**: 링크 상태, 대규모 네트워크, 다익스트라 알고리즘
- **BGP**: 경로 벡터, AS 간 라우팅 (인터넷)

## 5-2. IT 프로젝트 정보시스템 구축 관리

### 비용 산정 기법

**하향식**

- **전문가 감정**: 경험 기반
- **델파이 기법**: 전문가 합의

**상향식**

- **LOC(Lines of Code)**: 코드 라인 수 기반
- **Man Month**: LOC / 1인당 월 생산 코드 수
- **COCOMO**: 소프트웨어 규모(LOC) 기반
  - **조직형(Organic)**: 소규모, 5만 라인 이하
  - **반분리형(Semi-Detached)**: 중규모, 30만 라인 이하
  - **내장형(Embedded)**: 대규모, 30만 라인 이상
- **기능 점수(FP)**: 기능(입력·출력·질의·파일·인터페이스) 기반

### 일정 관리

- **CPM(Critical Path Method)**: 임계 경로, 최장 경로가 최소 소요 시간
- **PERT**: 낙관치·기대치·비관치로 예상 시간 산정
  - 기대시간 = (낙관치 + 4×기대치 + 비관치) / 6
- **간트 차트**: 작업·일정 막대그래프

### 위험 관리

- **위험 식별 → 위험 분석 → 위험 대응 → 위험 모니터링**
- 대응 전략: 회피, 전가, 완화, 수용

---

## 5-3. 소프트웨어 개발 보안 구축

### 보안 3대 요소 (CIA)

| 요소 | 설명 |
|------|------|
| **기밀성(Confidentiality)** | 인가된 사용자만 접근 |
| **무결성(Integrity)** | 데이터의 정확성·완전성 보장 |
| **가용성(Availability)** | 필요할 때 접근 가능 |

### 암호화 기법

**대칭키(비밀키)**

- 암호화 키 = 복호화 키
- **블록 암호**: DES(56비트), 3DES, **AES(128/192/256비트)**, SEED, ARIA
- **스트림 암호**: RC4
- 빠르지만 키 분배 문제

**비대칭키(공개키)**

- 공개키로 암호화, 개인키로 복호화 (또는 반대)
- **RSA**: 소인수분해 기반 (가장 대표적)
- **ECC**: 타원곡선 기반
- **DSA**: 디지털 서명
- 느리지만 키 분배 용이

**해시 함수**

- 단방향, 고정 길이 출력
- **MD5**(128비트), **SHA-1**(160비트), **SHA-256**(256비트)
- 무결성 검증, 비밀번호 저장

### 주요 보안 공격

| 공격 | 설명 |
|------|------|
| **SQL Injection** | SQL 구문 삽입으로 DB 조작 |
| **XSS(Cross-Site Scripting)** | 악성 스크립트 삽입 |
| **CSRF** | 사용자 권한으로 의도치 않은 요청 |
| **DDoS** | 대량 트래픽으로 서비스 마비 |
| **세션 하이재킹** | 세션 탈취 |
| **피싱(Phishing)** | 위장 사이트로 정보 탈취 |
| **스니핑(Sniffing)** | 네트워크 패킷 도청 |
| **스푸핑(Spoofing)** | IP/MAC 주소 위장 |
| **랜섬웨어** | 파일 암호화 후 금전 요구 |
| **APT** | 지능적·지속적 위협(타겟 공격) |
| **제로데이 공격** | 취약점 패치 전 공격 |
| **사회공학** | 사람의 심리를 이용한 공격 |

### 보안 솔루션

| 솔루션 | 설명 |
|--------|------|
| **방화벽(Firewall)** | 네트워크 접근 제어 |
| **IDS** | 침입 탐지 시스템 |
| **IPS** | 침입 방지 시스템 |
| **VPN** | 가상 사설 네트워크 |
| **WAF** | 웹 애플리케이션 방화벽 |
| **DLP** | 데이터 유출 방지 |
| **NAC** | 네트워크 접근 제어 |
| **ESM** | 통합 보안 관리 |

### 접근 통제 모델

- **DAC(임의적)**: 소유자가 권한 부여 (주체 기반)
- **MAC(강제적)**: 보안 등급 기반, 관리자 통제
- **RBAC(역할 기반)**: 역할에 권한 부여

## 5-4. 시스템 보안 구축

### 인증 방식

- **지식 기반**: 비밀번호, PIN
- **소유 기반**: OTP, 보안토큰, 스마트카드
- **생체 기반**: 지문, 홍채, 얼굴
- **행위 기반**: 서명, 타이핑 패턴
- **다중 인증(MFA)**: 2가지 이상 조합

### 서버 접근 통제

- **SSH**: 암호화된 원격 접속 (포트 22)
- **SSL/TLS**: 전송 계층 보안
- **IPSec**: 네트워크 계층 보안 (IP 패킷 암호화)

### 로그 관리

- **syslog**: 시스템 로그 표준
- **보안 로그**: 접근 기록, 감사 추적
- **로그 분석**: 이상 패턴 탐지, 포렌식

## 5-5. 소프트웨어 품질 관리

### ISO/IEC 25010 품질 특성

| 특성 | 설명 |
|------|------|
| **기능 적합성** | 요구 기능 충족 |
| **성능 효율성** | 자원 대비 성능 |
| **호환성** | 다른 시스템과 공존·상호 운용 |
| **사용성** | 사용 편의성 |
| **신뢰성** | 장애 없이 기능 유지 |
| **보안성** | 정보 보호 |
| **유지보수성** | 수정·개선 용이 |
| **이식성** | 다른 환경으로 이전 용이 |

### 테스트 관련 용어

- **리팩토링**: 기능 변경 없이 코드 구조 개선
- **클린 코드**: 가독성 높고 단순한 코드
- **기술 부채**: 빠른 개발을 위해 쌓인 품질 저하

### CMM/CMMI

- **Level 1 (Initial)**: 프로세스 비체계적
- **Level 2 (Managed)**: 프로젝트 단위 관리
- **Level 3 (Defined)**: 조직 표준 프로세스 정의
- **Level 4 (Quantitatively Managed)**: 정량적 관리
- **Level 5 (Optimizing)**: 지속적 개선
