## 1. Actions — 비동기 폼 처리의 새 표준

React 19 이전에는 폼 제출 시 `onSubmit` 핸들러에서 직접 상태를 관리해야 했습니다.

**Before (React 18)**

```tsx
function SignupForm() {
  const [isPending, setIsPending] = useState(false);
  const [error, setError] = useState(null);

  async function handleSubmit(e: React.FormEvent) {
    e.preventDefault();
    setIsPending(true);
    try {
      await submitForm(new FormData(e.currentTarget));
    } catch (err) {
      setError(err.message);
    } finally {
      setIsPending(false);
    }
  }

  return <form onSubmit={handleSubmit}>...</form>;
}
```

**After (React 19 Actions)**

```tsx
function SignupForm() {
  async function submitAction(formData: FormData) {
    // 비동기 함수를 바로 action prop에 전달
    await submitForm(formData);
  }

  return <form action={submitAction}>...</form>;
}
```

React가 자동으로 pending 상태를 추적합니다. `<form>`의 `action` prop에 비동기 함수를 직접 넘길 수 있습니다.

## 2. useActionState — 액션 상태 관리

`useActionState`는 액션의 상태(에러, 결과값, 로딩)를 한 번에 관리합니다.

```tsx
import { useActionState } from 'react';

async function createPost(prevState: any, formData: FormData) {
  const title = formData.get('title') as string;
  if (!title) return { error: '제목을 입력해주세요.' };

  await api.createPost({ title });
  return { success: true };
}

function CreatePostForm() {
  const [state, formAction, isPending] = useActionState(createPost, null);

  return (
    <form action={formAction}>
      <input name="title" placeholder="포스트 제목" />
      {state?.error && <p style={{ color: 'red' }}>{state.error}</p>}
      {state?.success && <p>포스트가 생성되었습니다!</p>}
      <button type="submit" disabled={isPending}>
        {isPending ? '저장 중...' : '저장'}
      </button>
    </form>
  );
}
```

| 반환값 | 설명 |
|--------|------|
| `state` | 액션 함수의 반환값 (초기값은 두 번째 인자) |
| `formAction` | form의 `action` prop에 전달할 래핑된 액션 |
| `isPending` | 액션 실행 중 여부 |

## 3. useFormStatus — 폼 제출 상태 공유

하위 컴포넌트에서 부모 폼의 제출 상태를 읽을 수 있습니다. Context 없이도 가능합니다.

```tsx
import { useFormStatus } from 'react-dom';

function SubmitButton() {
  const { pending } = useFormStatus();

  return (
    <button type="submit" disabled={pending}>
      {pending ? '처리 중...' : '제출'}
    </button>
  );
}

function LoginForm() {
  return (
    <form action={loginAction}>
      <input name="email" type="email" />
      <input name="password" type="password" />
      {/* SubmitButton이 부모 form의 pending 상태를 자동으로 읽음 */}
      <SubmitButton />
    </form>
  );
}
```

> **주의**: `useFormStatus`는 `<form>` 내부의 컴포넌트에서만 동작합니다. 같은 컴포넌트의 form은 읽지 못하고, 반드시 **자식 컴포넌트**에서 사용해야 합니다.

## 4. useOptimistic — 낙관적 UI 업데이트

서버 응답 전에 UI를 미리 업데이트하고, 실패 시 자동 롤백합니다.

```tsx
import { useOptimistic } from 'react';

type Message = { id: number; text: string; sending?: boolean };

function MessageList({ messages }: { messages: Message[] }) {
  const [optimisticMessages, addOptimisticMessage] = useOptimistic(
    messages,
    (state, newText: string) => [
      ...state,
      { id: Date.now(), text: newText, sending: true },
    ]
  );

  async function sendAction(formData: FormData) {
    const text = formData.get('text') as string;
    addOptimisticMessage(text); // 즉시 UI 반영
    await api.sendMessage(text); // 실제 요청
  }

  return (
    <>
      <ul>
        {optimisticMessages.map((msg) => (
          <li key={msg.id} style={{ opacity: msg.sending ? 0.5 : 1 }}>
            {msg.text} {msg.sending && '(전송 중...)'}
          </li>
        ))}
      </ul>
      <form action={sendAction}>
        <input name="text" />
        <button type="submit">전송</button>
      </form>
    </>
  );
}
```

- 액션이 성공하면 낙관적 상태가 실제 상태로 교체됩니다.
- 액션이 실패하면 자동으로 원래 상태로 롤백됩니다.

## 5. use() — Promise와 Context 읽기

`use()`는 렌더링 중에 Promise나 Context 값을 직접 읽을 수 있는 새 API입니다.

**Promise 읽기 (Suspense와 함께)**

```tsx
import { use, Suspense } from 'react';

function UserProfile({ userPromise }: { userPromise: Promise<User> }) {
  const user = use(userPromise); // Suspense가 resolve될 때까지 중단

  return <div>{user.name}</div>;
}

function App() {
  const userPromise = fetchUser(1);

  return (
    <Suspense fallback={<div>로딩 중...</div>}>
      <UserProfile userPromise={userPromise} />
    </Suspense>
  );
}
```

**Context 읽기**

```tsx
import { use } from 'react';

// useContext 대신 use()로도 Context 읽기 가능
function ThemeButton() {
  const theme = use(ThemeContext); // 조건문 안에서도 사용 가능!
  return <button className={theme}>클릭</button>;
}
```

> `use()`는 `useContext`와 달리 **조건문, 반복문 안에서 사용 가능**합니다. (단, 훅 규칙의 예외)

## 6. ref 콜백의 cleanup 지원

ref 콜백에서 cleanup 함수를 반환할 수 있게 되었습니다.

```tsx
function VideoPlayer() {
  return (
    <video
      ref={(node) => {
        if (!node) return;
        const player = initPlayer(node);

        // cleanup 함수 반환 — 언마운트 시 자동 호출
        return () => {
          player.destroy();
        };
      }}
    />
  );
}
```

이전에는 `useEffect`를 별도로 사용해야 했던 패턴을 ref 콜백 하나로 처리할 수 있습니다.

## 7. Document Metadata 지원

컴포넌트 내에서 `<title>`, `<meta>`, `<link>` 태그를 직접 렌더링하면 React가 자동으로 `<head>`에 호이스팅합니다.

```tsx
function BlogPost({ post }: { post: Post }) {
  return (
    <article>
      {/* React가 자동으로 <head>로 이동 */}
      <title>{post.title}</title>
      <meta name="description" content={post.summary} />
      <link rel="canonical" href={`/posts/${post.slug}`} />

      <h1>{post.title}</h1>
      <p>{post.content}</p>
    </article>
  );
}
```

`react-helmet` 같은 서드파티 라이브러리 없이 메타데이터를 관리할 수 있습니다.

## 8. 마이그레이션 체크리스트

| 항목 | 변경 내용 |
|------|-----------|
| `ReactDOM.render()` | `createRoot()` 필수 (React 18부터 deprecated, 19에서 제거) |
| `ReactDOM.hydrate()` | `hydrateRoot()` 사용 |
| `propTypes` / `defaultProps` | 함수형 컴포넌트의 defaultProps 제거됨 → ES default params 사용 |
| `forwardRef` | ref를 props로 직접 전달 가능 (forwardRef 불필요) |
| Context `<Provider value>` | `<MyContext value>` 로 축약 가능 |
| `string ref` | 완전히 제거됨 |

**forwardRef 제거 예시**

```tsx
// Before
const Input = forwardRef<HTMLInputElement, Props>((props, ref) => (
  <input {...props} ref={ref} />
));

// After (React 19)
function Input({ ref, ...props }: Props & { ref?: React.Ref<HTMLInputElement> }) {
  return <input {...props} ref={ref} />;
}
```

React 19는 폼과 비동기 로직을 다루는 방식을 크게 단순화했습니다. 특히 `useActionState` + `useOptimistic` 조합은 서버와 통신하는 UI에서 보일러플레이트를 획기적으로 줄여줍니다. 기존 프로젝트를 마이그레이션할 때는 `ReactDOM.render` 제거와 `propTypes` / `defaultProps` 정리부터 시작하세요.
