## 1. Figma란?

Figma는 **브라우저 기반의 UI/UX 디자인 협업 툴**입니다.
Adobe XD, Sketch의 대안으로 자리 잡았고, 현재 업계 표준에 가깝습니다.

| 특징 | 내용 |
|------|------|
| 플랫폼 | 브라우저 + 데스크탑 앱 (Windows/Mac) |
| 실시간 협업 | 여러 명이 동시에 같은 파일 작업 가능 |
| 무료 플랜 | 개인 3개 파일까지 무료 (팀 협업은 유료) |
| 개발자 모드 | CSS값, 에셋, 치수 자동 추출 가능 |
| 플러그인 | 오토 레이아웃, 더미 데이터 등 풍부한 플러그인 |

---

## 2. 개발자가 Figma를 써야 하는 이유

처음엔 "디자이너가 하면 되지 왜 나까지?" 했는데, 실제로 써보니 이유가 명확합니다.

- **디자이너 없이 혼자 개발할 때** → 직접 설계하고 개발하면 훨씬 빠름
- **디자인-개발 간 커뮤니케이션 비용 절감** → "이 패딩이 몇 px이에요?" 같은 질문 불필요
- **개발자 모드(Dev Mode)** → CSS, 간격, 컬러 코드를 자동으로 뽑아줌
- **컴포넌트 설계 사전 검토** → 어떤 컴포넌트가 필요한지 미리 구조화 가능
- **스타일 가이드 관리** → 컬러 팔레트, 타이포그래피, 간격 시스템을 한 곳에 관리

## 3. 첫 세팅 — 시작 전 알아야 할 것들

### 3-1. 계정 및 설치

1. [figma.com](https://www.figma.com) 가입 (Google 계정 연동 가능)
2. 브라우저에서 바로 사용 가능 (Chrome 권장)
3. 데스크탑 앱도 제공 — 로컬 폰트 사용 시 앱 설치 권장

### 3-2. 파일 구조 이해

```
Organization
└── Team
    └── Project
        └── File (.fig)
            └── Page (탭 개념)
                └── Frame (화면 단위)
                    └── Layer (컴포넌트/그룹/요소)
```

### 3-3. 인터페이스 구성

```
┌──────────────────────────────────────────────────────┐
│  상단 툴바: 도구 선택 / 공유 / 뷰 모드               │
├────────────┬─────────────────────────┬───────────────┤
│  왼쪽 패널 │      캔버스 (작업 영역)  │  오른쪽 패널  │
│  - Layers  │                         │  - Design     │
│  - Assets  │                         │  - Prototype  │
│  - Pages   │                         │  - Inspect    │
└────────────┴─────────────────────────┴───────────────┘
```

- **왼쪽 패널**: 레이어 구조, 에셋(컴포넌트/스타일), 페이지 목록
- **캔버스**: 실제 작업 영역 (무한 캔버스)
- **오른쪽 패널**: 선택한 요소의 속성 (크기, 색상, 폰트 등)

## 4. 디자인 작업 순서 (Workflow)

실제로 2~3일 써보면서 정착된 순서입니다.
이 순서대로 하면 나중에 수정이 훨씬 편합니다.

---

### Step 1. 정보 설계 (Information Architecture)

**코드 치기 전에 화면 구조부터 텍스트로 정리합니다.**

```
메인 페이지
├── 헤더 (로고 / 네비게이션 / 로그인 버튼)
├── 히어로 섹션 (타이틀 / 서브텍스트 / CTA 버튼)
├── 기능 소개 섹션 (카드 3개)
├── 후기 섹션 (슬라이더)
└── 푸터
```

이걸 Figma의 첫 번째 페이지에 텍스트로 적어두면 작업 중 길을 잃지 않습니다.

---

### Step 2. 스타일 가이드(Design Token) 설정

**이걸 먼저 안 하면 나중에 색상, 폰트 통일이 안 돼서 엄청 힘들어집니다.**

새 프레임 만들어서 스타일 가이드 페이지로 활용:

#### 컬러 팔레트 정의

```
Primary    #3B82F6  (Blue-500)
Secondary  #8B5CF6  (Violet-500)
Success    #10B981  (Emerald-500)
Warning    #F59E0B  (Amber-500)
Error      #EF4444  (Red-500)
Text       #111827  (Gray-900)
SubText    #6B7280  (Gray-500)
Border     #E5E7EB  (Gray-200)
Background #F9FAFB  (Gray-50)
```

각 색상을 사각형으로 만들고 → 오른쪽 패널 Fill에서 **Style로 저장** (+ 버튼 클릭 → 이름 지정)

> **팁**: Tailwind CSS를 쓴다면 Tailwind 팔레트 그대로 가져다 쓰면 개발 시 1:1 매핑됩니다.

#### 타이포그래피 정의

| 이름 | 폰트 | 크기 | 굵기 | 용도 |
|------|------|------|------|------|
| Heading/H1 | Inter | 36px | Bold (700) | 페이지 제목 |
| Heading/H2 | Inter | 28px | SemiBold (600) | 섹션 제목 |
| Heading/H3 | Inter | 22px | SemiBold (600) | 카드 제목 |
| Body/Large | Inter | 16px | Regular (400) | 본문 |
| Body/Small | Inter | 14px | Regular (400) | 보조 텍스트 |
| Caption | Inter | 12px | Regular (400) | 라벨, 캡션 |

텍스트 요소 선택 → 오른쪽 Text 영역 Style 저장 (같은 방식)

#### 간격(Spacing) 시스템

```
4px  →  xs
8px  →  sm
12px →  md (자주 씀)
16px →  lg
24px →  xl
32px →  2xl
48px →  3xl
64px →  4xl
```

> **팁**: 8px 배수 시스템을 쓰면 개발할 때 Tailwind 간격 클래스와 거의 정확히 매핑됩니다.
> (8px = p-2, 16px = p-4, 24px = p-6 등)

---

### Step 3. 컴포넌트 라이브러리 만들기

**반복 사용되는 요소는 컴포넌트로 먼저 만들어둡니다.**

자주 만드는 기본 컴포넌트:

- **Button** (Primary / Secondary / Ghost / Danger, 크기: sm/md/lg)
- **Input** (Default / Focus / Error / Disabled 상태)
- **Badge / Tag**
- **Avatar**
- **Card**
- **Modal / Dialog**
- **Navigation Bar**

컴포넌트 만드는 법:

1. 요소 선택 → 우클릭 → `Create component` (단축키: `Ctrl+Alt+K`)
2. 왼쪽 패널 Assets 탭에서 관리
3. 캔버스에서 드래그해서 재사용 (인스턴스 생성)

---

### Step 4. 와이어프레임 (Wireframe)

색상 없이 **회색 계열**로 레이아웃만 잡습니다.
완성도보다 **구조와 흐름** 파악이 목적입니다.

1. `F` 키 → 프레임 도구 선택
2. 오른쪽 패널에서 Device 프리셋 선택 (iPhone 14, Desktop 1440 등)
3. 회색 사각형으로 영역만 표시
4. 텍스트는 "Lorem ipsum" 또는 실제 텍스트로 대체

> **팁**: 와이어프레임 단계에서 레이아웃 피드백 받고 확정한 뒤 색상 입히세요.
> 색상 입히고 나서 레이아웃 바꾸면 두 배로 힘듭니다.

---

### Step 5. 시각 디자인 (Visual Design)

와이어프레임에 스타일을 입히는 단계입니다.

1. 앞서 정의한 컬러/타이포 스타일 적용
2. 컴포넌트 라이브러리에서 버튼, 인풋 등 가져와 배치
3. 그림자(Drop Shadow), 테두리(Stroke), 둥글기(Corner Radius) 적용
4. 이미지/아이콘 삽입

#### 그림자 팁

```
카드 기본 그림자:
X: 0, Y: 1, Blur: 3, Spread: 0, Color: #000000 / 10%

카드 hover 그림자:
X: 0, Y: 4, Blur: 12, Spread: 0, Color: #000000 / 15%

모달 그림자:
X: 0, Y: 20, Blur: 40, Spread: 0, Color: #000000 / 25%
```

---

### Step 6. 반응형 처리

**Auto Layout + Constraints** 로 반응형을 구현합니다.

1. 프레임 너비를 변경했을 때 내부 요소가 어떻게 반응할지 설정
2. 모바일(375px), 태블릿(768px), 데스크탑(1440px) 프레임을 각각 만들어두는 게 편함

---

### Step 7. 프로토타입 연결 (선택)

상단 탭 `Prototype` 선택 → 요소 간 화살표로 인터랙션 연결
버튼 클릭 시 다음 화면으로 이동하는 흐름 시뮬레이션 가능

팀원/클라이언트에게 공유 시 유용합니다.

---

### Step 8. 개발자 핸드오프

**개발 시작 전 Inspect 탭(또는 Dev Mode)에서 모든 수치를 확인합니다.**

- 요소 클릭 → 오른쪽 `Inspect` 탭
- CSS 속성 자동 생성 (margin, padding, font-size, color 등)
- 이미지/아이콘 SVG/PNG로 내보내기

## 5. 핵심 개념 정리

### 5-1. 프레임(Frame) vs 그룹(Group)

| 구분 | Frame | Group |
|------|-------|-------|
| 단축키 | `F` | `Ctrl+G` |
| 클리핑 | 내부 요소가 프레임 밖으로 나가면 잘림 | 요소 크기에 맞게 자동 조정 |
| Auto Layout | 적용 가능 | 불가 |
| Constraints | 설정 가능 | 불가 |
| 용도 | 화면/섹션/컴포넌트 단위 | 단순 묶음 |

**거의 대부분은 Frame을 씁니다.** Group은 단순 이동/복사용으로만 활용.

---

### 5-2. 컴포넌트(Component) & 인스턴스

```
메인 컴포넌트 (1개) ← 수정하면 모든 인스턴스에 반영
      ↓ 복사
인스턴스 A  인스턴스 B  인스턴스 C
(개별 오버라이드 가능)
```

- **메인 컴포넌트**: 💠 아이콘, Assets 패널에서 관리
- **인스턴스**: 드래그해서 사용, 개별 텍스트/색상 오버라이드 가능
- 메인 컴포넌트 수정 → 전체 인스턴스 자동 업데이트

> **팁**: 컴포넌트 이름을 `Button/Primary/Large` 처럼 `/` 로 구분하면
> Assets 패널에서 자동으로 폴더 구조로 분류됩니다.

---

### 5-3. Auto Layout

CSS의 `flexbox`와 거의 동일한 개념입니다.
개발자라면 바로 이해됩니다.

```
Auto Layout = display: flex
방향         = flex-direction (row / column)
간격         = gap
패딩         = padding
정렬         = justify-content / align-items
```

사용법:

1. 요소 선택 → `Shift+A`
2. 오른쪽 패널에서 방향, 간격, 패딩 조정

**Auto Layout을 쓰면 요소 추가/삭제 시 자동으로 크기가 조정됩니다.**
버튼 텍스트 길어져도 자동으로 늘어남 → 개발과 동일한 동작!

#### Hug vs Fill vs Fixed

| 옵션 | CSS 대응 | 설명 |
|------|----------|------|
| Hug contents | `width: fit-content` | 내용물 크기에 맞게 축소 |
| Fill container | `flex: 1` / `width: 100%` | 부모 크기에 맞게 확장 |
| Fixed | `width: 200px` | 고정 크기 |

---

### 5-4. Variant & Props

컴포넌트의 **상태(state)를 한 곳에 모아서 관리**하는 기능입니다.

예: Button 컴포넌트

```
Variant: Type     → Primary / Secondary / Ghost
Variant: Size     → sm / md / lg
Variant: State    → Default / Hover / Disabled / Loading
```

만드는 법:

1. 컴포넌트 여러 개 선택 → 오른쪽 패널 `Combine as variants`
2. 각 컴포넌트에 속성 이름/값 입력
3. 인스턴스에서 드롭다운으로 상태 전환 가능

---

### 5-5. 스타일(Styles) & 변수(Variables)

#### Styles (구버전 방식)

- 컬러, 타이포그래피, 그림자, 그리드를 스타일로 저장
- 라이브러리로 팀 전체에 공유 가능

#### Variables (신기능, 2023~)

- 디자인 토큰 개념과 동일
- 컬러 변수, 숫자 변수, 텍스트 변수 지원
- **라이트/다크 모드** 전환을 변수로 구현 가능

```
Color Variables:
  ├── Primitive
  │   ├── blue-500: #3B82F6
  │   └── gray-900: #111827
  └── Semantic
      ├── color/primary: {blue-500}
      └── color/text: {gray-900}
```

> **팁**: Variables의 Mode 기능으로 다크모드를 구현하면
> 버튼 하나로 라이트↔다크 미리보기 전환이 됩니다.

| `Ctrl+/` | 빠른 검색 (명령 팔레트) |

## 7. 프론트엔드 개발자를 위한 팁

### 팁 1. Inspect 탭 적극 활용

요소 클릭 → 오른쪽 `Inspect` 탭 →
CSS 코드, 크기, 간격, 색상 코드 자동 생성됩니다.

```css
/* Figma가 자동으로 뽑아주는 CSS 예시 */
.button {
  display: flex;
  align-items: center;
  padding: 10px 20px;
  background: #3B82F6;
  border-radius: 8px;
  font-size: 14px;
  font-weight: 600;
  color: #FFFFFF;
  box-shadow: 0px 1px 3px rgba(0, 0, 0, 0.1);
}
```

### 팁 2. 아이콘은 Iconify 플러그인 사용

플러그인 검색: `Ctrl+/` → "Plugins" → "Iconify"
Lucide, Heroicons, Tabler 등 개발에서 쓰는 아이콘 세트를 바로 사용 가능

### 팁 3. 실제 데이터로 테스트

"Lorem ipsum" 대신 실제 데이터를 넣어봐야 엣지 케이스를 발견합니다.

- 이름이 매우 길 때
- 데이터가 없을 때 (Empty state)
- 리스트가 1개일 때 vs 100개일 때

### 팁 4. 개발자 모드(Dev Mode) 활용

상단 툴바 우측 `</>` 버튼 → Dev Mode 전환
선택한 요소의 치수, 간격, 색상, 폰트를 정확하게 확인 가능
이미지/SVG 내보내기도 이 모드에서 가장 편합니다.

### 팁 5. Tailwind CSS 연동

**"Figma to Tailwind"** 플러그인 사용 시 선택한 요소를 Tailwind 클래스로 변환해줍니다.
완벽하진 않지만 초안 잡을 때 매우 유용합니다.

### 팁 6. 그리드 시스템 설정

프레임 선택 → 오른쪽 패널 `Grid` 추가

```
Desktop (1440px): Column 12개, Gutter 24px, Margin 80px
Tablet  (768px):  Column 8개,  Gutter 16px, Margin 32px
Mobile  (375px):  Column 4개,  Gutter 16px, Margin 16px
```

이 그리드 설정을 Style로 저장해두면 모든 프레임에서 재사용 가능합니다.

### 팁 7. 섬네일 설정

파일 목록에 보이는 썸네일 = 첫 번째 페이지의 첫 번째 프레임
`Cover` 라는 이름의 프레임이 있으면 그게 썸네일로 우선 사용됩니다.

### 팁 8. Figma Mirror (모바일 미리보기)

스마트폰에서 Figma 앱 설치 → 같은 계정 로그인
디자인 중인 모바일 화면을 실제 기기에서 실시간으로 미리보기 가능

## 8. 2~3일 사용 후기 (솔직 후기)

### 좋았던 점

**1. 개발자가 이해하기 쉬운 구조**
Auto Layout이 flexbox와 동일해서 "아 이게 `display: flex; gap: 12px`이구나" 바로 직결됩니다.
컴포넌트 시스템도 React 컴포넌트와 개념이 비슷해서 금방 익숙해졌습니다.

**2. Inspect로 개발 시간 절약**
디자인 → 개발 전환할 때 모든 수치가 자동으로 뽑혀서
"이 버튼 패딩이 얼마지?" 같은 고민이 사라집니다.

**3. 실시간 협업이 진짜임**
구글 독스처럼 팀원이 동시에 수정하고, 커서가 실시간으로 보입니다.
원격 팀과 화면 공유 없이 같이 디자인 리뷰 가능합니다.

**4. 커뮤니티 리소스가 풍부함**
Figma Community에서 무료 UI Kit, 템플릿, 아이콘 라이브러리를 바로 가져다 쓸 수 있습니다.
Material Design 3, Apple HIG 컴포넌트도 공식으로 제공됩니다.

---

### 어려웠던 점

**1. 처음엔 Auto Layout이 헷갈림**
Hug / Fill / Fixed 개념이 직관적이지 않아서 처음에 자꾸 레이아웃이 깨졌습니다.
flexbox 이해가 있으면 2~3시간이면 적응 가능합니다.

**2. 컴포넌트 구조 설계가 어려움**
어디까지 컴포넌트로 만들고 어디서부터 그냥 복사할지 판단이 쉽지 않습니다.
처음엔 너무 잘게 쪼개거나, 반대로 너무 크게 만드는 실수를 합니다.

**3. 한글 폰트 렌더링**
브라우저에서 쓰면 한글 폰트가 다소 어색하게 보일 때가 있습니다.
데스크탑 앱 + 로컬 폰트 사용하면 해결됩니다.

**4. 무료 플랜 제한**
팀 협업은 유료 플랜(Pro $15/월)이 필요합니다.
개인 작업 + 공유 링크(보기 전용)는 무료로 가능합니다.

---

### 결론

> **프론트엔드 개발자라면 Figma를 기본 소양으로 배워두는 게 맞습니다.**

디자이너와 협업하든, 혼자 개발하든 Figma를 모르면 손해입니다.
진입 장벽이 높지 않고, 2~3일이면 기본적인 화면 설계는 충분히 가능합니다.
Auto Layout과 컴포넌트 시스템만 제대로 이해하면 나머지는 쓰면서 익숙해집니다.

**추천 학습 순서:**

1. 유튜브 "Figma 기초" 영상 1개 (1~2시간짜리)
2. 직접 토이 프로젝트 하나 컴포넌트 설계해보기
3. Figma Community에서 잘 만든 UI Kit 뜯어보기
4. 실제 프로젝트에 적용
