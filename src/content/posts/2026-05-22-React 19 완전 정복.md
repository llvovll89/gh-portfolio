React 18이 Concurrent Mode와 Suspense를 안정화한 해였다면, **React 19는 그 위에 개발 경험(DX)을 대폭 개선한 해**입니다. 새로 추가된 훅들은 기존에 `useState + useEffect + try/catch`로 길게 작성하던 패턴을 훨씬 간결하게 만들어주고, 비동기 처리와 폼 상태 관리 방식을 근본적으로 바꿉니다. 이 글에서는 React 19의 핵심 변경사항을 실제 코드와 함께 하나씩 파헤칩니다.

> **목표:** React 19에서 새로 추가된 훅과 패턴을 이해하고, 기존 코드를 더 간결하고 안전하게 리팩터링할 수 있는 능력을 기른다.

## 1. React 19 한눈에 보기

React 19의 변화는 크게 세 가지 흐름으로 정리할 수 있습니다.

| 카테고리 | 주요 변경 | 해결하는 문제 |
|---------|----------|------------|
| **비동기/폼** | Actions, useActionState, useFormStatus | 폼 제출 + 로딩/에러 상태 보일러플레이트 제거 |
| **낙관적 UI** | useOptimistic | 서버 응답 전에 UI를 먼저 업데이트하는 패턴 표준화 |
| **리소스 읽기** | use() | Promise, Context를 렌더 중 직접 읽기 |
| **ref 개선** | ref as prop | forwardRef 불필요, cleanup 함수 지원 |
| **메타데이터** | `<title>`, `<meta>` 자동 호이스팅 | 컴포넌트 내 문서 메타데이터 직접 선언 |

## 2. Actions

React 19 이전에는 비동기 이벤트 핸들러를 이렇게 작성했습니다.

### 기존 패턴 (React 18)

```tsx
function UpdateNameForm() {
  const [name, setName] = useState('');
  const [isPending, setIsPending] = useState(false);
  const [error, setError] = useState<string | null>(null);

  const handleSubmit = async () => {
    setIsPending(true);
    setError(null);
    try {
      await updateName(name);
    } catch (e) {
      setError((e as Error).message);
    } finally {
      setIsPending(false);
    }
  };

  return (
    <div>
      <input value={name} onChange={e => setName(e.target.value)} />
      <button onClick={handleSubmit} disabled={isPending}>
        {isPending ? '저장 중...' : '저장'}
      </button>
      {error && <p className="text-red-500">{error}</p>}
    </div>
  );
}
```

### React 19 Actions 패턴

React 19에서는 **트랜지션 내에서 비동기 함수를 쓰면 자동으로 pending 상태를 관리**해줍니다.

```tsx
function UpdateNameForm() {
  const [name, setName] = useState('');
  const [error, setError] = useState<string | null>(null);
  const [isPending, startTransition] = useTransition();

  const handleSubmit = () => {
    startTransition(async () => {
      const result = await updateName(name);
      if (result.error) {
        setError(result.error);
      }
    });
  };

  return (
    <div>
      <input value={name} onChange={e => setName(e.target.value)} />
      <button onClick={handleSubmit} disabled={isPending}>
        {isPending ? '저장 중...' : '저장'}
      </button>
      {error && <p className="text-red-500">{error}</p>}
    </div>
  );
}
```

`useTransition`이 비동기 함수를 받을 수 있게 되면서, `isPending` 상태를 직접 관리할 필요가 없어졌습니다.

## 3. useActionState

폼 제출 패턴을 위해 특화된 훅입니다. 이전 상태와 액션 결과를 통합해서 관리합니다.

### 시그니처

```tsx
const [state, action, isPending] = useActionState(
  actionFn,   // (prevState, formData) => newState
  initialState
);
```

### 실전 예제 — 댓글 작성 폼

```tsx
interface CommentState {
  error: string | null;
  success: boolean;
}

async function submitComment(
  prevState: CommentState,
  formData: FormData
): Promise<CommentState> {
  const content = formData.get('content') as string;

  if (!content.trim()) {
    return { error: '내용을 입력해주세요.', success: false };
  }

  try {
    await createComment({ content });
    return { error: null, success: true };
  } catch {
    return { error: '댓글 작성에 실패했습니다.', success: false };
  }
}

function CommentForm() {
  const [state, action, isPending] = useActionState(submitComment, {
    error: null,
    success: false,
  });

  return (
    <form action={action}>
      <textarea name="content" placeholder="댓글을 입력하세요" rows={3} />

      {state.error && (
        <p className="text-red-500 text-sm">{state.error}</p>
      )}
      {state.success && (
        <p className="text-green-500 text-sm">댓글이 작성되었습니다!</p>
      )}

      <button type="submit" disabled={isPending}>
        {isPending ? '작성 중...' : '댓글 작성'}
      </button>
    </form>
  );
}
```

> **핵심 포인트:** `<form action={action}>`처럼 form의 action에 직접 함수를 넘길 수 있습니다. 이는 HTML의 네이티브 form 동작과 통합되어, 자바스크립트가 로드되기 전에도 작동할 수 있는 기반이 됩니다.

## 4. useFormStatus

`useFormStatus`는 **가장 가까운 부모 `<form>`의 제출 상태를 자식 컴포넌트에서 읽을 수 있게** 해주는 훅입니다. Context 없이 폼 상태를 자식에게 전달할 수 있습니다.

### 기존 방식의 문제

```tsx
// 기존: isPending을 prop으로 내려줘야 했음
function SubmitButton({ isPending }: { isPending: boolean }) {
  return <button disabled={isPending}>저장</button>;
}
```

### useFormStatus 활용

```tsx
import { useFormStatus } from 'react-dom';

// 이 컴포넌트는 반드시 <form> 내부에 있어야 함
function SubmitButton() {
  const { pending } = useFormStatus();

  return (
    <button
      type="submit"
      disabled={pending}
      className={pending ? 'opacity-50 cursor-not-allowed' : ''}
    >
      {pending ? (
        <span className="flex items-center gap-2">
          <Spinner size={16} />
          저장 중...
        </span>
      ) : (
        '저장'
      )}
    </button>
  );
}

// 사용처 — isPending prop 전달 불필요
function ProfileForm() {
  const [state, action] = useActionState(updateProfile, null);

  return (
    <form action={action}>
      <input name="name" />
      <input name="email" type="email" />
      <SubmitButton /> {/* prop 없이 자체적으로 pending 감지 */}
    </form>
  );
}
```

### useFormStatus가 반환하는 값

```tsx
const {
  pending,  // boolean — 폼 제출 중 여부
  data,     // FormData | null — 제출 중인 데이터
  method,   // string — GET/POST
  action,   // string | function — form의 action
} = useFormStatus();
```

## 5. useOptimistic

낙관적 UI(Optimistic UI)는 서버 응답을 기다리지 않고 **UI를 먼저 업데이트한 뒤 실패하면 롤백**하는 패턴입니다. React 19는 이를 내장 훅으로 지원합니다.

### 시그니처

```tsx
const [optimisticState, addOptimistic] = useOptimistic(
  actualState,      // 서버에서 오는 실제 상태
  updateFn          // (state, optimisticValue) => newState
);
```

### 실전 예제 — 좋아요 버튼

```tsx
interface Post {
  id: number;
  likeCount: number;
  isLiked: boolean;
}

function LikeButton({ post }: { post: Post }) {
  const [optimisticPost, addOptimisticLike] = useOptimistic(
    post,
    (state, liked: boolean) => ({
      ...state,
      isLiked: liked,
      likeCount: liked ? state.likeCount + 1 : state.likeCount - 1,
    })
  );

  const handleLike = async () => {
    const newLiked = !optimisticPost.isLiked;

    // UI 즉시 업데이트
    addOptimisticLike(newLiked);

    // 서버 요청 (실패 시 자동 롤백)
    try {
      await toggleLike(post.id, newLiked);
    } catch {
      // useOptimistic은 서버 상태(post)로 자동 복구됨
      toast.error('좋아요 처리에 실패했습니다.');
    }
  };

  return (
    <button
      onClick={handleLike}
      className={optimisticPost.isLiked ? 'text-red-500' : 'text-gray-400'}
    >
      ♥ {optimisticPost.likeCount}
    </button>
  );
}
```

### 실전 예제 — 할일 목록 추가

```tsx
function TodoList({ todos }: { todos: Todo[] }) {
  const [optimisticTodos, addOptimisticTodo] = useOptimistic(
    todos,
    (state, newTodo: Todo) => [...state, newTodo]
  );

  const [, submitAction, isPending] = useActionState(
    async (_: null, formData: FormData) => {
      const title = formData.get('title') as string;
      const tempTodo = { id: Date.now(), title, done: false };

      addOptimisticTodo(tempTodo); // 즉시 목록에 추가

      await createTodo({ title }); // 서버 저장
      return null;
    },
    null
  );

  return (
    <div>
      <ul>
        {optimisticTodos.map(todo => (
          <li key={todo.id} className={todo.id < 0 ? 'opacity-50' : ''}>
            {todo.title}
          </li>
        ))}
      </ul>
      <form action={submitAction}>
        <input name="title" placeholder="새 할일" />
        <button type="submit" disabled={isPending}>추가</button>
      </form>
    </div>
  );
}
```

## 6. use()

`use()`는 기존 훅과 달리 **조건문이나 반복문 안에서도 호출할 수 있는 특별한 훅**입니다. Promise나 Context를 읽을 때 사용합니다.

### Context 읽기

```tsx
// 기존
function MyComponent() {
  const theme = useContext(ThemeContext); // 항상 최상단에서만 호출 가능
  return <div>{theme}</div>;
}

// React 19 — 조건부로도 가능
function MyComponent({ showTheme }: { showTheme: boolean }) {
  if (showTheme) {
    const theme = use(ThemeContext); // ← 조건문 안에서 사용 가능
    return <div>{theme}</div>;
  }
  return <div>기본 UI</div>;
}
```

### Promise 읽기 (Suspense와 함께)

```tsx
// 데이터 페칭 함수가 Promise를 반환
async function fetchUser(id: number): Promise<User> {
  const res = await fetch(`/api/users/${id}`);
  return res.json();
}

// Suspense 경계 안에서 use()로 직접 읽기
function UserProfile({ userPromise }: { userPromise: Promise<User> }) {
  const user = use(userPromise); // 데이터가 없으면 Suspense로 위임

  return (
    <div>
      <h2>{user.name}</h2>
      <p>{user.email}</p>
    </div>
  );
}

// 부모 컴포넌트
function App() {
  const userPromise = fetchUser(1); // 렌더 외부에서 생성

  return (
    <Suspense fallback={<Skeleton />}>
      <UserProfile userPromise={userPromise} />
    </Suspense>
  );
}
```

> **주의:** `use(promise)`에서 Promise는 **컴포넌트 외부에서 생성**해야 합니다. 렌더마다 새 Promise가 생성되면 무한 재시도가 발생합니다.

## 7. ref의 변화

### forwardRef 없이 ref 전달

React 19부터 함수형 컴포넌트에서 `ref`를 일반 prop처럼 받을 수 있습니다.

```tsx
// 기존 — forwardRef 래핑 필요
const Input = forwardRef<HTMLInputElement, InputProps>(
  ({ label, ...props }, ref) => (
    <div>
      <label>{label}</label>
      <input ref={ref} {...props} />
    </div>
  )
);

// React 19 — ref를 그냥 prop으로 받음
function Input({ label, ref, ...props }: InputProps & { ref?: React.Ref<HTMLInputElement> }) {
  return (
    <div>
      <label>{label}</label>
      <input ref={ref} {...props} />
    </div>
  );
}

// 사용 방법은 동일
const inputRef = useRef<HTMLInputElement>(null);
<Input label="이름" ref={inputRef} />
```

### ref 클린업 함수

```tsx
function VideoPlayer({ src }: { src: string }) {
  return (
    <video
      ref={(node) => {
        if (node) {
          // 마운트 시 실행
          node.play();
        }
        // 클린업 함수 반환 — 언마운트 시 실행
        return () => {
          node?.pause();
        };
      }}
      src={src}
    />
  );
}
```

## 8. 기타 개선사항

### 메타데이터 자동 호이스팅

컴포넌트 안에서 `<title>`, `<meta>`, `<link>`를 선언하면 React가 자동으로 `<head>`로 옮겨줍니다.

```tsx
function BlogPost({ post }: { post: Post }) {
  return (
    <article>
      {/* React가 자동으로 <head>에 삽입 */}
      <title>{post.title} | 내 블로그</title>
      <meta name="description" content={post.summary} />
      <link rel="canonical" href={`https://myblog.com/posts/${post.slug}`} />

      <h1>{post.title}</h1>
      <div>{post.content}</div>
    </article>
  );
}
```

별도의 `react-helmet`이나 `next/head` 없이 네이티브하게 동작합니다.

### Context 간소화

```tsx
// 기존 — <ThemeContext.Provider>
<ThemeContext.Provider value={theme}>
  {children}
</ThemeContext.Provider>

// React 19 — Context를 직접 사용
<ThemeContext value={theme}>
  {children}
</ThemeContext>
```

### useDeferredValue 초기값 지원

```tsx
function SearchResults({ query }: { query: string }) {
  // 초기 렌더에 사용할 기본값 설정 가능
  const deferredQuery = useDeferredValue(query, '');

  return <Results query={deferredQuery} />;
}
```

## 9. 마이그레이션 가이드

기존 React 18 코드를 React 19 스타일로 점진적으로 전환하는 방법입니다.

### 폼 제출 패턴 변환

```tsx
// Before (React 18)
function LoginForm() {
  const [email, setEmail] = useState('');
  const [password, setPassword] = useState('');
  const [isLoading, setIsLoading] = useState(false);
  const [error, setError] = useState('');

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    setIsLoading(true);
    setError('');
    try {
      await login({ email, password });
      navigate('/dashboard');
    } catch (err) {
      setError('로그인에 실패했습니다.');
    } finally {
      setIsLoading(false);
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      <input value={email} onChange={e => setEmail(e.target.value)} />
      <input value={password} onChange={e => setPassword(e.target.value)} type="password" />
      <button disabled={isLoading}>로그인</button>
      {error && <p>{error}</p>}
    </form>
  );
}
```

```tsx
// After (React 19)
interface LoginState {
  error: string | null;
}

async function loginAction(
  _: LoginState,
  formData: FormData
): Promise<LoginState> {
  const email = formData.get('email') as string;
  const password = formData.get('password') as string;

  try {
    await login({ email, password });
    redirect('/dashboard');
    return { error: null };
  } catch {
    return { error: '로그인에 실패했습니다.' };
  }
}

function SubmitBtn() {
  const { pending } = useFormStatus();
  return <button disabled={pending}>{pending ? '로그인 중...' : '로그인'}</button>;
}

function LoginForm() {
  const [state, action] = useActionState(loginAction, { error: null });

  return (
    <form action={action}>
      <input name="email" type="email" />
      <input name="password" type="password" />
      <SubmitBtn />
      {state.error && <p>{state.error}</p>}
    </form>
  );
}
```

**코드량이 절반 이하로 줄었고, controlled input이 필요 없어졌습니다.**

## 10. 판단 기준

새 API들이 생겼다고 해서 모든 곳에 적용할 필요는 없습니다.

| 상황 | 권장 접근 |
|------|---------|
| 폼 제출 + 로딩/에러 상태 | `useActionState` + `useFormStatus` |
| 서버 응답 전 UI 미리 반영 | `useOptimistic` |
| 간단한 버튼 클릭 + 비동기 | `useTransition` (Actions) |
| Context를 조건부로 읽기 | `use(Context)` |
| Suspense와 함께 데이터 페칭 | `use(Promise)` |
| 컴포넌트 ref 전달 | ref as prop (forwardRef 제거) |
| **기존 코드가 잘 동작 중** | **그대로 유지** |

### 한 줄 요약

- `useActionState` — 폼의 상태 머신을 한 훅으로
- `useFormStatus` — 폼 상태를 자식이 직접 구독
- `useOptimistic` — 낙관적 업데이트 표준화
- `use()` — 조건부 Context/Promise 읽기
- ref as prop — forwardRef 보일러플레이트 제거

## 마무리

React 19의 변화는 "새로운 기능 추가"라기보다 **"기존에 반복적으로 작성하던 코드를 줄이는 것"**에 가깝습니다. 특히 폼 처리, 낙관적 업데이트, 비동기 상태 관리에서 그 효과가 두드러집니다.

당장 모든 코드를 마이그레이션할 필요는 없지만, **새로 작성하는 폼 컴포넌트부터 `useActionState`와 `useFormStatus`를 적용**해보는 것을 추천합니다. 코드량이 줄고 가독성이 높아지는 것을 바로 체감할 수 있습니다.

React 팀은 이 새 API들을 통해 클라이언트 컴포넌트와 서버 컴포넌트(RSC) 사이의 경계를 자연스럽게 넘나들 수 있는 기반을 만들고 있습니다. React 19를 충분히 익혀두면, 앞으로 나올 서버 중심 패러다임에도 훨씬 유연하게 적응할 수 있을 것입니다.
