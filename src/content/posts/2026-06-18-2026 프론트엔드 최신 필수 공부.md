## 1. React 19 핵심 변경점

React 19는 2024년 말 정식 출시됐고, 2026년 현재 사실상 표준이 됐다. 단순한 버전 업이 아니라 **서버와 클라이언트의 경계를 재정의**한 릴리즈다.

### `use` 훅 — Promise와 Context를 직접 언래핑

```tsx
import { use } from 'react';

function UserProfile({ userPromise }: { userPromise: Promise<User> }) {
  // Suspense 경계 안에서 Promise를 동기처럼 읽는다
  const user = use(userPromise);
  return <h1>{user.name}</h1>;
}
```

조건문 안에서도 사용할 수 있다는 점이 기존 훅 규칙과 다르다.

### Server Actions — 폼 처리의 패러다임 전환

```tsx
// actions.ts (서버에서 실행)
'use server';

export async function updateProfile(formData: FormData) {
  const name = formData.get('name') as string;
  await db.user.update({ where: { id: session.userId }, data: { name } });
  revalidatePath('/profile');
}

// ProfileForm.tsx (클라이언트)
import { updateProfile } from './actions';

export function ProfileForm() {
  return (
    <form action={updateProfile}>
      <input name="name" />
      <button type="submit">저장</button>
    </form>
  );
}
```

API 라우트 없이 서버 함수를 폼에 직접 연결한다.

### useOptimistic — 낙관적 업데이트

```tsx
function TodoList({ todos }: { todos: Todo[] }) {
  const [optimisticTodos, addOptimisticTodo] = useOptimistic(
    todos,
    (state, newTodo: Todo) => [...state, newTodo]
  );

  async function handleAdd(formData: FormData) {
    const title = formData.get('title') as string;
    addOptimisticTodo({ id: Date.now(), title, pending: true });
    await addTodoAction(formData);
  }

  return (
    <form action={handleAdd}>
      {optimisticTodos.map(todo => (
        <li key={todo.id} style={{ opacity: todo.pending ? 0.5 : 1 }}>
          {todo.title}
        </li>
      ))}
      <input name="title" />
    </form>
  );
}
```

### ref가 props로

React 19부터 `forwardRef` 없이 `ref`를 props로 받는다.

```tsx
// Before (React 18)
const Input = forwardRef<HTMLInputElement, Props>((props, ref) => (
  <input ref={ref} {...props} />
));

// After (React 19)
function Input({ ref, ...props }: Props & { ref?: React.Ref<HTMLInputElement> }) {
  return <input ref={ref} {...props} />;
}
```

## 2. TypeScript 5.x — 알아야 할 새 문법

### Const Type Parameters (5.0)

```ts
function identity<const T>(value: T): T {
  return value;
}

const result = identity({ x: 10, y: 20 });
// result: { x: 10, y: 20 } (리터럴 타입 유지, as const 불필요)
```

### Variadic Tuple Types 실용 패턴

```ts
type Middleware<T extends unknown[]> = (...args: T) => void;

function compose<T extends unknown[]>(...fns: Middleware<T>[]): Middleware<T> {
  return (...args) => fns.forEach(fn => fn(...args));
}
```

### satisfies 연산자 (5.x에서 더 많이 쓰임)

```ts
const palette = {
  red: [255, 0, 0],
  green: '#00ff00',
} satisfies Record<string, string | number[]>;

// palette.red는 number[] 타입으로 추론됨 (as 캐스팅 불필요)
palette.red.map(v => v * 2);
```

### 타입 가져오기 명시화 (`import type`)

```ts
// 5.x 권장 — 번들러가 타입만 지우면 되는 import를 구분
import type { User } from './types';
import { createUser } from './api';
```

## 3. Next.js 15와 App Router 성숙화

App Router는 2026년 현재 Pages Router를 완전히 대체하는 메인스트림이 됐다.

### 알아야 할 핵심 개념

| 개념 | 설명 |
|------|------|
| Server Components | 서버에서만 실행, JS 번들에 포함 안 됨 |
| Client Components | `'use client'` 선언, 브라우저에서 실행 |
| Streaming | `<Suspense>`로 부분적 HTML 스트리밍 |
| Parallel Routes | `@slot` 폴더로 동시 렌더링 레이아웃 |
| Intercepting Routes | `(.)` 문법으로 모달 라우팅 구현 |

### 서버 컴포넌트에서 직접 데이터 페칭

```tsx
// app/blog/[slug]/page.tsx
// fetch는 자동으로 캐싱됨 (Next.js 확장 fetch)
async function BlogPost({ params }: { params: { slug: string } }) {
  const post = await fetch(`https://api.example.com/posts/${params.slug}`, {
    next: { revalidate: 3600 }, // 1시간 캐시
  }).then(r => r.json());

  return <article>{post.content}</article>;
}
```

### `next/cache` API

```tsx
import { revalidatePath, revalidateTag, unstable_cache } from 'next/cache';

const getUser = unstable_cache(
  async (id: string) => db.user.findUnique({ where: { id } }),
  ['user'],
  { tags: ['users'], revalidate: 60 }
);
```

## 4. Tailwind CSS v4 — 설정 방식의 전환

`tailwind.config.js`가 사라지고 CSS 파일이 설정의 중심이 됐다.

```css
/* global.css */
@import "tailwindcss";

@theme {
  --color-brand: #6366f1;
  --font-sans: 'Pretendard', sans-serif;
  --spacing-18: 4.5rem;
  --breakpoint-3xl: 1920px;
}

@utility card {
  border-radius: var(--radius-lg);
  padding: var(--spacing-6);
  background: var(--color-white);
  box-shadow: var(--shadow-md);
}
```

### 달라진 점 요약

| 항목 | v3 | v4 |
|------|-----|-----|
| 설정 파일 | `tailwind.config.js` | CSS `@theme` 블록 |
| 커스텀 유틸리티 | `addUtilities` 플러그인 | `@utility` 지시어 |
| CSS 변수 | 수동 작성 | `@theme`에서 자동 생성 |
| JIT | 기본 | 항상 적용 |
| 번들 크기 | 큼 | Oxide 엔진으로 ~35% 감소 |

## 5. 네이티브 CSS가 달라진 것들

JS 없이도 가능한 게 많아졌다. 2026년 기준 주요 브라우저 지원률이 90%를 넘어선 기능들이다.

### CSS Anchor Positioning

```css
.tooltip {
  position: absolute;
  position-anchor: --button;
  top: anchor(bottom);
  left: anchor(center);
  translate: -50% 8px;
}

.button {
  anchor-name: --button;
}
```

JS로 직접 좌표 계산하던 툴팁, 팝오버, 드롭다운이 CSS만으로 된다.

### Scroll-Driven Animations

```css
@keyframes reveal {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}

.card {
  animation: reveal linear both;
  animation-timeline: view();
  animation-range: entry 0% entry 30%;
}
```

IntersectionObserver + JS 없이 스크롤 진입 애니메이션 구현.

### View Transitions API

```tsx
function navigate(path: string) {
  if (!document.startViewTransition) {
    router.push(path);
    return;
  }

  document.startViewTransition(() => {
    router.push(path);
  });
}
```

```css
::view-transition-old(root) {
  animation: slide-out 300ms ease-in;
}
::view-transition-new(root) {
  animation: slide-in 300ms ease-out;
}
```

### Container Queries

```css
.card-wrapper {
  container-type: inline-size;
}

@container (min-width: 400px) {
  .card {
    display: grid;
    grid-template-columns: 1fr 2fr;
  }
}
```

뷰포트 대신 부모 컨테이너 크기에 반응하는 반응형 컴포넌트.

## 6. 상태 관리의 현재 — Zustand, Jotai, TanStack Query

Redux는 레거시로 분류되는 추세고, 2026년 실무에서 가장 많이 쓰이는 조합은 아래와 같다.

### Zustand — 전역 UI 상태

```ts
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

interface ThemeStore {
  theme: 'light' | 'dark';
  toggle: () => void;
}

const useThemeStore = create<ThemeStore>()(
  persist(
    set => ({
      theme: 'light',
      toggle: () => set(s => ({ theme: s.theme === 'light' ? 'dark' : 'light' })),
    }),
    { name: 'theme-storage' }
  )
);
```

### TanStack Query v5 — 서버 상태

```ts
import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query';

function useUser(id: string) {
  return useQuery({
    queryKey: ['user', id],
    queryFn: () => fetchUser(id),
    staleTime: 5 * 60 * 1000, // 5분
  });
}

function useUpdateUser() {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: updateUser,
    onSuccess: (data) => {
      queryClient.setQueryData(['user', data.id], data);
    },
  });
}
```

### Jotai — 원자적 상태 (복잡한 파생 상태)

```ts
import { atom, useAtom } from 'jotai';

const countAtom = atom(0);
const doubleAtom = atom(get => get(countAtom) * 2);

function Counter() {
  const [count, setCount] = useAtom(countAtom);
  const [double] = useAtom(doubleAtom);
  return <div>{count} × 2 = {double}</div>;
}
```

**선택 가이드:**
- 서버에서 오는 데이터 → TanStack Query
- 전역 UI 상태 (모달, 테마, 유저 세션) → Zustand
- 복잡한 파생/비동기 상태 → Jotai

## 7. 번들러 지형도 — Vite, Turbopack, Rolldown

### Vite 6 + Rolldown

Vite는 개발 서버는 Rollup 기반이었지만, Rolldown(Rust 기반 Rollup 재구현)을 프로덕션 번들러로 채택하면서 빌드 속도가 크게 빨라졌다.

```ts
// vite.config.ts
import { defineConfig } from 'vite';

export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ['react', 'react-dom'],
          router: ['react-router-dom'],
        },
      },
    },
  },
});
```

### Turbopack

Next.js 15 기본 번들러. Rust로 작성됐으며 개발 서버 HMR이 Webpack 대비 최대 10배 빠르다. 아직 프로덕션 빌드는 안정화 진행 중.

### 선택 기준

| 프레임워크 | 권장 번들러 |
|-----------|-----------|
| Next.js | Turbopack (개발), SWC (프로덕션) |
| 그 외 React / Vue / Svelte | Vite + Rolldown |
| 라이브러리 패키징 | tsup (내부적으로 esbuild + rollup) |

## 8. Bun과 Node.js 22

### Bun

Node.js 대체제로 나왔지만, 2026년 현재 대부분 팀은 **패키지 매니저로만** 쓴다. `npm install` 대비 3~5배 빠르다.

```bash
# 설치
bun install

# 스크립트 실행
bun run dev

# 단일 파일 실행
bun index.ts
```

런타임으로 프로덕션에 쓰는 팀은 아직 소수. Node.js 호환성 이슈가 남아있기 때문.

### Node.js 22 주목할 기능

```bash
# --experimental-require-module 없이도 ESM/CJS 혼용 가능
node --experimental-vm-modules test.mjs

# 네이티브 .env 로딩
node --env-file=.env server.js
```

## 10. AI 통합 — Vercel AI SDK와 스트리밍 UI

AI 기능을 프론트엔드에 붙이는 게 일반 피처가 됐다. 가장 많이 쓰이는 패턴은 스트리밍 응답이다.

### Vercel AI SDK

```tsx
'use client';
import { useChat } from 'ai/react';

export function ChatWidget() {
  const { messages, input, handleInputChange, handleSubmit, isLoading } = useChat({
    api: '/api/chat',
  });

  return (
    <div>
      <ul>
        {messages.map(m => (
          <li key={m.id} className={m.role === 'user' ? 'text-right' : 'text-left'}>
            {m.content}
          </li>
        ))}
      </ul>
      <form onSubmit={handleSubmit}>
        <input value={input} onChange={handleInputChange} disabled={isLoading} />
        <button type="submit">전송</button>
      </form>
    </div>
  );
}
```

```ts
// app/api/chat/route.ts
import { streamText } from 'ai';
import { anthropic } from '@ai-sdk/anthropic';

export async function POST(req: Request) {
  const { messages } = await req.json();

  const result = streamText({
    model: anthropic('claude-sonnet-4-6'),
    messages,
  });

  return result.toDataStreamResponse();
}
```

### 알아야 할 AI UI 패턴

- **Streaming**: 토큰 단위 점진적 렌더링
- **Tool Calling**: AI가 함수를 호출해 데이터 조회
- **Generative UI**: AI 응답으로 React 컴포넌트 직접 렌더링
- **RAG UI**: 검색 결과를 컨텍스트로 붙여 응답 품질 향상

## 11. 성능 지표 — INP가 CLS를 밀어냈다

2024년 3월, Google Core Web Vitals에서 **FID(First Input Delay)가 INP(Interaction to Next Paint)로 교체**됐다.

| 지표 | 의미 | 좋음 | 개선 필요 | 나쁨 |
|------|------|------|----------|------|
| LCP | 가장 큰 요소 렌더링 시간 | ≤2.5s | 2.5~4s | >4s |
| INP | 모든 인터랙션 응답성 | ≤200ms | 200~500ms | >500ms |
| CLS | 레이아웃 누적 이동 | ≤0.1 | 0.1~0.25 | >0.25 |

### INP 개선 전략

```tsx
// 무거운 작업을 스케줄링으로 분산
function handleClick() {
  // 즉각 UI 반응 (INP에 기록되는 시점)
  setIsLoading(true);

  // 무거운 작업은 microtask 이후로 밀기
  startTransition(() => {
    processHeavyData();
  });
}
```

```tsx
// scheduler.yield()로 메인 스레드 양보 (Chrome 129+)
async function processLargeList(items: string[]) {
  for (let i = 0; i < items.length; i++) {
    process(items[i]);

    // 50개마다 브라우저에 제어권 반환
    if (i % 50 === 0 && 'scheduler' in globalThis) {
      await (globalThis as any).scheduler.yield();
    }
  }
}
```

## 마무리

2026년 프론트엔드는 **서버와 클라이언트의 경계 재정의**와 **네이티브 브라우저 API의 부상**이라는 두 키워드로 요약된다. React 19의 Server Actions와 `use` 훅, CSS Anchor Positioning과 Scroll-Driven Animations — 이전에 JS 라이브러리에 의존하던 것들이 플랫폼 자체로 흡수되고 있다.

가장 빠른 공부 경로는 현재 프로젝트에 적용해보는 것이다. 새 프로젝트라면 Next.js 15 + TanStack Query + Zustand + Vitest 조합을 기본으로 잡고, 기존 프로젝트라면 React 19 마이그레이션과 Tailwind v4 전환부터 시작해볼 것을 권한다.
