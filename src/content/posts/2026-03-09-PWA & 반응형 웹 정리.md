# PWA & 반응형 웹 정리

**PWA(Progressive Web App)** 는 웹 기술로 만들어졌지만 네이티브 앱처럼 동작하는 웹 애플리케이션입니다.
**반응형 웹(Responsive Web Design)** 은 하나의 코드베이스로 다양한 화면 크기에 최적화된 레이아웃을 제공하는 기법입니다.

> 목표: PWA의 **핵심 구성요소와 동작 원리**를 이해하고, **반응형 웹 디자인 기법**을 실전에서 활용하는 능력 키우기

## 1. PWA란?

### 1.1 핵심 특성

| 특성 | 설명 |
|------|------|
| **Progressive** | 모든 브라우저에서 점진적으로 동작 |
| **Responsive** | 데스크톱·모바일·태블릿 모든 화면 대응 |
| **Connectivity Independent** | 오프라인 또는 불안정한 네트워크에서도 동작 |
| **App-like** | 앱 스타일의 인터랙션과 네비게이션 |
| **Installable** | 홈 화면에 아이콘 추가, 앱처럼 실행 |
| **Linkable** | URL로 공유 가능, 설치 불필요 |

### 1.2 PWA 구성요소

```
PWA
├── Web App Manifest (manifest.json)   → 앱 메타데이터, 아이콘, 시작 URL
├── Service Worker (sw.js)             → 오프라인 캐싱, 백그라운드 동기화
├── HTTPS                              → 보안 연결 필수
└── Responsive Design                  → 모든 기기 대응
```

### 1.3 PWA vs 네이티브 앱

| 항목 | PWA | 네이티브 앱 |
|------|-----|------------|
| 설치 | 브라우저에서 바로 | 앱 스토어 |
| 업데이트 | 자동 (서버 배포) | 사용자가 직접 |
| 용량 | 매우 작음 | 수십~수백 MB |
| 기기 접근 | 제한적 | 풍부함 |
| 오프라인 | Service Worker로 지원 | 기본 지원 |
| 개발 비용 | 단일 코드베이스 | 플랫폼별 개발 |

### 2.2 display 모드

| 모드 | 설명 |
|------|------|
| `standalone` | 브라우저 UI 없이 앱처럼 실행 (가장 많이 사용) |

| `minimal-ui` | 최소한의 브라우저 UI (뒤로가기 버튼 등) |

## 3. Service Worker

### 3.1 개념

Service Worker는 브라우저 백그라운드에서 실행되는 **JavaScript 파일**로, 네트워크 요청을 가로채고 캐싱을 관리합니다.

```
브라우저(페이지) ──요청──→ Service Worker ──→ 캐시 or 네트워크
                            (중간자 역할)
```

### 3.2 Service Worker 생명주기

```
1. Register  → navigator.serviceWorker.register('/sw.js')
2. Install   → self.addEventListener('install', ...)  → 캐시 초기화
3. Activate  → self.addEventListener('activate', ...) → 오래된 캐시 삭제
4. Fetch     → self.addEventListener('fetch', ...)    → 요청 인터셉트
```

### 3.3 등록 (main.js)

```javascript
// 브라우저 지원 확인 후 등록
if ('serviceWorker' in navigator) {
  window.addEventListener('load', async () => {
    try {
      const registration = await navigator.serviceWorker.register('/sw.js', {
        scope: '/',
      });
      console.log('SW registered:', registration.scope);
    } catch (error) {
      console.error('SW registration failed:', error);
    }
  });
}
```

### 3.4 Service Worker 기본 구현 (sw.js)

```javascript
const CACHE_NAME = 'my-app-v1';
const STATIC_ASSETS = [
  '/',
  '/index.html',
  '/styles/main.css',
  '/scripts/app.js',
  '/icons/icon-192x192.png',
];

// Install: 정적 자산 사전 캐싱
self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(CACHE_NAME).then((cache) => {
      console.log('Caching static assets');
      return cache.addAll(STATIC_ASSETS);
    })
  );
  // 즉시 활성화 (대기 없이)
  self.skipWaiting();
});

// Activate: 오래된 캐시 정리
self.addEventListener('activate', (event) => {
  event.waitUntil(
    caches.keys().then((cacheNames) =>
      Promise.all(
        cacheNames
          .filter((name) => name !== CACHE_NAME)
          .map((name) => caches.delete(name))
      )
    )
  );
  // 즉시 클라이언트 제어
  self.clients.claim();
});

// Fetch: 네트워크 요청 인터셉트
self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.match(event.request).then((cached) => {
      // 캐시가 있으면 반환, 없으면 네트워크 요청
      return cached || fetch(event.request);
    })
  );
});
```

## 4. 캐싱 전략

### 4.1 Cache First (캐시 우선)

정적 자산(이미지, CSS, JS)에 적합. 캐시 미스 시에만 네트워크 요청.

```javascript
self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.match(event.request).then((cached) => {
      if (cached) return cached;
      return fetch(event.request).then((response) => {
        const copy = response.clone();
        caches.open(CACHE_NAME).then((cache) => cache.put(event.request, copy));
        return response;
      });
    })
  );
});
```

### 4.2 Network First (네트워크 우선)

최신 데이터가 중요한 API 요청에 적합. 네트워크 실패 시 캐시 폴백.

```javascript
self.addEventListener('fetch', (event) => {
  event.respondWith(
    fetch(event.request)
      .then((response) => {
        const copy = response.clone();
        caches.open(CACHE_NAME).then((cache) => cache.put(event.request, copy));
        return response;
      })
      .catch(() => caches.match(event.request))
  );
});
```

### 4.3 Stale While Revalidate

캐시를 즉시 반환하면서 백그라운드에서 업데이트. 속도와 최신성 균형.

```javascript
self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.open(CACHE_NAME).then((cache) =>
      cache.match(event.request).then((cached) => {
        const networkFetch = fetch(event.request).then((response) => {
          cache.put(event.request, response.clone());
          return response;
        });
        // 캐시가 있으면 즉시 반환, 백그라운드에서 업데이트
        return cached || networkFetch;
      })
    )
  );
});
```

### 4.4 전략 선택 가이드

| 리소스 유형 | 권장 전략 |
|------------|----------|
| HTML 페이지 | Network First |
| CSS / JS (해시 포함) | Cache First |
| 이미지 | Cache First or Stale While Revalidate |
| API 데이터 | Network First or Stale While Revalidate |
| 폰트 | Cache First (장기 캐싱) |

## 5. Push 알림 & Background Sync

### 5.1 Push 알림 흐름

```
1. 사용자 권한 요청 → Notification.requestPermission()
2. 구독 생성       → registration.pushManager.subscribe()
3. 서버에 구독 전송 → fetch('/api/subscribe', { body: subscription })
4. 서버에서 푸시    → web-push 라이브러리 사용
5. SW에서 수신     → self.addEventListener('push', ...)
6. 알림 표시       → self.registration.showNotification(...)
```

```javascript
// sw.js - 푸시 이벤트 처리
self.addEventListener('push', (event) => {
  const data = event.data?.json() ?? { title: '알림', body: '새 메시지가 있습니다.' };
  event.waitUntil(
    self.registration.showNotification(data.title, {
      body: data.body,
      icon: '/icons/icon-192x192.png',
      badge: '/icons/badge-72x72.png',
      data: { url: data.url },
    })
  );
});

// 알림 클릭 처리
self.addEventListener('notificationclick', (event) => {
  event.notification.close();
  event.waitUntil(clients.openWindow(event.notification.data.url));
});
```

### 5.2 Background Sync

오프라인 상태에서 작업을 큐에 쌓고, 온라인 복귀 시 자동 전송.

```javascript
// 페이지: sync 등록
async function sendMessage(data) {
  if ('serviceWorker' in navigator && 'SyncManager' in window) {
    const sw = await navigator.serviceWorker.ready;
    // IndexedDB에 저장 후 sync 등록
    await saveToIndexedDB(data);
    await sw.sync.register('send-message');
  } else {
    // 폴백: 직접 전송
    await fetch('/api/messages', { method: 'POST', body: JSON.stringify(data) });
  }
}

// sw.js: sync 이벤트 처리
self.addEventListener('sync', (event) => {
  if (event.tag === 'send-message') {
    event.waitUntil(flushMessageQueue());
  }
});
```

## 6. PWA 설치 & 배포

### 6.1 설치 조건 (Install Criteria)

- HTTPS 제공 (localhost 예외)
- 유효한 Web App Manifest
- Service Worker 등록됨
- 아이콘 192x192, 512x512 포함

### 6.2 설치 프롬프트 제어

```javascript
let deferredPrompt;

window.addEventListener('beforeinstallprompt', (event) => {
  event.preventDefault();
  deferredPrompt = event;
  // 커스텀 설치 버튼 표시
  document.getElementById('install-btn').style.display = 'block';
});

document.getElementById('install-btn').addEventListener('click', async () => {
  if (!deferredPrompt) return;
  deferredPrompt.prompt();
  const { outcome } = await deferredPrompt.userChoice;
  console.log('Install outcome:', outcome); // 'accepted' or 'dismissed'
  deferredPrompt = null;
});

window.addEventListener('appinstalled', () => {
  console.log('PWA installed!');
});
```

### 6.3 Vite PWA 플러그인 (React/Vue)

```bash
npm install -D vite-plugin-pwa
```

```javascript
// vite.config.js
import { VitePWA } from 'vite-plugin-pwa';

export default {

          },
        ],
      },
    }),
  ],
};
```

## 7. 반응형 웹이란?

### 7.1 핵심 원칙 3가지

```
1. 유동적 그리드 (Fluid Grid)      → % 기반 레이아웃
2. 유연한 이미지 (Flexible Images) → max-width: 100%
3. 미디어 쿼리 (Media Queries)     → 화면 크기별 스타일 분기
```

### 7.2 일반적인 브레이크포인트

| 이름 | 범위 | 대표 기기 |
|------|------|----------|
| `xs` | ~479px | 소형 스마트폰 |
| `sm` | 480~767px | 스마트폰 (landscape) |
| `md` | 768~1023px | 태블릿 |
| `lg` | 1024~1279px | 소형 데스크톱 |
| `xl` | 1280~1535px | 데스크톱 |
| `2xl` | 1536px~ | 대형 모니터 |

## 8. 뷰포트(Viewport)

### 8.1 필수 메타태그

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
```

| 속성 | 설명 |
|------|------|
| `width=device-width` | 뷰포트 너비를 기기 화면 너비로 설정 |
| `initial-scale=1.0` | 초기 줌 레벨 1배 |
| `maximum-scale=1.0` | 줌 비활성화 (접근성 이슈로 비권장) |
| `user-scalable=no` | 사용자 줌 금지 (접근성 이슈로 비권장) |

### 8.2 뷰포트 단위

| 단위 | 설명 | 예시 |
|------|------|------|
| `vw` | 뷰포트 너비의 1% | `100vw` = 전체 너비 |
| `vh` | 뷰포트 높이의 1% | `100vh` = 전체 높이 |
| `vmin` | vw, vh 중 작은 값의 1% | 정사각형 요소에 유용 |
| `vmax` | vw, vh 중 큰 값의 1% | |
| `dvh` | Dynamic viewport height | 모바일 주소창 고려 |
| `svh` | Small viewport height | 주소창 있을 때 높이 |
| `lvh` | Large viewport height | 주소창 없을 때 높이 |

```css
/* 모바일에서 주소창 문제 해결 */
.hero {
  height: 100svh; /* 안전한 뷰포트 높이 */
  /* fallback */
  height: 100vh;
}
```

## 9. 미디어 쿼리(Media Query)

### 9.1 기본 문법

```css
/* 최소 너비 (모바일 퍼스트) */
@media (min-width: 768px) { ... }

/* 최대 너비 (데스크톱 퍼스트) */
@media (max-width: 767px) { ... }

/* 범위 지정 */
@media (min-width: 768px) and (max-width: 1023px) { ... }

/* Modern range syntax (CSS Media Queries Level 4) */
@media (768px <= width <= 1023px) { ... }

/* 다크 모드 */
@media (prefers-color-scheme: dark) { ... }

/* 고해상도 디스플레이 (Retina) */
@media (-webkit-min-device-pixel-ratio: 2), (min-resolution: 192dpi) { ... }

/* 프린트 */
@media print { ... }

/* 가로/세로 모드 */
@media (orientation: landscape) { ... }
@media (orientation: portrait) { ... }

/* 애니메이션 감소 설정 */
@media (prefers-reduced-motion: reduce) {
  * { animation: none !important; transition: none !important; }
}
```

### 9.2 Tailwind CSS 브레이크포인트

```jsx
// Tailwind: 모바일 퍼스트 방식
<div className="
  w-full          /* 기본 (모바일) */
  sm:w-1/2        /* 640px~ */
  md:w-1/3        /* 768px~ */
  lg:w-1/4        /* 1024px~ */
  xl:w-1/5        /* 1280px~ */
">
```

## 10. 유동적 단위(Fluid Units)

### 10.1 상대 단위 비교

| 단위 | 기준 | 사용 예 |
|------|------|--------|
| `px` | 절대 픽셀 | 보더, 그림자 |
| `em` | 현재 요소의 font-size | padding, margin |
| `rem` | 루트(html)의 font-size | 폰트 크기, 간격 |
| `%` | 부모 요소 기준 | 너비, 높이 |
| `vw/vh` | 뷰포트 기준 | 히어로 섹션 |
| `ch` | 현재 폰트 '0'의 너비 | 텍스트 영역 너비 |

### 10.2 clamp()로 유동 폰트 크기

```css
/* clamp(최소값, 선호값, 최대값) */
h1 {
  font-size: clamp(1.5rem, 4vw, 3rem);
  /* 뷰포트에 따라 1.5rem ~ 3rem 사이에서 자동 조절 */
}

.container {
  width: clamp(320px, 90%, 1200px);
  margin: 0 auto;
}
```

### 10.3 CSS 변수로 반응형 관리

```css
:root {
  --spacing-base: clamp(1rem, 2vw, 1.5rem);
  --font-body: clamp(0.875rem, 1.5vw, 1rem);
  --container-max: min(90vw, 1200px);
}

.container {
  max-width: var(--container-max);
  padding: var(--spacing-base);
}
```

## 11. Flexbox

### 11.1 핵심 속성 요약

```css
.container {
  display: flex;

  /* 방향 */
  flex-direction: row | row-reverse | column | column-reverse;

  /* 줄 바꿈 */
  flex-wrap: nowrap | wrap | wrap-reverse;

  /* 주축 정렬 */
  justify-content: flex-start | flex-end | center | space-between | space-around | space-evenly;

  /* 교차축 정렬 (단일 줄) */
  align-items: stretch | flex-start | flex-end | center | baseline;

  /* 교차축 정렬 (여러 줄) */
  align-content: flex-start | flex-end | center | space-between | space-around | stretch;

  gap: 1rem;         /* 아이템 간격 */
  gap: 1rem 2rem;    /* row-gap column-gap */
}

.item {
  flex-grow: 1;      /* 남은 공간 비율로 차지 */
  flex-shrink: 1;    /* 공간 부족 시 줄어드는 비율 */
  flex-basis: auto;  /* 기본 크기 */
  flex: 1 1 auto;    /* 단축 속성 */

  align-self: auto | flex-start | flex-end | center | stretch;
  order: 0;          /* 순서 변경 */
}
```

### 11.2 실전 패턴

```css
/* 수평·수직 중앙 정렬 */
.center {
  display: flex;
  justify-content: center;
  align-items: center;
}

/* 네비게이션 바 */
.navbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
}

/* 카드 그리드 (반응형) */
.cards {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;
}
.card {
  flex: 1 1 280px; /* 최소 280px, 공간 있으면 늘어남 */
  max-width: 400px;
}

/* 사이드바 레이아웃 */
.layout {
  display: flex;
  gap: 2rem;
}
.sidebar { flex: 0 0 280px; } /* 고정 너비 */
.main    { flex: 1; }         /* 남은 공간 */
```

## 12. CSS Grid

### 12.1 핵심 속성 요약

```css
.grid {
  display: grid;

  /* 컬럼 정의 */
  grid-template-columns: 200px 1fr 200px;
  grid-template-columns: repeat(3, 1fr);
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));

  /* 행 정의 */
  grid-template-rows: auto 1fr auto;

  /* 템플릿 영역 */
  grid-template-areas:
    "header header header"
    "sidebar main   main"
    "footer footer footer";

  /* 간격 */
  gap: 1rem;
  row-gap: 1rem;
  column-gap: 2rem;

  /* 정렬 */
  justify-items: stretch | start | end | center;
  align-items: stretch | start | end | center;
}

.item {
  /* 셀 배치 */
  grid-column: 1 / 3;        /* 1번~3번 컬럼 */
  grid-column: span 2;       /* 2칸 차지 */
  grid-row: 1 / 3;

  /* 영역 이름으로 배치 */
  grid-area: header;

  /* 개별 정렬 */
  justify-self: start;
  align-self: center;
}
```

### 12.2 실전 패턴

```css
/* 페이지 전체 레이아웃 */
.page {
  display: grid;
  grid-template-areas:
    "header"
    "main"
    "footer";
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
}

@media (min-width: 768px) {
  .page {
    grid-template-areas:
      "header  header"
      "sidebar main"
      "footer  footer";
    grid-template-columns: 280px 1fr;
  }
}

header { grid-area: header; }
.sidebar { grid-area: sidebar; }
main   { grid-area: main; }
footer { grid-area: footer; }

/* 자동 반응형 카드 그리드 */
.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: 1.5rem;
}

/* Holy Grail 레이아웃 */
.holy-grail {
  display: grid;
  grid-template: auto 1fr auto / auto 1fr auto;
}
```

### 12.3 Flexbox vs Grid 선택 기준

| 상황 | 권장 |
|------|------|
| 1차원 레이아웃 (행 또는 열) | Flexbox |
| 2차원 레이아웃 (행과 열 동시) | Grid |
| 콘텐츠 크기 기준 배치 | Flexbox |
| 엄격한 격자 정렬 | Grid |
| 카드 목록, 네비게이션 | Flexbox |
| 전체 페이지 레이아웃 | Grid |

## 13. 반응형 이미지 & 미디어

### 13.1 기본 반응형 이미지

```css
img {
  max-width: 100%;
  height: auto;
```

### 13.2 srcset / sizes

```html
<!-- 해상도별 이미지 제공 -->
<img
  src="image-800.jpg"
  srcset="
    image-400.jpg  400w,
    image-800.jpg  800w,
    image-1200.jpg 1200w
  "
  sizes="
    (max-width: 600px) 100vw,
    (max-width: 1200px) 50vw,
    800px
  "
  alt="반응형 이미지"
/>
```

### 13.3 picture 요소 (아트 디렉션)

```html
<picture>
  <!-- 모바일: 정사각형 크롭 -->
  <source media="(max-width: 767px)" srcset="hero-mobile.jpg" />
  <!-- 태블릿 -->
  <source media="(max-width: 1023px)" srcset="hero-tablet.jpg" />
  <!-- 데스크톱: 와이드 크롭 -->
  <img src="hero-desktop.jpg" alt="히어로 이미지" />
</picture>

<!-- WebP + 폴백 -->
<picture>
  <source type="image/avif" srcset="image.avif" />
  <source type="image/webp" srcset="image.webp" />
  <img src="image.jpg" alt="최적화된 이미지" />
</picture>
```

### 13.4 반응형 비디오 / iframe

```css
  height: 100%;
```

## 14. 모바일 퍼스트 vs 데스크톱 퍼스트

### 14.1 모바일 퍼스트 (권장)

```css
/* 기본 스타일: 모바일 */
.container {
  width: 100%;
  padding: 1rem;
  flex-direction: column;
}

/* 태블릿 이상 */
@media (min-width: 768px) {
  .container {
    padding: 2rem;
    flex-direction: row;
  }
}

/* 데스크톱 이상 */
@media (min-width: 1024px) {
  .container {
    max-width: 1200px;
    margin: 0 auto;
  }
}
```

### 14.2 데스크톱 퍼스트

```css
/* 기본 스타일: 데스크톱 */
.container {
  max-width: 1200px;
  flex-direction: row;
}

/* 태블릿 이하 */
@media (max-width: 1023px) {
  .container { padding: 2rem; }
}

/* 모바일 */
@media (max-width: 767px) {
  .container {
    width: 100%;
    flex-direction: column;
  }
}
```

### 14.3 비교

| 항목 | 모바일 퍼스트 | 데스크톱 퍼스트 |
|------|--------------|----------------|
| 성능 | 모바일에서 불필요 CSS 로드 없음 | 모바일에서 오버라이드 필요 |
| 트렌드 | 현재 표준 | 레거시 |
| 구현 난이도 | 처음엔 어색 | 직관적 |
| SEO | Google 권장 | 불리 |

### 15.1 PWA 체크리스트

```
□ HTTPS 적용

```

### 15.2 반응형 체크리스트

```
□ <meta name="viewport"> 설정
□ 픽셀 단위 고정 너비 제거
□ 이미지 max-width: 100% 적용
□ 주요 브레이크포인트 (768px, 1024px) 대응
□ 터치 타겟 최소 44x44px 확보

□ 접근성(ARIA, 색상 대비) 확인
```
