## 1. 트리 (Tree)

트리는 **계층 구조**를 표현하는 비선형 자료구조다. 노드(Node)와 간선(Edge)으로 구성되며, 사이클이 없는 연결 그래프다.

### 핵심 용어

| 용어 | 설명 |
|---|---|
| 루트(Root) | 최상위 노드 |
| 리프(Leaf) | 자식이 없는 노드 |
| 깊이(Depth) | 루트에서 해당 노드까지의 거리 |
| 높이(Height) | 리프까지의 최장 경로 길이 |
| 차수(Degree) | 노드의 자식 수 |
| 서브트리 | 특정 노드를 루트로 하는 부분 트리 |

### 이진 트리 순회

```python
class Node:
    def __init__(self, val):
        self.val = val
        self.left = None
        self.right = None

def preorder(node):   # 전위: 루트 → 왼 → 오
    if not node: return
    print(node.val)
    preorder(node.left)
    preorder(node.right)

def inorder(node):    # 중위: 왼 → 루트 → 오 (BST에서 정렬 순서)
    if not node: return
    inorder(node.left)
    print(node.val)
    inorder(node.right)

def postorder(node):  # 후위: 왼 → 오 → 루트
    if not node: return
    postorder(node.left)
    postorder(node.right)
    print(node.val)

def level_order(root):  # 레벨 순회 (BFS)
    from collections import deque
    q = deque([root])
    while q:
        node = q.popleft()
        print(node.val)
        if node.left: q.append(node.left)
        if node.right: q.append(node.right)
```

## 2. 이진 탐색 트리 (BST)

**규칙:** 왼쪽 자식 < 부모 < 오른쪽 자식

| 연산 | 평균 | 최악 (편향 트리) |
|---|---|---|
| 탐색 | O(log n) | O(n) |
| 삽입 | O(log n) | O(n) |
| 삭제 | O(log n) | O(n) |

```python
class BST:
    def __init__(self):
        self.root = None

    def insert(self, val):
        def _insert(node, val):
            if not node:
                return Node(val)
            if val < node.val:
                node.left = _insert(node.left, val)
            elif val > node.val:
                node.right = _insert(node.right, val)
            return node
        self.root = _insert(self.root, val)

    def search(self, val):
        def _search(node, val):
            if not node or node.val == val:
                return node
            if val < node.val:
                return _search(node.left, val)
            return _search(node.right, val)
        return _search(self.root, val)

    def delete(self, val):
        def _min_node(node):
            while node.left:
                node = node.left
            return node

        def _delete(node, val):
            if not node:
                return None
            if val < node.val:
                node.left = _delete(node.left, val)
            elif val > node.val:
                node.right = _delete(node.right, val)
            else:
                # 자식이 하나이거나 없는 경우
                if not node.left: return node.right
                if not node.right: return node.left
                # 자식이 둘인 경우: 오른쪽 서브트리의 최솟값으로 대체
                successor = _min_node(node.right)
                node.val = successor.val
                node.right = _delete(node.right, successor.val)
            return node
        self.root = _delete(self.root, val)
```

> **편향 트리 주의:** 정렬된 데이터를 순서대로 삽입하면 O(n) 연결리스트가 된다. 이를 해결하는 게 균형 트리다.

## 3. 균형 이진 트리 — AVL & Red-Black

### AVL 트리

모든 노드에서 **좌우 서브트리 높이 차가 1 이하**임을 보장한다. 삽입/삭제 시 회전(Rotation)으로 균형을 유지한다.

```
균형 인수(Balance Factor) = 왼쪽 높이 - 오른쪽 높이
  -1, 0, 1 → 균형
  2 이상 또는 -2 이하 → 회전 필요
```

**4가지 회전 케이스**

| 케이스 | 해결 |
|---|---|
| LL (왼쪽-왼쪽) | 우회전 |
| RR (오른쪽-오른쪽) | 좌회전 |
| LR (왼쪽-오른쪽) | 좌회전 후 우회전 |
| RL (오른쪽-왼쪽) | 우회전 후 좌회전 |

### Red-Black 트리

각 노드에 **색상(Red/Black)** 을 부여해 균형을 유지한다. Java의 `TreeMap`, C++의 `std::map`이 내부적으로 사용한다.

**규칙:**
1. 모든 노드는 Red 또는 Black
2. 루트는 항상 Black
3. Red 노드의 자식은 반드시 Black (Red 연속 불가)
4. 모든 경로의 Black 노드 수가 동일

| | AVL | Red-Black |
|---|---|---|
| 균형 조건 | 엄격 (높이 차 ≤ 1) | 완화 (2배 이내) |
| 탐색 속도 | 빠름 | 약간 느림 |
| 삽입/삭제 | 회전 더 많음 | 회전 적음 |
| 사용 | 탐색 빈번 시 | 삽입/삭제 빈번 시 |

## 4. 힙 (Heap)

**완전 이진 트리** 형태로, 부모-자식 간 대소 관계가 일정하게 유지된다.

- **최대 힙(Max-Heap):** 부모 ≥ 자식 → 루트가 최댓값
- **최소 힙(Min-Heap):** 부모 ≤ 자식 → 루트가 최솟값

| 연산 | 시간 복잡도 |
|---|---|
| 삽입 | O(log n) |
| 최댓/최솟값 조회 | O(1) |
| 최댓/최솟값 삭제 | O(log n) |
| 힙 구성(heapify) | O(n) |

```python
import heapq

# 파이썬 heapq는 기본이 최소 힙
heap = []
heapq.heappush(heap, 3)
heapq.heappush(heap, 1)
heapq.heappush(heap, 2)

print(heapq.heappop(heap))  # 1 (최솟값)

# 최대 힙: 음수로 변환
max_heap = []
for val in [3, 1, 4, 1, 5]:
    heapq.heappush(max_heap, -val)

print(-heapq.heappop(max_heap))  # 5 (최댓값)

# 배열을 힙으로 변환 O(n)
data = [3, 1, 4, 1, 5, 9]
heapq.heapify(data)
```

### 힙 직접 구현

```python
class MinHeap:
    def __init__(self):
        self.heap = []

    def push(self, val):
        self.heap.append(val)
        self._sift_up(len(self.heap) - 1)

    def pop(self):
        if len(self.heap) == 1:
            return self.heap.pop()
        root = self.heap[0]
        self.heap[0] = self.heap.pop()
        self._sift_down(0)
        return root

    def _sift_up(self, i):
        parent = (i - 1) // 2
        if i > 0 and self.heap[i] < self.heap[parent]:
            self.heap[i], self.heap[parent] = self.heap[parent], self.heap[i]
            self._sift_up(parent)

    def _sift_down(self, i):
        n = len(self.heap)
        smallest = i
        left, right = 2 * i + 1, 2 * i + 2
        if left < n and self.heap[left] < self.heap[smallest]:
            smallest = left
        if right < n and self.heap[right] < self.heap[smallest]:
            smallest = right
        if smallest != i:
            self.heap[i], self.heap[smallest] = self.heap[smallest], self.heap[i]
            self._sift_down(smallest)
```

> **활용:** 우선순위 큐, 힙 정렬, 다익스트라 알고리즘, k번째 최솟값 문제

## 5. 그래프 (Graph)

노드(정점, Vertex)와 간선(Edge)으로 구성된 자료구조. 트리는 그래프의 특수한 형태다.

### 그래프 분류

| 구분 | 설명 |
|---|---|
| 방향 그래프 | 간선에 방향이 있음 (A → B) |
| 무방향 그래프 | 양방향 (A — B) |
| 가중치 그래프 | 간선에 비용이 있음 |
| 사이클 그래프 | 순환 경로 존재 |
| DAG | 방향이 있고 사이클 없는 그래프 |

### 표현 방식

**인접 행렬 (Adjacency Matrix)**

```python
# V개의 정점, 공간 O(V²)
V = 5
graph = [[0] * V for _ in range(V)]
graph[0][1] = 1  # 0 → 1 연결
graph[1][2] = 1  # 1 → 2 연결
```

**인접 리스트 (Adjacency List)**

```python
# 공간 O(V + E), 희소 그래프에 효율적
from collections import defaultdict

graph = defaultdict(list)
graph[0].append(1)
graph[1].append(2)
graph[0].append(3)

# 가중치 그래프
weighted = defaultdict(list)
weighted[0].append((1, 5))   # (도착 정점, 가중치)
weighted[0].append((3, 10))
```

| | 인접 행렬 | 인접 리스트 |
|---|---|---|
| 공간 | O(V²) | O(V + E) |
| 간선 확인 | O(1) | O(degree) |
| 모든 간선 순회 | O(V²) | O(V + E) |
| 적합한 경우 | 밀집 그래프 | 희소 그래프 |

## 6. 그래프 탐색 — BFS & DFS

### BFS (너비 우선 탐색)

큐를 사용해 **가까운 노드부터** 탐색. **최단 거리** 문제에 적합.

```python
from collections import deque

def bfs(graph, start):
    visited = set([start])
    queue = deque([start])
    result = []

    while queue:
        node = queue.popleft()
        result.append(node)
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
    return result

# 최단 거리 BFS
def bfs_distance(graph, start, end):
    visited = {start: 0}
    queue = deque([start])
    while queue:
        node = queue.popleft()
        if node == end:
            return visited[node]
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited[neighbor] = visited[node] + 1
                queue.append(neighbor)
    return -1  # 도달 불가
```

### DFS (깊이 우선 탐색)

스택(또는 재귀)으로 **깊이 방향으로** 탐색. 경로 탐색, 사이클 감지에 적합.

```python
def dfs_recursive(graph, node, visited=None):
    if visited is None:
        visited = set()
    visited.add(node)
    print(node)
    for neighbor in graph[node]:
        if neighbor not in visited:
            dfs_recursive(graph, neighbor, visited)

def dfs_iterative(graph, start):
    visited = set()
    stack = [start]
    result = []
    while stack:
        node = stack.pop()
        if node not in visited:
            visited.add(node)
            result.append(node)
            for neighbor in reversed(graph[node]):
                if neighbor not in visited:
                    stack.append(neighbor)
    return result
```

| | BFS | DFS |
|---|---|---|
| 자료구조 | 큐 | 스택(재귀) |
| 시간 복잡도 | O(V + E) | O(V + E) |
| 공간 복잡도 | O(V) | O(V) |
| 최단 경로 | 보장 (무가중치) | 보장 안 됨 |
| 활용 | 최단 거리, 레벨 순회 | 위상 정렬, 사이클 감지, 경로 탐색 |

## 7. 최단 경로 알고리즘

### 다익스트라 (Dijkstra) — 단일 출발점

**음수 가중치 없는** 그래프에서 단일 출발점 최단 경로. 우선순위 큐 사용.

- 시간 복잡도: **O((V + E) log V)**

```python
import heapq

def dijkstra(graph, start, V):
    dist = [float('inf')] * V
    dist[start] = 0
    pq = [(0, start)]  # (거리, 노드)

    while pq:
        cost, node = heapq.heappop(pq)
        if cost > dist[node]:  # 이미 처리된 노드 스킵
            continue
        for neighbor, weight in graph[node]:
            new_cost = dist[node] + weight
            if new_cost < dist[neighbor]:
                dist[neighbor] = new_cost
                heapq.heappush(pq, (new_cost, neighbor))
    return dist

# 사용 예시
V = 5
graph = [[] for _ in range(V)]
graph[0].append((1, 4))  # 0→1, 비용 4
graph[0].append((2, 1))  # 0→2, 비용 1
graph[2].append((1, 2))  # 2→1, 비용 2
graph[1].append((3, 1))  # 1→3, 비용 1

print(dijkstra(graph, 0, V))  # [0, 3, 1, 4, inf]
```

### 벨만-포드 (Bellman-Ford) — 음수 가중치 허용

음수 가중치 간선이 있을 때 사용. **음수 사이클 감지** 가능.

- 시간 복잡도: **O(V × E)**

```python
def bellman_ford(edges, V, start):
    dist = [float('inf')] * V
    dist[start] = 0

    for _ in range(V - 1):
        for u, v, w in edges:
            if dist[u] != float('inf') and dist[u] + w < dist[v]:
                dist[v] = dist[u] + w

    # 음수 사이클 감지
    for u, v, w in edges:
        if dist[u] != float('inf') and dist[u] + w < dist[v]:
            return None  # 음수 사이클 존재
    return dist
```

### 플로이드-워셜 (Floyd-Warshall) — 모든 쌍 최단 경로

- 시간 복잡도: **O(V³)**

```python
def floyd_warshall(V, edges):
    INF = float('inf')
    dist = [[INF] * V for _ in range(V)]
    for i in range(V):
        dist[i][i] = 0
    for u, v, w in edges:
        dist[u][v] = w

    for k in range(V):        # 경유 정점
        for i in range(V):    # 출발
            for j in range(V):  # 도착
                if dist[i][k] + dist[k][j] < dist[i][j]:
                    dist[i][j] = dist[i][k] + dist[k][j]
    return dist
```

| | 다익스트라 | 벨만-포드 | 플로이드-워셜 |
|---|---|---|---|
| 음수 가중치 | 불가 | 가능 | 가능 |
| 음수 사이클 감지 | 불가 | 가능 | 가능 |
| 출발점 | 단일 | 단일 | 모든 쌍 |
| 시간 복잡도 | O((V+E)logV) | O(VE) | O(V³) |

## 8. 해시 테이블 (Hash Table)

**키(Key)를 해시 함수로 변환**해 배열 인덱스로 매핑. 평균 O(1) 탐색/삽입/삭제.

### 충돌 해결 방법

**1. 체이닝 (Chaining)** — 같은 버킷을 연결 리스트로 연결

```python
class HashTable:
    def __init__(self, size=16):
        self.size = size
        self.table = [[] for _ in range(size)]

    def _hash(self, key):
        return hash(key) % self.size

    def set(self, key, value):
        idx = self._hash(key)
        for i, (k, v) in enumerate(self.table[idx]):
            if k == key:
                self.table[idx][i] = (key, value)
                return
        self.table[idx].append((key, value))

    def get(self, key):
        idx = self._hash(key)
        for k, v in self.table[idx]:
            if k == key:
                return v
        return None

    def delete(self, key):
        idx = self._hash(key)
        self.table[idx] = [(k, v) for k, v in self.table[idx] if k != key]
```

**2. 오픈 어드레싱 (Open Addressing)** — 충돌 시 다른 빈 슬롯 탐색

- **선형 탐사:** 다음 슬롯 순서대로 탐색 → 군집화(Clustering) 문제
- **이중 해싱:** 두 번째 해시 함수로 탐사 간격 결정 → 군집화 최소화

### 성능 지표: 적재율 (Load Factor)

```
적재율 = 저장된 항목 수 / 버킷 수
```

- 적재율이 0.7~0.75 초과 시 **리해싱(Rehashing)** — 테이블 크기 2배 확장
- Python `dict`는 적재율 2/3 초과 시 자동 리해싱

## 9. 트라이 (Trie)

**문자열 검색에 특화된 트리**. 각 노드가 문자 하나를 표현하며, 공통 접두사를 공유한다.

| 연산 | 시간 복잡도 |
|---|---|
| 삽입 | O(L) — 문자열 길이 |
| 탐색 | O(L) |
| 접두사 검색 | O(L) |

```python
class TrieNode:
    def __init__(self):
        self.children = {}
        self.is_end = False

class Trie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, word: str):
        node = self.root
        for ch in word:
            if ch not in node.children:
                node.children[ch] = TrieNode()
            node = node.children[ch]
        node.is_end = True

    def search(self, word: str) -> bool:
        node = self.root
        for ch in word:
            if ch not in node.children:
                return False
            node = node.children[ch]
        return node.is_end

    def starts_with(self, prefix: str) -> bool:
        node = self.root
        for ch in prefix:
            if ch not in node.children:
                return False
            node = node.children[ch]
        return True

    def auto_complete(self, prefix: str) -> list:
        """접두사로 시작하는 모든 단어 반환"""
        node = self.root
        for ch in prefix:
            if ch not in node.children:
                return []
            node = node.children[ch]

        result = []
        def dfs(node, path):
            if node.is_end:
                result.append(prefix + path)
            for ch, child in node.children.items():
                dfs(child, path + ch)
        dfs(node, "")
        return result

# 사용 예시
trie = Trie()
for word in ["apple", "app", "application", "apply", "banana"]:
    trie.insert(word)

print(trie.search("app"))          # True
print(trie.starts_with("appl"))    # True
print(trie.auto_complete("app"))   # ['app', 'apple', 'application', 'apply']
```

> **활용:** 자동완성, 맞춤법 검사, IP 라우팅, 사전 구현

## 10. 세그먼트 트리 (Segment Tree)

**구간 쿼리(Range Query)를 O(log n)으로 처리**하는 트리. 구간 합, 구간 최솟/최댓값 등에 사용.

- 배열의 각 원소를 리프 노드로, 내부 노드는 구간의 합(또는 최솟값 등)을 저장

```python
class SegmentTree:
    def __init__(self, data):
        self.n = len(data)
        self.tree = [0] * (4 * self.n)
        self._build(data, 1, 0, self.n - 1)

    def _build(self, data, node, start, end):
        if start == end:
            self.tree[node] = data[start]
        else:
            mid = (start + end) // 2
            self._build(data, 2 * node, start, mid)
            self._build(data, 2 * node + 1, mid + 1, end)
            self.tree[node] = self.tree[2 * node] + self.tree[2 * node + 1]

    def query(self, node, start, end, l, r):
        """[l, r] 구간의 합"""
        if r < start or end < l:
            return 0
        if l <= start and end <= r:
            return self.tree[node]
        mid = (start + end) // 2
        left = self.query(2 * node, start, mid, l, r)
        right = self.query(2 * node + 1, mid + 1, end, l, r)
        return left + right

    def update(self, node, start, end, idx, val):
        """idx번 원소를 val로 갱신"""
        if start == end:
            self.tree[node] = val
        else:
            mid = (start + end) // 2
            if idx <= mid:
                self.update(2 * node, start, mid, idx, val)
            else:
                self.update(2 * node + 1, mid + 1, end, idx, val)
            self.tree[node] = self.tree[2 * node] + self.tree[2 * node + 1]

    def range_sum(self, l, r):
        return self.query(1, 0, self.n - 1, l, r)

    def point_update(self, idx, val):
        self.update(1, 0, self.n - 1, idx, val)

# 사용 예시
data = [1, 3, 5, 7, 9, 11]
st = SegmentTree(data)

print(st.range_sum(1, 3))   # 15 (3+5+7)
st.point_update(1, 10)       # index 1을 10으로 변경
print(st.range_sum(1, 3))   # 22 (10+5+7)
```

| 연산 | 단순 배열 | 세그먼트 트리 |
|---|---|---|
| 구간 쿼리 | O(n) | O(log n) |
| 원소 갱신 | O(1) | O(log n) |
| 공간 | O(n) | O(4n) |

## 11. 유니온-파인드 (Union-Find)

**서로소 집합(Disjoint Set)** 자료구조. 여러 원소가 같은 집합인지 빠르게 판별.

- **Find:** 원소가 속한 집합의 루트 반환
- **Union:** 두 집합을 하나로 합침

```python
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n  # 트리 높이

    def find(self, x):
        # 경로 압축 (Path Compression)
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, x, y):
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return False  # 이미 같은 집합

        # 랭크 기반 합치기 (Union by Rank)
        if self.rank[rx] < self.rank[ry]:
            rx, ry = ry, rx
        self.parent[ry] = rx
        if self.rank[rx] == self.rank[ry]:
            self.rank[rx] += 1
        return True

    def connected(self, x, y):
        return self.find(x) == self.find(y)

# 사용 예시 — 사이클 감지
def has_cycle(edges, V):
    uf = UnionFind(V)
    for u, v in edges:
        if not uf.union(u, v):
            return True  # 이미 같은 집합이면 사이클
    return False

# 크루스칼 알고리즘 — 최소 신장 트리
def kruskal(V, edges):
    edges.sort(key=lambda x: x[2])  # 가중치 기준 정렬
    uf = UnionFind(V)
    mst_cost = 0
    mst_edges = []

    for u, v, w in edges:
        if uf.union(u, v):
            mst_cost += w
            mst_edges.append((u, v, w))
    return mst_cost, mst_edges
```

**시간 복잡도:** 경로 압축 + 랭크 합치기 적용 시 거의 **O(1)** (정확히는 O(α(n)), α는 역 아커만 함수)

## 12. 자료구조 선택 가이드

```
문제 유형별 자료구조 선택 흐름

탐색이 핵심?
  ├── 키-값 매핑 → 해시 테이블 O(1)
  ├── 정렬된 탐색 → BST / 균형 트리 O(log n)
  └── 문자열 접두사 → 트라이 O(L)

순서/우선순위가 필요?
  ├── 최솟/최댓값 반복 추출 → 힙(우선순위 큐) O(log n)
  └── FIFO → 큐, LIFO → 스택

관계/연결이 핵심?
  ├── 계층 구조 → 트리
  ├── 네트워크/경로 → 그래프
  └── 집합 판별/합치기 → 유니온-파인드

구간 쿼리가 많음?
  └── 세그먼트 트리 O(log n)
```

| 자료구조 | 핵심 연산 | 대표 문제 유형 |
|---|---|---|
| BST / AVL | 탐색·삽입·삭제 O(log n) | 정렬 유지, 범위 조회 |
| 힙 | 최솟/최댓값 O(log n) | 우선순위, k번째 원소 |
| 해시 테이블 | 탐색·삽입 O(1) 평균 | 빠른 조회, 중복 제거 |
| 트라이 | 문자열 탐색 O(L) | 자동완성, 사전 |
| 세그먼트 트리 | 구간 쿼리 O(log n) | 구간 합, 구간 최솟값 |
| 유니온-파인드 | 집합 연산 O(α(n)) | 사이클 감지, MST |
| BFS | 최단 거리 O(V+E) | 미로, 최소 이동 횟수 |
| 다익스트라 | 최단 경로 O((V+E)logV) | 가중치 최단 경로 |

---

> **핵심 요약**
> - **트리/BST:** 계층 구조, 정렬된 탐색
> - **힙:** 우선순위 큐, 반복적 최솟/최댓값 추출
> - **그래프 + BFS/DFS:** 연결 관계, 경로 탐색
> - **해시:** O(1) 탐색이 필요할 때
> - **트라이:** 문자열 접두사 검색
> - **세그먼트 트리:** 빈번한 구간 쿼리 + 갱신
> - **유니온-파인드:** 집합 분류, 사이클 감지, MST
