# Zustand 정리 (v5 최신 기준)

**Zustand**는 독일어로 "상태(State)"를 뜻하며, React 생태계에서 가장 가볍고 직관적인 **전역 상태 관리 라이브러리**입니다.
Redux의 보일러플레이트 없이, Context API의 리렌더링 문제 없이, 단 몇 줄로 전역 상태를 구성할 수 있습니다.

> 목표: Zustand의 **동작 원리를 이해**하고, **v5 최신 패턴**으로 실전 프로젝트에 바로 적용하기

## 1. Zustand란?

### 1.1 핵심 특징

| 특징 | 설명 |
|------|------|
| **초경량** | 번들 사이즈 ~1.2KB (gzip 기준) |
| **보일러플레이트 없음** | Action, Reducer, Provider 불필요 |
| **선택적 리렌더링** | 구독한 상태가 바뀔 때만 리렌더 |
| **React 외부 사용 가능** | 컴포넌트 밖에서도 상태 접근/변경 |
| **미들웨어 확장** | persist, immer, devtools 등 조합 가능 |

### 1.2 상태 관리 라이브러리 선택 기준

```
클라이언트 상태 (UI, 로컬 인터랙션)
├── 단순/소규모  → useState / useReducer
├── 중간 규모    → Zustand ✅
└── 대규모/팀   → Zustand + 슬라이스 패턴 ✅

서버 상태 (API 데이터)
└── TanStack Query / SWR
```

### 1.3 Zustand의 동작 원리

```
┌─────────────────────────────────────┐
│             Store                   │
│  state: { count: 0, user: null }    │
│  actions: { increment, setUser }    │
└──────────────┬──────────────────────┘
               │ 구독 (selector)
       ┌───────┴────────┐
       ▼                ▼
  <Counter />       <Header />
  count만 구독      user만 구독
  (count 변경 시    (user 변경 시
   만 리렌더)        만 리렌더)
```

Zustand는 내부적으로 **pub/sub 패턴**을 사용합니다. 컴포넌트가 특정 상태를 구독하면, 해당 상태가 변경될 때만 리렌더링이 발생합니다.

## 2. 설치 및 기본 사용법

```bash
npm install zustand
```

### 2.1 스토어 생성

```ts
// store/counterStore.ts
import { create } from 'zustand'

interface CounterStore {
  count: number
  increment: () => void
  decrement: () => void
  reset: () => void
}

export const useCounterStore = create<CounterStore>((set) => ({
  count: 0,
  increment: () => set((state) => ({ count: state.count + 1 })),
  decrement: () => set((state) => ({ count: state.count - 1 })),
  reset: () => set({ count: 0 }),
}))
```

### 2.2 컴포넌트에서 사용

```tsx
// Provider 없이 바로 사용 가능!
function Counter() {
  const count = useCounterStore((state) => state.count)
  const { increment, decrement, reset } = useCounterStore()

  return (
    <div>
      <p>{count}</p>
      <button onClick={increment}>+</button>
      <button onClick={decrement}>-</button>
      <button onClick={reset}>초기화</button>
    </div>
  )
}
```

### 2.3 set 함수 상세

```ts
const useStore = create<Store>((set, get) => ({
  count: 0,
  name: 'zustand',

  // 상태 일부 업데이트 (자동 병합)
  updateCount: () => set({ count: 1 }),

  // 이전 상태 기반 업데이트
  increment: () => set((state) => ({ count: state.count + 1 })),

  // 현재 상태 읽기 (get)
  double: () => set({ count: get().count * 2 }),

  // 상태 완전 교체 (replace: true)
  reset: () => set({ count: 0, name: 'zustand' }, true),
}))
```

## 3. TypeScript와 함께 사용하기

### 3.1 기본 타입 정의

```ts
import { create } from 'zustand'

// 상태와 액션을 분리해서 타입 정의하는 것이 권장 패턴
type State = {
  count: number
  user: User | null
}

type Actions = {
  increment: () => void
  setUser: (user: User) => void
  reset: () => void
}

type Store = State & Actions

export const useStore = create<Store>((set) => ({
  // State 초기값
  count: 0,
  user: null,
  // Actions
  increment: () => set((s) => ({ count: s.count + 1 })),
  setUser: (user) => set({ user }),
  reset: () => set({ count: 0, user: null }),
}))
```

### 3.2 createStore + useStore 패턴 (v5 권장)

v5에서는 `createStore`로 바닐라 스토어를 만들고 `useStore`로 React에 연결하는 패턴을 권장합니다.

```ts
import { createStore, useStore } from 'zustand'

// 바닐라 스토어 생성 (React 외부에서도 사용 가능)
const counterStore = createStore<Store>((set) => ({
  count: 0,
  increment: () => set((s) => ({ count: s.count + 1 })),
}))

// React 컴포넌트용 훅
function useCounterStore<T>(selector: (state: Store) => T) {
  return useStore(counterStore, selector)
}

// 컴포넌트 외부에서도 사용 가능
counterStore.getState().increment()
counterStore.setState({ count: 100 })
const state = counterStore.getState()
```

## 4. 선택적 구독 (Selector)

불필요한 리렌더링을 막는 Zustand의 핵심 기능입니다.

### 4.1 기본 selector

```tsx
const useStore = create<Store>((set) => ({
  count: 0,
  user: { name: 'Kim', age: 30 },
  theme: 'dark',
}))

// ✅ count만 구독 → user, theme 변경 시 리렌더 없음
const count = useStore((state) => state.count)

// ✅ user.name만 구독
const name = useStore((state) => state.user.name)

// ❌ 스토어 전체 구독 → 어떤 상태든 변하면 리렌더
const store = useStore()
```

### 4.2 여러 상태를 한 번에 구독

```tsx
import { useShallow } from 'zustand/react/shallow'

// ✅ useShallow: 객체/배열 반환 시 얕은 비교로 불필요한 리렌더 방지
const { count, user } = useStore(
  useShallow((state) => ({ count: state.count, user: state.user }))
)

// 배열도 동일
const [count, user] = useStore(
  useShallow((state) => [state.count, state.user])
)
```

### 4.3 computed value (파생 상태)

```tsx
// 스토어 외부에서 파생 상태 계산
const totalPrice = useCartStore(
  (state) => state.items.reduce((sum, item) => sum + item.price * item.qty, 0)
)

// 또는 스토어 내부에 getter 정의
const useCartStore = create<CartStore>((set, get) => ({
  items: [],
  get totalPrice() {
    return get().items.reduce((sum, item) => sum + item.price * item.qty, 0)
  },
}))
```

## 5. 비동기 액션

Zustand는 비동기 처리를 위한 별도 문법이 없습니다. 액션 안에서 async/await를 그대로 사용합니다.

```ts
type UserStore = {
  user: User | null
  isLoading: boolean
  error: string | null
  fetchUser: (id: number) => Promise<void>
}

export const useUserStore = create<UserStore>((set) => ({
  user: null,
  isLoading: false,
  error: null,

  fetchUser: async (id) => {
    set({ isLoading: true, error: null })
    try {
      const res = await fetch(`/api/users/${id}`)
      if (!res.ok) throw new Error('유저 조회 실패')
      const user = await res.json()
      set({ user, isLoading: false })
    } catch (e) {
      set({ error: (e as Error).message, isLoading: false })
    }
  },
}))
```

```tsx
function UserProfile({ id }: { id: number }) {
  const { user, isLoading, error, fetchUser } = useUserStore()

  useEffect(() => {
    fetchUser(id)
  }, [id])

  if (isLoading) return <Spinner />
  if (error) return <p>{error}</p>
  return <div>{user?.name}</div>
}
```

> **참고**: 서버 상태(API 데이터)가 복잡해지면 TanStack Query와 함께 사용하는 것이 좋습니다.
> Zustand → 클라이언트 상태 / TanStack Query → 서버 상태로 역할을 분리하세요.

## 6. 슬라이스 패턴 (Slice Pattern)

스토어가 커지면 기능별로 분리해서 관리합니다. Redux Toolkit의 slice 개념과 유사합니다.

### 6.1 슬라이스 분리

```ts
// store/slices/counterSlice.ts
import { StateCreator } from 'zustand'
import { RootStore } from '../useRootStore'

export type CounterSlice = {
  count: number
  increment: () => void
  decrement: () => void
}

export const createCounterSlice: StateCreator<
  RootStore,
  [],
  [],
  CounterSlice
> = (set) => ({
  count: 0,
  increment: () => set((s) => ({ count: s.count + 1 })),
  decrement: () => set((s) => ({ count: s.count - 1 })),
})
```

```ts
// store/slices/userSlice.ts
import { StateCreator } from 'zustand'
import { RootStore } from '../useRootStore'

export type UserSlice = {
  user: User | null
  setUser: (user: User) => void
  logout: () => void
}

export const createUserSlice: StateCreator<
  RootStore,
  [],
  [],
  UserSlice
> = (set) => ({
  user: null,
  setUser: (user) => set({ user }),
  logout: () => set({ user: null }),
})
```

### 6.2 슬라이스 합치기

```ts
// store/useRootStore.ts
import { create } from 'zustand'
import { devtools, persist } from 'zustand/middleware'
import { CounterSlice, createCounterSlice } from './slices/counterSlice'
import { UserSlice, createUserSlice } from './slices/userSlice'

export type RootStore = CounterSlice & UserSlice

export const useRootStore = create<RootStore>()(
  devtools(
    persist(
      (...args) => ({
        ...createCounterSlice(...args),
        ...createUserSlice(...args),
      }),
      { name: 'root-store' }
    )
  )
)
```

```tsx
// 슬라이스별로 필요한 것만 구독
const count = useRootStore((s) => s.count)
const user = useRootStore((s) => s.user)
```

## 8. persist - 상태 영구 저장

새로고침 후에도 상태를 유지합니다. 기본 저장소는 `localStorage`입니다.

```ts
import { create } from 'zustand'
import { persist, createJSONStorage } from 'zustand/middleware'

type ThemeStore = {
  theme: 'light' | 'dark'
  toggleTheme: () => void
}

export const useThemeStore = create<ThemeStore>()(
  persist(
    (set) => ({
      theme: 'light',
      toggleTheme: () =>
        set((s) => ({ theme: s.theme === 'light' ? 'dark' : 'light' })),
    }),
    {
      name: 'theme-storage',                        // localStorage key
      storage: createJSONStorage(() => localStorage), // 기본값
    }
  )
)
```

### 8.1 persist 고급 옵션

```ts
persist(
  (set) => ({ ... }),
  {
    name: 'my-store',

    // sessionStorage 사용
    storage: createJSONStorage(() => sessionStorage),

    // 일부 상태만 저장 (partialize)
    partialize: (state) => ({ theme: state.theme, user: state.user }),

    // 버전 관리 및 마이그레이션
    version: 2,
    migrate: (persistedState: unknown, version: number) => {
      if (version === 1) {
        // 이전 버전 데이터 변환
        return { ...persistedState as Store, newField: 'default' }
      }
      return persistedState as Store
    },

    // 저장 전/후 훅
    onRehydrateStorage: () => (state) => {
      console.log('hydration finished:', state)
    },
  }
)
```

### 8.2 persist hydration 처리

```tsx
// SSR/Next.js 환경에서 hydration mismatch 방지
function ThemeToggle() {
  const [hydrated, setHydrated] = useState(false)
  const theme = useThemeStore((s) => s.theme)

  useEffect(() => {
    // persist hydration 완료 후 렌더
    const unsub = useThemeStore.persist.onFinishHydration(() => {
      setHydrated(true)
    })
    setHydrated(useThemeStore.persist.hasHydrated())
    return unsub
  }, [])

  if (!hydrated) return null
  return <button>{theme === 'dark' ? '🌙' : '☀️'}</button>
}
```

## 9. immer - 불변성 자동 관리

중첩된 객체 상태를 직접 변경하는 것처럼 작성할 수 있습니다.

```bash
npm install immer
```

```ts
import { create } from 'zustand'
import { immer } from 'zustand/middleware/immer'

type TodoStore = {
  todos: { id: number; text: string; done: boolean }[]
  addTodo: (text: string) => void
  toggleTodo: (id: number) => void
  removeTodo: (id: number) => void
}

export const useTodoStore = create<TodoStore>()(
  immer((set) => ({
    todos: [],

    addTodo: (text) =>
      set((state) => {
        // immer 덕분에 push 직접 사용 가능 (불변성 자동 처리)
        state.todos.push({ id: Date.now(), text, done: false })
      }),

    toggleTodo: (id) =>
      set((state) => {
        const todo = state.todos.find((t) => t.id === id)
        if (todo) todo.done = !todo.done // 직접 변경 OK
      }),

    removeTodo: (id) =>
      set((state) => {
        state.todos = state.todos.filter((t) => t.id !== id)
      }),
  }))
)
```

**immer 없이 동일한 코드**:
```ts
// ❌ 불변성 직접 관리 (번거로움)
toggleTodo: (id) =>
  set((state) => ({
    todos: state.todos.map((t) =>
      t.id === id ? { ...t, done: !t.done } : t
    ),
  })),
```

## 10. devtools - Redux DevTools 연동

브라우저의 Redux DevTools 확장에서 Zustand 상태를 확인할 수 있습니다.

```ts
import { create } from 'zustand'
import { devtools } from 'zustand/middleware'

export const useStore = create<Store>()(
  devtools(
    (set) => ({
      count: 0,
      increment: () => set(
        (s) => ({ count: s.count + 1 }),
        false,           // replace 여부
        'counter/increment'  // DevTools에 표시될 액션 이름
      ),
    }),
    {
      name: 'MyAppStore',  // DevTools에 표시될 스토어 이름
      enabled: process.env.NODE_ENV === 'development',
    }
  )
)
```

DevTools에서 액션 이름을 명시하면 타임라인 추적이 훨씬 편해집니다.

```ts
// 액션 이름 지정 패턴
set((s) => ({ count: s.count + 1 }), false, 'counter/increment')
set({ user: null }, false, 'user/logout')
set({ theme: 'dark' }, false, 'theme/setDark')
```

## 11. v5 주요 변경사항

Zustand v5는 2024년 출시되었으며 다음이 주요하게 변경되었습니다.

### 11.1 `useShallow` 위치 변경

```ts
// v4 (구버전)
import { shallow } from 'zustand/shallow'
const { a, b } = useStore(shallow((s) => ({ a: s.a, b: s.b })))

// v5 (최신) ✅
import { useShallow } from 'zustand/react/shallow'
const { a, b } = useStore(useShallow((s) => ({ a: s.a, b: s.b })))
```

### 11.2 `createStore` / `useStore` 분리

```ts
// v5 권장 패턴: 바닐라 스토어와 React 훅 분리
import { createStore } from 'zustand/vanilla'
import { useStore } from 'zustand'

const store = createStore<Store>()((set) => ({
  count: 0,
  increment: () => set((s) => ({ count: s.count + 1 })),
}))

// React 외부에서
store.getState().increment()

// React 컴포넌트에서
const count = useStore(store, (s) => s.count)
```

### 11.3 Context와 함께 사용 (다중 인스턴스)

같은 스토어를 여러 인스턴스로 사용할 때 (테스트, 다중 위젯 등) Context 패턴을 권장합니다.

```tsx
import { createContext, useContext, useRef } from 'react'
import { createStore, useStore } from 'zustand'

type CounterStore = { count: number; increment: () => void }

const CounterContext = createContext<ReturnType<typeof createStore<CounterStore>> | null>(null)

export function CounterProvider({ children }: { children: React.ReactNode }) {
  const storeRef = useRef<ReturnType<typeof createStore<CounterStore>>>()

  if (!storeRef.current) {
    storeRef.current = createStore<CounterStore>()((set) => ({
      count: 0,
      increment: () => set((s) => ({ count: s.count + 1 })),
    }))
  }

  return (
    <CounterContext.Provider value={storeRef.current}>
      {children}
    </CounterContext.Provider>
  )
}

export function useCounterStore<T>(selector: (state: CounterStore) => T) {
  const store = useContext(CounterContext)
  if (!store) throw new Error('CounterProvider 안에서 사용해야 합니다')
  return useStore(store, selector)
}
```

## 12. Zustand vs 다른 라이브러리 비교

| 항목 | Zustand | Redux Toolkit | Jotai | Recoil |
|------|---------|---------------|-------|--------|
| **번들 사이즈** | ~1.2KB | ~50KB | ~8KB | ~21KB |
| **보일러플레이트** | 최소 | 보통 | 최소 | 보통 |
| **학습 곡선** | 낮음 | 높음 | 낮음 | 보통 |
| **DevTools** | ✅ Redux DevTools | ✅ 전용 DevTools | ✅ | ✅ |
| **SSR 지원** | ✅ | ✅ | ✅ | ⚠️ 제한적 |
| **React 외부 사용** | ✅ | ✅ | ❌ | ❌ |
| **비동기 처리** | 직접 작성 | createAsyncThunk | 내장 | Selector |
| **적합한 규모** | 소~대 | 대규모 팀 | 소~중 | 소~중 |

### 언제 Zustand를 선택하나?

```
✅ Zustand 선택 상황
- 빠른 설정, 적은 코드량 원할 때
- 소~중규모 프로젝트
- React 외부에서 상태 접근이 필요할 때
- Redux의 복잡성은 피하고 싶을 때

✅ RTK (Redux Toolkit) 선택 상황
- 대규모 팀, 엄격한 패턴 강제가 필요할 때
- RTK Query로 서버 상태도 통합 관리할 때
- 이미 Redux 사용 중인 프로젝트 유지보수
```

## 13. 실전 Best Practices

### ✅ 스토어 파일 구조

```
src/
└── store/
    ├── useRootStore.ts          # 슬라이스 합성 (대규모)
    ├── useAuthStore.ts          # 인증 상태
    ├── useCartStore.ts          # 장바구니
    └── slices/
        ├── authSlice.ts
        └── cartSlice.ts
```

### ✅ 스토어 내 액션만 상태 변경

```ts
// ❌ 컴포넌트 내에서 직접 상태 조작
useStore.setState({ count: 10 })

// ✅ 항상 스토어에 정의된 액션을 통해 변경
const increment = useStore((s) => s.increment)
increment()
```

### ✅ selector 메모이제이션

```tsx
// ❌ 매 렌더마다 새 함수 생성 → 리렌더 무한 반복 위험
const data = useStore((s) => ({ a: s.a, b: s.b }))  // 객체 반환 시 주의

// ✅ useShallow로 얕은 비교 사용
const data = useStore(useShallow((s) => ({ a: s.a, b: s.b })))

// ✅ 또는 원시값은 그냥 구독
const a = useStore((s) => s.a)
const b = useStore((s) => s.b)
```

### ✅ 로그아웃 시 상태 초기화

```ts
// 공통 reset 액션 패턴
const initialState = { count: 0, user: null, theme: 'light' }

const useStore = create<Store & { resetAll: () => void }>()((set) => ({
  ...initialState,
  resetAll: () => set(initialState, true),  // replace: true로 완전 초기화
}))

// 로그아웃 핸들러
const handleLogout = () => {
  useStore.getState().resetAll()
}
```

### ✅ React 외부에서 상태 접근

```ts
// 컴포넌트 밖 (유틸, 인터셉터 등)에서 사용
import { useAuthStore } from '@/store/useAuthStore'

// axios 인터셉터에서 토큰 가져오기
axiosInstance.interceptors.request.use((config) => {
  const token = useAuthStore.getState().token
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})
```

### ✅ 구독 취소 (컴포넌트 외부 구독)

```ts
// 컴포넌트 밖에서 구독할 때는 반드시 구독 취소
const unsubscribe = useStore.subscribe(
  (state) => state.count,
  (count) => console.log('count changed:', count)
)

// 정리
unsubscribe()
```

## 정리

| 개념 | 핵심 요약 |
|------|-----------|
| `create` | 스토어 생성 (상태 + 액션 포함) |
| `set` | 상태 업데이트 (자동 병합) |
| `get` | 액션 내에서 현재 상태 읽기 |
| selector | 필요한 상태만 구독 → 최소 리렌더 |
| `useShallow` | 객체/배열 반환 시 얕은 비교로 최적화 |
| `persist` | localStorage 등에 상태 자동 저장 |
| `immer` | 중첩 상태 불변성 자동 처리 |
| `devtools` | Redux DevTools 연동 |
| 슬라이스 패턴 | 기능별 스토어 분리 후 합성 |
| `createStore` | React 외부에서도 사용 가능한 바닐라 스토어 |

> Zustand의 가장 큰 강점은 **단순함**입니다.
> `create` 하나로 시작해서 필요에 따라 `persist`, `immer`, `devtools`를 조합하고,
> 규모가 커지면 **슬라이스 패턴**으로 구조화하면 어떤 프로젝트에도 대응할 수 있습니다.
