## 1) 기본 규칙(경로/권한/파이프)

### 1.1 경로

- 절대경로: `/home/user/project`
- 상대경로: `./src`, `../..`
- 홈: `~` (예: `~/Downloads`)

```bash
pwd                 # 현재 경로
cd /var/log          # 이동
cd ..                # 상위 폴더
cd -                 # 이전 위치로
```

### 1.2 도움말/매뉴얼

```bash
man ls               # 매뉴얼
ls --help            # 간단 도움말
which python         # 명령의 실제 위치
command -v git       # which보다 안전하게 쓰는 경우도 많음
```

### 1.3 리다이렉션/파이프

- `>`: 덮어쓰기, `>>`: 이어쓰기
- `2>`: 에러(stderr) 리다이렉션
- `|`: 앞 출력(stdout)을 뒤 입력으로

```bash
echo "hello" > a.txt
cat a.txt | wc -l

# stdout + stderr 모두 파일로
some_command > out.log 2>&1
```

## 2) 파일/디렉터리 다루기

### 2.1 목록/이동/생성

```bash
ls                   # 목록
ls -al               # 숨김 포함 + 자세히
mkdir test           # 폴더 생성
mkdir -p a/b/c       # 중첩 폴더 한번에
```

### 2.2 파일 생성/복사/이동/삭제

```bash
touch note.txt       # 빈 파일 생성(또는 타임스탬프 갱신)
cp a.txt b.txt       # 복사
cp -r src dist       # 디렉터리 복사
mv a.txt docs/       # 이동(=이름 변경)
rm a.txt             # 삭제
rm -r dist           # 폴더 삭제
rm -rf dist          # 강제(매우 주의)
```

> `rm -rf`는 “정말 확실할 때만”.  
> 습관적으로 `rm -rf` 치기 전에 `pwd` / `ls`로 한 번 확인하는 버릇 추천.

### 2.3 파일/폴더 찾기

```bash
find . -name "*.log"              # 이름으로 찾기
find . -type f -maxdepth 2         # 2단계까지만 파일
find . -type f -mtime -1           # 1일 이내 변경 파일

# 실행 결과를 바로 삭제(조심!)
# find . -name "*.tmp" -delete
```

## 3) 파일 내용 보기/검색/편집

### 3.1 보기: cat / less / head / tail

```bash
cat file.txt                       # 전체 출력(짧을 때)
less file.txt                      # 스크롤 보기(q로 종료)
head -n 50 file.txt                # 위 50줄
tail -n 50 file.txt                # 아래 50줄
tail -f app.log                    # 실시간 로그 보기
```

### 3.2 검색: grep

```bash
grep "ERROR" app.log              # 포함 줄 검색
grep -n "ERROR" app.log           # 라인 번호 포함
grep -i "error" app.log           # 대소문자 무시
grep -r "TODO" src                # 폴더 재귀 검색

grep -E "warn|error" app.log      # 정규식(확장)
```

### 3.3 빠른 텍스트 처리: wc / sort / uniq / cut

```bash
wc -l app.log                      # 줄 수
sort names.txt                     # 정렬
sort names.txt | uniq              # 중복 제거
sort names.txt | uniq -c           # 중복 개수

cut -d "," -f 1 data.csv           # CSV 1번째 컬럼(단순 케이스)
```

### 3.4 편집기: nano / vim

```bash
nano file.txt
vim file.txt
```

- `nano`: 화면에 단축키가 보이는 편
- `vim`: 익숙해지면 빠름 (`i` 입력 → `Esc` → `:wq` 저장 종료)

## 4) 권한/소유자

### 4.1 권한 보기

```bash
ls -l
# -rw-r--r--  1 user group  1234 Feb  5 12:34 file.txt
```

- 3묶음: `user / group / other`
- `r` 읽기, `w` 쓰기, `x` 실행

### 4.2 권한 변경: chmod

```bash
chmod +x script.sh                 # 실행 권한 추가
chmod 644 file.txt                 # (권장 패턴) 일반 파일
chmod 755 script.sh                # (권장 패턴) 실행 파일
```

### 4.3 소유자 변경: chown (보통 sudo 필요)

```bash
sudo chown user:group file.txt
sudo chown -R user:group folder
```

## 5) 프로세스/리소스

### 5.1 실행 중인 프로세스

```bash
ps aux | head
ps aux | grep node

top                                 # 실시간(좀 올드하지만 기본)
htop                                # 있으면 더 편함(미설치일 수 있음)
```

### 5.2 종료/시그널

```bash
kill 1234                           # PID 종료 요청(SIGTERM)
kill -9 1234                        # 강제 종료(SIGKILL, 최후 수단)

pkill node                          # 이름으로 종료
```

### 5.3 디스크/메모리

```bash
df -h                               # 디스크 사용량

du -sh .                            # 현재 폴더 용량
# 큰 폴더 찾기
# du -h --max-depth=1 | sort -h

free -h                             # 메모리(리눅스에 따라 옵션 상이)
```

## 6) 네트워크/다운로드

```bash
ip a                                # 네트워크 정보(구버전은 ifconfig)

curl -I https://example.com         # 헤더만
curl -s https://example.com | head  # 조용히 가져오기

wget https://example.com/file.zip   # 다운로드
```

포트 확인(환경마다 다름):

```bash
# 리눅스 배포판마다 명령이 다를 수 있음
ss -tulpn | head                    # 듣는 포트 확인
```

## 7) 압축/아카이브

### 7.1 tar (리눅스 기본)

```bash
# 압축 만들기
 tar -czf archive.tar.gz folder/

# 풀기
 tar -xzf archive.tar.gz
```

### 7.2 zip

```bash
zip -r archive.zip folder/
unzip archive.zip
```

## 8) 원격(SSH / SCP)

### 8.1 SSH 접속

```bash
ssh user@server-ip
ssh -p 2222 user@server-ip          # 포트 지정
```

### 8.2 파일 전송: scp

```bash
# 로컬 → 서버
scp file.txt user@server-ip:/home/user/

# 서버 → 로컬
scp user@server-ip:/var/log/app.log ./

# 폴더 전송
scp -r dist user@server-ip:/home/user/
```

> 운영 서버에서는 실수 방지를 위해 **경로를 한 번 더 확인**하고 전송하는 습관이 좋아요.

## 9) 한 장 치트시트

### 9.1 자주 쓰는 조합

| 목적 | 추천 조합 |
|---|---|
| 현재 위치/목록 | `pwd`, `ls -al` |
| 파일 찾기 | `find . -name "패턴"` |
| 내용 검색 | `grep -rn "키워드" 폴더` |
| 로그 보기 | `tail -n 200 app.log`, `tail -f app.log` |
| 크기 확인 | `du -sh *`, `df -h` |
| 프로세스 찾기 | `ps aux | grep 이름`, `top` |
| 다운로드 | `curl -O URL`, `wget URL` |
| 압축/해제 | `tar -czf`, `tar -xzf` |

### 9.2 실수 방지 체크

- 삭제 전: `pwd` → `ls` 한 번 확인
- 서버 작업 전: 어떤 서버인지(운영/개발) 프롬프트/호스트명 확인
- 권한 문제: `ls -l`로 권한 확인 후 `chmod`/`chown` 적용
- 파이프 사용 시: `| head`로 먼저 샘플 확인하고 적용
