Agents Subagents

특정 작업을 위한 전문화된 AI 어시스턴트

개념이란?

Subagent는 특정 유형의 작업을 처리하기 위한 전문화된 AI 어시스턴트입니다. 각 서브에이전트는 독립적인 컨텍스트 창에서 실행되며, 커스텀 시스템 프롬프트·특정 도구 접근 권한·독립적인 퍼미션을 가집니다. Claude가 서브에이전트의 설명과 매칭되는 작업을 만나면 해당 서브에이전트에 위임하고, 서브에이전트가 독립적으로 작업하여 결과를 반환합니다.

핵심 장점
컨텍스트 보존

Context Preservation

각 서브에이전트가 독립적인 컨텍스트에서 동작하므로 메인 대화가 오염되지 않고 고수준 목표에 집중됩니다.

전문화

Specialized Expertise

특정 도메인에 최적화된 상세한 지시로 파인튜닝할 수 있어 지정된 작업에서 높은 성공률을 보입니다.

재사용성

Reusability

한번 만든 서브에이전트는 여러 프로젝트에서 재사용하고 팀과 공유하여 일관된 워크플로우를 유지할 수 있습니다.

권한 분리

Flexible Permissions

서브에이전트별로 도구 접근 권한을 다르게 설정할 수 있어 강력한 도구를 특정 서브에이전트에만 제한할 수 있습니다.

파일 위치 및 구조
타입경로범위우선순위
Project.claude/agents/현재 프로젝트만높음
User~/.claude/agents/모든 프로젝트낮음
CLI 임시--agents 플래그현재 세션만중간
.claude/agents/code-reviewer.md
---
name: code-reviewer
description: Expert code review specialist. Proactively reviews code
             for quality, security, and maintainability.
tools: Read, Grep, Glob, Bash
model: inherit          # sonnet | opus | haiku | inherit
permissionMode: default
---

# 시스템 프롬프트 (마크다운으로 자유롭게 작성)
You are a senior code reviewer ensuring high standards.

When invoked:
1. Run git diff to see recent changes
2. Focus on modified files
3. Begin review immediately

Review checklist:
- Code is simple and readable
- No exposed secrets or API keys
- Input validation implemented
- Good test coverage
설정 필드 레퍼런스
필드필수설명
name✅소문자 + 하이픈 사용. 고유 식별자
description✅언제 이 에이전트를 사용할지 자연어로 설명 (자동 위임 기준)
tools❌콤마 구분. 생략 시 메인 쓰레드의 모든 도구 상속
model❌sonnet / opus / haiku / inherit. 기본값 sonnet
permissionMode❌default | acceptEdits | bypassPermissions | plan | ignore
skills❌자동 로드할 스킬 이름 목록 (콤마 구분)
빌트인 서브에이전트
General Purpose

일반 에이전트

복잡한 멀티스텝 작업용. 파일 읽기·쓰기·명령 실행 모두 가능. 모델: Sonnet

Plan

플랜 에이전트

Plan Mode 전용. 코드베이스 탐색 후 계획 수립. 읽기 전용 도구만 사용. 모델: Sonnet

Explore

탐색 에이전트

코드베이스 검색·분석 특화. 완전 읽기 전용. 빠른 응답. 모델: Haiku

사용 방법
서브에이전트 관리 & 호출
# 1. 대화형 관리 인터페이스 열기 (권장)
> /agents

# 2. 명시적으로 특정 에이전트 호출
> Use the code-reviewer subagent to check my recent changes
> Have the debugger subagent investigate this error

# 3. CLI 플래그로 임시 에이전트 정의
claude --agents '{
  "code-reviewer": {
    "description": "Expert code reviewer",
    "prompt": "You are a senior code reviewer...",
    "tools": ["Read", "Grep", "Glob", "Bash"],
    "model": "sonnet"
  }
}'

# 4. 직접 파일 생성
mkdir -p .claude/agents
cat > .claude/agents/test-runner.md << 'EOF'
---
name: test-runner
description: Use proactively to run tests and fix failures
---
You are a test automation expert...
EOF
💡
자동 위임 팁: description 필드에 "Use PROACTIVELY" 또는 "MUST BE USED" 를 포함하면 Claude가 더 적극적으로 서브에이전트를 사용합니다.
🔗
서브에이전트 체이닝: 복잡한 워크플로우에서는 여러 에이전트를 순서대로 연결할 수 있습니다. "First use code-analyzer, then use the optimizer to fix them"

Skills Agent Skills

Claude의 능력을 확장하는 모듈형 역량 패키지

개념이란?

Agent Skills는 전문 지식을 발견 가능한 역량으로 패키징합니다. 각 스킬은 Claude가 관련 상황에서 읽는 지시가 담긴 SKILL.md 파일과 선택적인 스크립트·템플릿으로 구성됩니다. 슬래시 커맨드와 달리 Skills는 모델이 자동으로 호출합니다 — Claude가 요청과 스킬 설명을 기반으로 언제 사용할지 스스로 결정합니다.

파일 위치 및 디렉토리 구조
Personal Skills

~/.claude/skills/

모든 프로젝트에서 사용 가능한 개인 스킬. 개인 워크플로우, 실험적 스킬에 적합.

Project Skills

.claude/skills/

팀과 공유되는 프로젝트 스킬. git에 커밋하면 팀원이 자동으로 사용 가능.

스킬 디렉토리 구조
my-skill/
├── SKILL.md        # 필수 — 메인 지시 파일
├── reference.md    # 선택 — 참조 문서
├── examples.md     # 선택 — 예시 모음
├── scripts/
│   └── helper.py   # 선택 — 유틸리티 스크립트
└── templates/
    └── template.txt # 선택 — 템플릿 파일
SKILL.md 작성법
.claude/skills/code-reviewer/SKILL.md
---
name: code-reviewer              # 소문자+하이픈, 64자 이하
description: Review code for best   # 1024자 이하. 언제 사용할지 구체적으로!
  practices and potential issues.
  Use when reviewing code, checking
  PRs, or analyzing code quality.
allowed-tools: Read, Grep, Glob  # 선택 — 이 스킬 활성화 시 허용 도구
---

# Code Reviewer

## Review checklist

1. Code organization and structure
2. Error handling
3. Performance considerations
4. Security concerns
5. Test coverage

## Instructions
For advanced usage, see [reference.md](reference.md).
SKILL.md 프론트매터 필드
필드필수설명
name✅슬래시 커맨드 이름이 됨. 소문자+숫자+하이픈만 허용 (64자 이하)
description✅Claude가 자동 호출 여부를 판단하는 핵심. 무엇을 하는지 + 언제 쓸지 모두 포함 (1024자 이하)
allowed-tools❌이 스킬이 활성화될 때 허용되는 도구 목록. 미지정 시 기본 퍼미션 모델 따름
disable-model-invocation❌true 설정 시 Claude가 자동으로 이 스킬을 호출하지 않음
context❌fork 설정 시 독립된 컨텍스트에서 실행 (서브에이전트 분기)
agent❌context: fork 사용 시 어떤 에이전트로 실행할지 지정
description 잘 쓰는 법
❌ 나쁜 예

너무 모호

description: Helps with documents

✅ 좋은 예

구체적이고 트리거 포함

description: Extract text and tables from PDF files, fill forms, merge documents. Use when working with PDF files or when the user mentions PDFs, forms, or document extraction.

Hooks Automation

Claude Code 라이프사이클의 특정 시점에 자동으로 실행되는 쉘 명령어

개념이란?

Hooks는 Claude Code 라이프사이클의 특정 시점에 자동으로 실행되는 사용자 정의 쉘 명령어입니다. LLM이 알아서 실행해주길 기대하는 게 아니라 항상 결정론적으로 실행되어야 하는 동작을 보장합니다. 파일 포맷팅, 린트, 로깅, 파일 보호 등에 활용됩니다.

훅 이벤트 종류
이벤트발생 시점차단 가능
PreToolUse도구 호출 전✅
PermissionRequest권한 다이얼로그 표시 시✅ (허용/거부)
PostToolUse도구 호출 완료 후❌
UserPromptSubmit사용자 프롬프트 제출 시 (Claude 처리 전)✅
NotificationClaude Code 알림 전송 시❌
StopClaude Code 응답 완료 시❌
SubagentStop서브에이전트 작업 완료 시❌
PreCompact컨텍스트 압축 실행 전❌
SessionStart새 세션 시작 / 기존 세션 재개 시❌
SessionEnd세션 종료 시❌

종료 코드 의미
Exit Code의미동작
exit 0성공작업 진행. stdout의 JSON 출력 처리됨
exit 2차단 오류작업 차단. stderr 텍스트가 Claude에 피드백으로 전달됨
기타비차단 오류오류 표시 후 실행 계속 (verbose 모드에서만 표시)
설정 방법 (settings.json)
~/.claude/settings.json 또는 .claude/settings.json
{
  "hooks": {
    // PostToolUse — 파일 편집 후 자동 포맷팅
    "PostToolUse": [
      {
        "matcher": "Edit|Write",          // 정규식 매처
        "hooks": [
          {
            "type": "command",
            "command": "jq -r '.tool_input.file_path' | { read f; if echo \"$f\" | grep -q '\\.ts$'; then npx prettier --write \"$f\"; fi; }"
          }
        ]
      }
    ],

    // PreToolUse — 민감한 파일 보호
    "PreToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "python3 -c \"import json, sys; d=json.load(sys.stdin); p=d.get('tool_input',{}).get('file_path',''); sys.exit(2 if any(x in p for x in ['.env', '.git/']) else 0)\""
          }
        ]
      }
    ],

    // Notification — 작업 완료 데스크탑 알림
    "Notification": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "osascript -e 'display notification \"Claude is waiting\" with title \"Claude Code\"'"
          }
        ]
      }
    ]
  }
}

차단 훅 작성 예시 (Bash 명령 검증)
validate-command.sh
#!/bin/bash
INPUT=$(cat)
COMMAND=$(echo "$INPUT" | jq -r '.tool_input.command')

# DROP TABLE 명령 차단
if echo "$COMMAND" | grep -q "drop table"; then
  echo "Blocked: dropping tables is not allowed" >&2
  exit 2   # stderr가 Claude 피드백으로 전달됨
fi

exit 0   # 진행 허용
빠른 시작 — /hooks 커맨드
Claude Code REPL 내에서
# 1. 훅 설정 인터페이스 열기
> /hooks

# 2. PreToolUse 이벤트 선택
# 3. Matcher 입력 (예: Bash, Edit|Write, * 등)
# 4. 실행할 쉘 명령어 입력
# 5. 저장 위치 선택 (User settings / Project settings)

# 예시: Bash 명령 로깅 훅
jq -r '"\(.tool_input.command) - \(.tool_input.description // "No desc")"' \
  >> ~/.claude/bash-command-log.txt

보안 주의: 훅은 현재 환경의 자격 증명으로 자동 실행됩니다. 악의적인 훅 코드는 데이터를 외부로 전송할 수 있습니다. 훅 등록 전 항상 코드를 검토하세요.

PostToolUse

자동 포맷팅

파일 편집 후 prettier, gofmt 등 자동 실행. TypeScript 파일이면 Prettier, Go 파일이면 gofmt.

PreToolUse

파일 보호

.env, package-lock.json, .git/ 등 민감 파일 수정 시도를 exit 2로 차단.

Notification

데스크탑 알림

Claude가 입력을 기다릴 때 OS 알림 전송. macOS, Windows, Linux 모두 지원.

SessionStart

세션 초기화

세션 시작 시 환경 설정, 의존성 확인, 로그 파일 초기화 등 자동 처리.
