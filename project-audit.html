Firestore 보안 규칙 완전 미설정
firestore.rules 파일이 비어 있습니다. Firestore 기본 설정이 "테스트 모드"일 경우 누구나 모든 데이터를 읽고 쓰고 삭제할 수 있습니다. 방명록, 댓글, 조회수 데이터가 완전히 무방비 상태입니다.
firestore.rules (빈 파일) 
// ✅ 최소한 이 정도 규칙은 적용해야 합니다
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // 방명록: 읽기는 누구나, 쓰기는 인증 불필요하지만 필드 검증 강제
    match /guestbook/{docId} {
      allow read: if true;
      allow create: if request.resource.data.keys()
        .hasAll(['name', 'message', 'createdAt'])
        && request.resource.data.message.size() <= 500;
      allow update, delete: if false; // 서버사이드 비밀번호 검증 필요
    }

    // 댓글: 읽기는 누구나, 생성 시 필드 검증
    match /comments/{docId} {
      allow read: if true;
      allow create: if request.resource.data.keys()
        .hasAll(['slug', 'name', 'message'])
        && request.resource.data.message.size() <= 1000;
      allow update, delete: if false;
    }

    // 조회수: 읽기만 허용, 쓰기는 서버에서만
    match /views/{docId} {
      allow read: if true;
      allow write: if false;
    }
  }
}

비밀번호 해싱에 SHA-256 사용
방명록/댓글 수정·삭제용 비밀번호를 SHA-256으로 해싱하고 있습니다. SHA-256은 매우 빠르기 때문에 브루트포스 공격에 취약합니다. bcrypt, scrypt, 또는 Argon2를 사용하는 것이 보안 표준입니다. (단, 브라우저 환경 제약으로 Web Crypto API 기반 PBKDF2를 대안으로 사용 가능)
src/utils/hashPassword.ts 
// ✅ Web Crypto API (브라우저 호환) 기반 PBKDF2 대안
async function hashPassword(password: string, salt: string): Promise<string> {
  const encoder = new TextEncoder();
  const keyMaterial = await crypto.subtle.importKey(
    "raw", encoder.encode(password), "PBKDF2", false, ["deriveBits"]
  );
  const bits = await crypto.subtle.deriveBits(
    { name: "PBKDF2", salt: encoder.encode(salt), iterations: 310000, hash: "SHA-256" },
    keyMaterial, 256
  );
  return btoa(String.fromCharCode(...new Uint8Array(bits)));
}

Img 컴포넌트 alt 속성 미필수화
공통 Img 컴포넌트에서 alt가 optional 타입으로 정의되어 있습니다. alt 없는 이미지는 스크린 리더 사용자에게 의미를 전달하지 못하며, WCAG 2.1 기준 위반입니다. 장식용 이미지는 alt=""를 명시적으로 전달하도록 타입을 강제해야 합니다.
src/components/Img.tsx:4 
// ❌ 현재
interface ImgProps {
  alt?: string  // optional
}

// ✅ 수정 (빈 문자열도 허용해서 장식용 이미지 지원)
interface ImgProps {
  alt: string  // required (장식용이면 "" 전달)
}

user-scalable=no 사용 — 모바일 접근성 위반
index.html의 viewport 메타 태그에 user-scalable=no가 설정되어 있습니다. 저시력 사용자가 브라우저 핀치줌을 사용할 수 없게 되며, WCAG 1.4.4 기준 위반입니다. iOS Safari는 이 설정을 무시하지만 Android Chrome은 그렇지 않습니다.
index.html (viewport meta) 
<!-- ❌ 현재 -->
<meta name="viewport" content="width=device-width, initial-scale=1.0, user-scalable=no">

<!-- ✅ 수정 -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">
